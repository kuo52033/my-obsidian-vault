一種特殊的 complete binary tree，滿足
```
Min Heap：每個父節點 ≤ 子節點 → 根節點永遠是最小值
Max Heap：每個父節點 ≥ 子節點 → 根節點永遠是最大值
```

底層用陣列儲存
```
[1, 3, 2, 7, 4, 5, 6]

對於 index i
左子子節點 = 2 * i + 1
右子子節點 = 2 * i + 2
父節點 = (i-1) // 2

		1          (index 0)
      /   \
     3     2       (index 1, 2)
    / \   / \
   7   4 5   6     (index 3, 4, 5, 6)
```
## 複雜度

| 操作           | 複雜度      | 原因                    |
| ------------ | -------- | --------------------- |
| heappush     | O(log n) | sift up，最多走 log n 層   |
| heappop      | O(log n) | sift down，最多走 log n 層 |
| heapify      | O(n)     | 底層節點多但移動少             |
| 查看極值 heap[0] | O(1)     | 直接讀根節點                |

---
### heappush 運作方式（sift up）

```
新元素加到陣列最後
→ 跟父節點比較，比父節點小就交換
→ 一直往上直到滿足 heap 條件

push 1 進 [2, 3, 4]：
[2, 3, 4, 1]
index 3，父節點 index 1（值=3）
1 < 3 → 交換：[2, 1, 4, 3]
index 1，父節點 index 0（值=2）
1 < 2 → 交換：[1, 2, 4, 3] ✓
```

---
### heappop 運作方式（sift down）

```
取出根節點
→ 把最後一個元素移到根
→ 跟子節點比較，比子節點大就跟較小的子節點交換
→ 一直往下直到滿足 heap 條件

pop [1, 3, 2, 7, 4, 5, 6]：
取出 1，把 6 移到根：[6, 3, 2, 7, 4, 5]
6 > min(3,2)=2 → 跟 2 交換：[2, 3, 6, 7, 4, 5]
6 > 5 → 跟 5 交換：[2, 3, 5, 7, 4, 6] ✓
```
