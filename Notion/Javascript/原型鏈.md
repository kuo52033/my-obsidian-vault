Javascript 並沒有類別( class )以及實體( instance )會主動繼承的概念，即使在 ES6 出現class關鍵字，也只是個語法糖(對語法的功能沒有影響，為了更方便人使用)， 因此如果要建立實體，就是利用 new 一個 constructor(建構函式)。如下，也可以在 constructor 中加入方法，但問題是這個方法雖然是在做同一件事，但其實還是佔用了多個空間，變成不同的函式。

```javascript
function Person(name, age) {
  this.name = name;
  this.age = age;
  this.log = function () {
    console.log(this.name + ', age:' + this.age);
  }
}
  
var nick = new Person('nick', 18);
nick.log(); // nick, age:18
  
var peter = new Person('peter', 20);
peter.log(); // peter, age:20

console.log(nick.log === peter.log) // false
```

每個 constructor 都有屬於自己的一個模型，成為原型( Prototype )，而我們可以在該原型上新增一個屬性，使的該屬於可以被 construtor 建構出的實體分享。

```javascript
Person.prototype.log = function(){
	 console.log(this.name + ', age:' + this.age);
}
var nick = new Person('nick', 18);
var peter = new Person('peter', 20);
console.log(nick.log === peter.log) // true
```

## **小結**

- Javascript沒有類別，也沒有建構子。創建物件靠的是函式
- 只要在new之後的函式就是構建函式
- 在原型中加入屬性可使實體共用該屬性
- 非必要請不要更改原生原型

有關於上述 nick 是如何找到 log function 的，因為 Person 本身是沒有 log 這個 function 的，因此他會往 Person.prototype 去找，而將 nick 以及 Person.prototype 連接在一起的方法是 __**proto**__，nick.__proto__ 等於 Person.prototype，所以當 nick裡面沒有 log 這個 funciton 時，會往 nick.__proto__ 去找，如果還是沒有的話，再往 Person.prototype.__proto__ 去找，直到某個 __proto__ 指向 null 為止，代表到了最上層。

## __**proto**__ 指向必定為某一個 prototype。某某.__proto__ 指的是找該原型， 某某.prototype  指的是利用某某所建立新物件的原型。

這一連串透過 __**proto**__ 連接的鏈，就叫做原型鏈，可以達到繼承的功能，也可以呼叫 parent 的 method。

**小結**

- `__proto__`會指向某個東西的prototype，以利Javascript向上查找屬性
- `Object.prototype`是原型鏈的頂點
- 原型鏈可達成繼承的效果，但不等於Javascript具有類別

instanceof 可以用來檢測某物件是否為指定的建構函式所指定的實體，例如: Function instanceof Object // true ，左邊為被檢測的物件，右方為指定的建構函式，而 instanceof 會往上搜查原型鏈，找的到就算 true。
