## 程式碼目標 🎯

根據 **前序遍歷 (Preorder)** 和 **中序遍歷 (Inorder)** 的結果，重建出一棵二元樹。

- **Preorder (根 -> 左 -> 右)**：決定**誰是當前的根節點**（依序讀取即可）。
- **Inorder (左 -> 根 -> 右)**：決定**左右子樹的範圍**（透過根節點的位置來切割）。

---

## 核心優化技巧 🚀

這份程式碼使用了兩個關鍵技巧來提升速度：

1. **Hash Map (**`**inorder_map**`**)**：
    - 原本我們每次都要用 `inorder.index(val)` 找位置，這是 $O(N)$ 的動作。
    - 現在預先建立一個 `數值 -> 索引` 的對照表，查找變成 **$O(1)$**。
2. **索引指標 (**`**helper(left, right)**`**)**：
    - 原本使用 `preorder[1:mid+1]` 會複製產生新陣列，這是昂貴的記憶體操作。
    - 現在我們只傳遞 `left` 和 `right` 兩個整數，告訴遞迴函式「請關注 `inorder` 陣列的哪一段範圍」，完全不消耗額外記憶體。

## 程式碼詳解 Breakdown

Python

```python
class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        
        # 1. 建立快速查找表 (Hash Map)
        #    key: 節點的值, value: 在 inorder 陣列中的索引
        #    目的：讓我們能瞬間知道 "根節點" 把 inorder 切成兩半的位置
        inorder_map = {val: idx for idx, val in enumerate(inorder)}
        
        # 2. 全域指標 (Global Index)
        #    用來追蹤我們目前讀取到 preorder 的第幾個元素
        #    因為 preorder 的順序就是「根 -> 左 -> 右」，
        #    所以我們只需要一個指針一直往後移動即可。
        self.curr_idx = 0

        # 3. 定義遞迴函式
        #    left, right 代表當前子樹在 "inorder" 中的範圍 (包含 left 和 right)
        def helper(left, right):
            
            # Base Case: 如果左邊界超過右邊界，表示範圍內沒有元素了 (空子樹)
            if left > right:
                return None
            
            # A. 建立根節點
            #    從 preorder 拿出當前的數字
            root_val = preorder[self.curr_idx]
            node = TreeNode(root_val)
            
            # B. 找到切割點 (mid)
            #    利用 Map 快速找出這個根節點在 inorder 的哪個位置
            mid = inorder_map[root_val]
            
            # C. 移動 preorder 指標
            #    準備處理下一個節點 (這一步很重要，必須在遞迴之前做)
            self.curr_idx += 1

            # D. 遞迴建立左右子樹
            #    利用 mid 把 inorder 範圍切開
            
            # 左子樹範圍：[left, mid - 1]
            node.left = helper(left, mid - 1)
            
            # 右子樹範圍：[mid + 1, right]
            node.right = helper(mid + 1, right)

            return node

        # 4. 啟動遞迴
        #    一開始的範圍是 inorder 的全部 (0 到 len-1)
        return helper(0, len(inorder) - 1)
```

## 複雜度分析

假設樹的節點總數為 $N$。

### ⏰ 時間複雜度: $O(N)$

- 建立 Map 需要遍歷一次 `inorder`: $O(N)$。
- 遞迴過程中，每個節點都會被 `helper` 函式拜訪並建立一次: $O(N)$。
- Map 的查找是 $O(1)$。
- 總和 = $O(N)$。

### 💾 空間複雜度: $O(N)$

- `inorder_map` 儲存了 $N$ 個元素: $O(N)$。
- 遞迴的呼叫堆疊 (Call Stack) 在最壞情況下 (樹長得像一直線) 為 $O(N)$。
- 總和 = $O(N)$。
