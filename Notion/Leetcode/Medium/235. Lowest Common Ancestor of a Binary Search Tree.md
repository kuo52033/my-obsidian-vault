## 📌 題目重點

- 給定一棵 **Binary Search Tree（BST）**
- 以及兩個節點 `p`、`q`
- 找出 **最近的共同祖先（Lowest Common Ancestor, LCA）**
- BST 性質：
    - 左子樹 < root < 右子樹

---

## 🧠 解題核心思路（利用 BST 性質）

### 為什麼 BST 可以直接判斷？

在 BST 中：

- 若 `p`、`q` 都 **小於 root**
👉 LCA 一定在 **左子樹**
- 若 `p`、`q` 都 **大於 root**
👉 LCA 一定在 **右子樹**
- 否則（一大一小，或其中一個等於 root）
👉 **root 就是 LCA**

---

## 🧩 使用到的資料結構

- **Binary Search Tree**
- **Recursion**
    - 每次只往一個子樹走（不是全樹 DFS）

---

## 🧪 程式碼解析

```python
classSolution:
	def lowestCommonAncestor(self, root: 'TreeNode', p:'TreeNode', q:'TreeNode') ->'TreeNode':


```

---

### 🔹 Case 1：p、q 都在左子樹

```python
if root.val > p.valand root.val > q.val:
		return self.lowestCommonAncestor(root.left, p, q)


```

- root 比 p、q 都大
- LCA 不可能是 root 或右子樹
- 繼續往左找

---

### 🔹 Case 2：p、q 都在右子樹

```python
if root.val < p.valand root.val < q.val:
	return self.lowestCommonAncestor(root.right, p, q)
```

- root 比 p、q 都小
- LCA 一定在右子樹

---

### 🔹 Case 3：分岔點（答案）

```python
return root
```

以下任一情況都會走到這行：

- `p < root < q`
- `q < root < p`
- `root == p` 或 `root == q`

👉 **當前 root 就是最近共同祖先**

---

## ⏱️ 時間複雜度

- 每一層只往一邊走
- 走訪高度 = 樹高 `h`

✅ **Time Complexity：O(h)**

- 平衡 BST：O(log n)
- 最壞（退化成鏈）：O(n)

---

## 💾 空間複雜度

- 遞迴呼叫堆疊深度 = 樹高

✅ **Space Complexity：O(h)**

---

## 🎯 解題技巧總結（面試必背）

- **BST 的 LCA = 比較大小就好**
- 不需要 DFS 全樹
- 只要找到「第一次分岔的位置」
- 題目一看到：
    - BST
    - LCA
👉 立刻想到「左右大小判斷」

---

## 🔄 常見錯誤

- 把 BST 當成一般 Binary Tree（用 DFS 全掃）
- 忘記 root 本身可能就是 LCA
- 先找 p 路徑、再找 q 路徑（多餘）

## 🔁 Iterative 版本（延伸）

```python
while root:
if root.val > p.valand root.val > q.val:
        root = root.left
elif root.val < p.valand root.val < q.val:
        root = root.right
else:
return root


```

- Time：O(h)
- Space：O(1)
- 面試加分 ⭐
