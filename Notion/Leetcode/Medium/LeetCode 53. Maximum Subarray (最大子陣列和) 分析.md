### 1. 難度 (最高五星)

- **難度：** ★★☆☆☆ (2/5)
- **說明：** 這是 LeetCode 上的 "Easy" (現在是 "Medium") 題目，但它可能是你學「動態規劃」時會遇到的**第一個經典範例**。它的難點不在於程式碼的長度，而在於 `curr = max(curr + num, num)` 這一行 DP 狀態轉移的巧妙思想。

### 2. 運用到的技巧

1. **動態規劃 (Dynamic Programming):**
    - 你的解法沒有開一個 `dp` 陣列，而是巧妙地用了 `curr` 這一個變數來儲存「滾動的」DP 狀態，使空間複雜度從 $O(N)$ 優化到了 $O(1)$。
2. **貪心法 (Greedy Approach):**
    - 卡登演算法也可以被視為一種貪心法。在每一步，你都做出「局部最優」的選擇：
        - 「我是要*加入*前一個子陣列？」
        - 「還是要*自立門戶*，從我這裡開始一個新陣列？」

### 3. 核心邏輯分析 (你寫法的精妙之處)

你的程式碼有兩個關鍵變數：

- `result`：儲存「**全局最優解**」。也就是到目前為止，我們在*任何*位置找到的*最大*子陣列和。
- `curr`：儲存「**局部最優解**」。這是 DP 的精髓，它代表「**以 *****當前***** **`**num**`** 這個元素結尾**的子陣列，所能得到的*最大*和」。

**關鍵的 **`**for**`** 迴圈：**

Python

`for num in nums:
    # 1. 核心 DP 狀態轉移
    curr = max(curr + num, num)
    
    # 2. 更新全局最優解
    result = max(result, curr)`

**1. **`**curr = max(curr + num, num)**`** 這一行在做什麼？**

- 它在回答一個問題：「**對於目前的 **`**num**`**，我是該『加入前面的團隊』還是『自己重組一個團隊』？**」
- **選項 A (**`**curr + num**`**)：**
    - **意義：**「加入團隊」。
    - `curr` 是「以上一個元素結尾」的最大和。如果 `curr` 是正數 (例如 10)，那麼 `curr + num` 就代表「把 `num` 加進來，讓這個子陣列繼續*延伸*」。
- **選項 B (**`**num**`**)：**
    - **意義：**「自己重組團隊」。
    - **時機：** 什麼時候「加入團隊」會更糟？當 `curr` (前面團隊的總和) 是**負數**時 (例如 -5)。
    - 如果 `curr = -5`，`num = 3`，那麼 `curr + num = -2`。但如果我「拋棄」前面的負數，自己從 `num` 開始一個新的子陣列，我的和是 `3`。
    - `max(-2, 3)` 會選擇 `3`。這一步就巧妙地「**拋棄了所有會拖累我的負數前綴**」。

**2. **`**result = max(result, curr)**`** 這一行在做什麼？**

- `curr` 只記錄「*必須*以我 ( `num` ) 結尾」的最大和。
- 但「全局」的最大和，可能在 `curr` 之前就出現了。
- **範例：** `[10, -1, 3]`
    - `num=10`: `curr=10`, `result=10`
    - `num=-1`: `curr=9` (選 `10 + -1`), `result=10`
    - `num=3`: `curr=12` (選 `9 + 3`), `result=12`
- **範例：** `[10, -50, 3]`
    - `num=10`: `curr=10`, `result=10`
    - `num=-50`: `curr=-40` (選 `10 + -50`)，`result=10`
    - `num=3`: `curr=3` (選 `3`，拋棄 `40`)，`result=10`
- `result` 會記住那個曾經出現過的 `10`，即使 `curr` 後來變小了也沒關係。

### 4. 時間與空間複雜度

3. **時間複雜度 (Time Complexity): $O(N)$**
    - $N$ 是 `nums` 陣列的長度。
    - 你只用了一個 `for` 迴圈，對陣列進行了**一次**完整的遍歷 (Single Pass)。
4. **空間複雜度 (Space Complexity): $O(1)$**
    - 你只使用了 `result` 和 `curr` 兩個額外的變數，不論 $N$ 多大，你用的空間都是固定的常數。

### 程式碼逐行說明 (你的程式碼)

Python

```python
class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        
        # 1. 初始化 "全局最大和"
        #    設為 nums[0]，確保能處理 [-1, -2] 這種全負數的情況
        result = nums[0] 
        
        # 2. 初始化 "當前最大和" (以當前元素結尾)
        #    設為 0，這樣第一個 num 進來時，
        #    max(0 + num, num) 就會是 num
        curr = 0 
        
        # 3. 遍歷 nums 中的每一個數字
        for num in nums:
            
            # 4. [DP 核心] 決定 "curr" 的新值
            #    (A) 繼續延伸： curr + num
            #    (B) 重新開始： num
            #    取兩者中的較大值
            curr = max(curr + num, num)
            
            # 5. [更新答案]
            #    "以 num 結尾" 的最大和 (curr) 
            #    是否 "全局" 最大和 (result) 更大？
            result = max(result, curr)

        # 6. 返回全局最大
```
