### 1. 難度 (最高五星)

- **難度：** ★★☆☆☆ (2/5)
- **說明：** 這是一道 "Medium" 難度的題目。它不考高深的演算法，而是考察你**操作 2D 陣列 (Matrix) 的基本功**，以及你是否能想出 $O(1)$ 額外空間的 "in-place" 解法。

### 2. 運用到的技巧

你的解法巧妙地將「**順時針旋轉 90 度**」這個複雜動作，拆解為兩個獨立且更簡單的步驟：

1. **原地矩陣轉置 (In-place Transpose)**：沿著「左上到右下」的對角線進行翻轉。
2. **原地水平翻轉 (In-place Horizontal Reverse)**：將矩陣的每一行進行左右翻轉。

### 核心邏輯分析 (你的程式碼)

### 步驟一：原地轉置 (In-place Transpose)

Python

`n = len(matrix)

# 1. 遍歷 "下三角" (包含對角線)
for i in range(n):
    # j 只跑到 i+1，即 j <= i
    for j in range(0, i+1): 
        # 2. 沿著 "左上到右下" 的對角線交換
        matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]`

- **分析：** 這是矩陣轉置的標準 $O(1)$ 空間做法。
    - 「轉置」就是把 `(i, j)` 和 `(j, i)` 互換。
    - 你非常聰明地只遍歷了「**下三角**」 (`j` 從 `0` 到 `i`)。如果遍歷整個矩陣 (`j` 從 `0` 到 `n-1`)，每個元素會被交換兩次，等於又換了回來。
- **範例 **`**(i=2, j=0)**`**：** `matrix[2][0]` (例如 7) 會和 `matrix[0][2]` (例如 3) 交換。
- **此步驟結果 (以 3x3 為例)：**
`[1, 2, 3]      [1, 4, 7]
[4, 5, 6]  --> [2, 5, 8]
[7, 8, 9]      [3, 6, 9]`

### 步驟二：原地水平翻轉 (In-place Reverse)

Python

`# 3. 遍歷 "每一行" (row)
for i in range(n):
    left = 0
    right = n-1

    # 4. 使用 "雙指針" 翻轉該行
    while left < right:
        matrix[i][left], matrix[i][right] = matrix[i][right], matrix[i][left]
        left += 1
        right -= 1`

- **分析：** 你沒有使用 `.reverse()` 函式，而是手動用「雙指針」實現了原地翻轉，這更體現了你對 in-place 操作的理解。
- **範例 **`**(i=0)**`**：**
    - `[1, 4, 7]`
    - `left=0` (1), `right=2` (7) $\to$ 交換 $\to$ `[7, 4, 1]`
- **此步驟結果 (最終答案)：**
`[1, 4, 7]      [7, 4, 1]
[2, 5, 8]  --> [8, 5, 2]
[3, 6, 9]      [9, 6, 3]`

**結論：** (轉置) + (水平翻轉) = (順時針旋轉 90 度)。你的邏輯完美。

### 4. 時間與空間複雜度

3. **時間複雜度 (Time Complexity): $O(N^2)$**
    - $N$ 是矩陣的邊長。
    - **轉置：** 你遍歷了矩陣的「下三角」，大約 $\frac{N^2}{2}$ 次操作，即 $O(N^2)$。
    - **翻轉：** 你遍歷了 $N$ 行，每行 $N/2$ 次交換，總共 $\frac{N^2}{2}$ 次操作，即 $O(N^2)$。
    - 總時間 = $O(N^2) + O(N^2) = O(N^2)$。這是必須的，因為你至少要碰每個元素一次。
4. **空間複雜度 (Space Complexity): $O(1)$**
    - 你**沒有**使用任何額外的 `temp` 陣列。
    - 你只用了 `n`, `i`, `j`, `left`, `right` 這幾個變數，它們佔用的空間是固定的常數。
    - 這完美地達成了題目 "modify matrix in-place instead" 的要求。
