## 📌 題目重點

- 給定股價陣列 `prices`
- 可以多次買賣股票
- **每次賣出後，隔一天才能再買（cooldown = 1 day）**
- 同一時間只能持有 **一張股票**
- 求最大利潤

---

## 🧠 解題核心思路（DP with 3 States）

### 為什麼需要 3 個狀態？

因為「沒股票」其實分成兩種不同情況：

1. 沒股票，**而且可以買**
2. 沒股票，**但昨天剛賣（冷卻中，不能買）**

為了記住這個差異，必須拆成三個狀態。

---

## 🧩 三個狀態定義（每天「收盤時」）

在第 `i` 天結束時：

- **buy**
👉 手上 **有股票** 時的最大利潤
（可能是今天買的，也可能之前買的一直拿著）
- **sold**
👉 **今天剛賣掉股票** 的最大利潤
（明天會進入 cooldown）
- **rest**
👉 手上 **沒股票，且不在 cooldown** 的最大利潤
（明天可以買）

> 💡 關鍵：
> **只要手上有股票，就一定是 buy，不可能是 rest**

---

## 🔁 狀態轉移邏輯（人話版）

### 🟢 buy（今天收盤時持有股票）

可能來源：

3. 昨天就有股票，今天什麼都不做
4. 昨天是 rest（可買），今天買入

```plain text
buy = max(昨天 buy, 昨天 rest - price)


```

❌ 不能從 `sold` 來（因為 cooldown）

---

### 🔴 sold（今天剛賣）

可能來源：

5. 昨天一定要持有股票，今天賣出

```plain text
sold = 昨天 buy + price


```

---

### 🟡 rest（今天沒股票且可買）

可能來源：

6. 昨天就在 rest，今天繼續休息
7. 昨天剛賣（sold），今天冷卻一天

```plain text
rest = max(昨天 rest, 昨天 sold)


```

---

## 🧪 程式碼（O(1) 空間版本）

```python
classSolution:
defmaxProfit(self, prices: List[int]) ->int:
        buy = -prices[0]
        sold =0
        rest =0

for iinrange(1,len(prices)):
            temp = sold# 保存 sold[i-1]
            sold = buy + prices[i]# sold[i]
            buy =max(buy, rest - prices[i])# buy[i]
            rest =max(rest, temp)# rest[i]

returnmax(rest, sold)


```

---

## 🔍 為什麼要用 `temp = sold`？

因為：

- `rest[i]` 需要用到 **昨天的 sold**
- 但 `sold` 會在本輪被更新

👉 所以要先把 `sold[i-1]` 存起來，避免被覆蓋。

---

## 🧠 對照「陣列版 DP」的概念

這個寫法其實等同於：

```python
buy[i]  =max(buy[i-1], rest[i-1] - price)
sold[i] = buy[i-1] + price
rest[i] =max(rest[i-1], sold[i-1])


```

只是把 `buy[i-1] / sold[i-1] / rest[i-1]`

壓縮成三個變數在滾動更新。

✅ **Time Complexity：O(n)**

- 只使用三個變數

✅ **Space Complexity：O(1)**

## 🎯 解題關鍵總結（面試必背）

- cooldown 的本質：**昨天賣，今天不能買**
- 為了記住「昨天是否剛賣」，必須有 `sold` 狀態
- `buy` 可以一路延續（代表長期持有）
- 答案一定在 `rest` 或 `sold`，不會是 `buy`

---

## 🔄 常見錯誤

- 把「中間不操作」誤以為是 rest
👉 只要手上還有股票，就是 buy
- 忘記 `sold → rest` 的 cooldown 緩衝
- 更新順序錯誤，覆蓋掉昨天的狀態
