## ⭐ **難度：Easy (3/10)**

## 📌 **面試常考程度：9/10**

這題是 **Stack 類型的超高頻面試題**，幾乎是：

- Stack 入門必考
- 系統設計（追問 O(1)）常見題
- 很多題目的基礎（如 Max Stack）

---

## 🎯 **題意**

設計一個 stack，支援以下操作，**全部都必須是 O(1)**：

- `push(val)`
- `pop()`
- `top()`
- `getMin()` → 回傳目前 stack 中的最小值

---

## ❌ **直覺但錯誤的作法**

- 每次 `getMin()` 都去掃整個 stack
→ **O(n)** ❌

---

## 💡 **正確核心觀念**

### 👉 **在 push 的時候，就把「到目前為止的最小值」一起存起來**

這樣：

- `getMin()` 只要看 stack top
- 完全不需要額外計算

---

## 🧠 **資料結構設計**

Stack 中的每個元素存兩個資訊：

```plain text
{
  val: 當前值,
  m: 目前 stack 的最小值
}


```

也就是：

- `m = min(自己, 前一個最小值)`

---

## 🧩 **你的實作（正確且推薦）**

```python
classMinStack:

def__init__(self):
self.stack = []

defpush(self, val: int) ->None:
iflen(self.stack):
            minVal =min(self.stack[-1]['m'], val)
self.stack.append({'val': val,'m': minVal})
else:
self.stack.append({'val': val,'m': val})

defpop(self) ->None:
self.stack.pop()

deftop(self) ->int:
returnself.stack[-1]['val']

defgetMin(self) ->int:
returnself.stack[-1]['m']


```

---

## 🔍 **為什麼這個解法是 O(1)？**

### `push`

- 只做一次比較 → O(1)

### `pop`

- 直接 pop → O(1)

### `top`

- 讀 stack[-1] → O(1)

### `getMin`

- 直接回傳 stack[-1]['m'] → O(1)

---

## ⏱ **時間複雜度**

| 操作 | 複雜度 |
| --- | --- |
| push | O(1) |
| pop | O(1) |
| top | O(1) |
| getMin | O(1) |

---

## 📦 **空間複雜度**

### **O(n)**

每個元素多存一個 `min` 值。

---

## ⚠️ **常見陷阱**

- pop 之後忘記更新 min（本解法自動處理）
- 用全域 min 變數（pop 時會壞）
- 用 list 每次掃最小值（O(n)）

---

## 🔄 **另一種常見寫法（雙 Stack 法）**

補充觀念（Notion 可選）：

- 一個 stack 存值
- 一個 stack 存 min
- 當新值 ≤ min 時，同步 push

你目前的寫法其實是 **把兩個 stack 合成一個**，更精簡。

---

## 📘 **一句話總結（Notion 摘要）**

> Min Stack 的關鍵是在 push 時同步記錄當前最小值，
讓 getMin() 永遠只需 O(1) 查看 stack top。
