## 程式碼目標 🎯

程式碼的目標是逐層 (level by level) 拜訪一個二元樹，並將**每一層**的節點值 (val) 收集到一個子陣列中，最後返回一個包含所有層級的陣列 (list of lists)。

## 核心概念：廣度優先搜尋 (BFS)

BFS 的核心精神是使用一個「**佇列 (Queue)**」。

1. 從「根節點」開始，把它放入佇列。
2. 只要佇列不是空的：
a.  從佇列中取出一個節點 (FIFO: 先進先出)。
b.  處理這個節點 (例如：讀取它的值)。
c.  將這個節點的子節點 (左、右) 加入佇列。

而「層序遍歷」是 BFS 的一個變體，我們需要知道「**何時一層結束了，下一層開始了**」。你的程式碼透過 `temp` 列表巧妙地解決了這個問題。

## 程式碼詳解 breakdown

Python

```python
from collections import deque
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        # 1. 邊界情況：如果樹是空的，返回空陣列
        if not root:
            return []

        result = []
        # 2. 建立一個 "雙向佇列" (deque)，在 Python 中
        #    它作為佇列 (Queue) 的效率最高
        queue = deque()
        # 3. 將根節點 (第 0 層) 放入佇列，開始 BFS
        queue.append(root)

        # 4. 外層迴圈：
        #    只要佇列中還有節點 (表示還有 "下一層" 需要處理)，就繼續
        while len(queue):
            # 5. 為 "當前層" 準備兩個空列表：
            temp = []     
            #    curr_level: 存放 "當前層" 節點的 "值" (val)
            curr_level = []

            # 6. 內層迴圈：
            #    這個迴圈會執行 N 次 (N 是 "當前層" 的節點數)
            #    它會 "清空" 目前的 queue，把裡面的節點 (當前層)
            #    一個一個拿出來處理
            while len(queue):
                # 7. 從佇列左邊 (頭部) 取出一個節點 (FIFO)
                node = queue.popleft()
                # 8. 將節點的 "值" 加入 curr_level
                curr_level.append(node.val)
                
                # 9. 關鍵步驟：
                #    如果這個節點有子節點，就把它們加入 "temp" 列表
                #    "temp" 列表收集的
                #    就是 "下一層" 的所有節點
                if node.left:
                    temp.append(node.left)
                
                if node.right:
                    temp.append(node.right)
            
            # 10. (內層迴圈結束)
            #     此時，"當前層" 已處理完畢 (queue 已空)
            #     "當前層的值" 已存入 curr_level
            #     "下一層的節點" 已存入 temp
            result.append(curr_level)
            
            # 11. 最重要的步驟："交接"
            #     將 temp (下一層的節點) 轉換為 deque
            #     並重新賦值給 queue，
            #     以供 "外層迴圈" 的下一次迭代使用
            queue = deque(temp)

        # 12. (外層迴圈結束)
        #      當最後一層處理完，它們的子節點 (None) 不會被加入 temp
        #      所以 temp 會是空的，queue = deque(temp) 也會是空的
        #      len(queue) == 0，迴圈停止
        return result
```

## 複雜度分析

假設樹中有 $N$ 個節點。

### ⏰ 時間複雜度: $O(N)$

- 每個節點都會被 `append` (加入) 到 `queue` (或 `temp`) 中**一次**。
- 每個節點都會被 `popleft` (取出) **一次**。
- 所有的操作都是 $O(1)$，所以總時間複雜度是 $O(N)$。

### 💾 空間複雜度: $O(W)$ 或 $O(N)$

- 空間複雜度由 `queue` (或 `temp` 列表) 的最大長度決定。
- 在 BFS 中，佇列在任一時刻儲存的節點最多是「樹最寬的那一層」的節點數。我們稱之為 $W$ (Width)。
- 在**最壞情況**下（一個「完美二元樹」或「茂密」的樹），最後一層的節點數 $W$ 可能接近 $N/2$。
- 因此，空間複雜度可以寫作 $O(W)$，或者更寬鬆地寫作 $O(N)$。
