## ⭐ **難度：Medium (5/10)**

## 📌 **面試常考程度：9/10**

這題是 **Grid + DFS/BFS 的超高頻經典題**，

後續大量題目（130、695、417、1254…）都以此為基礎。

---

## 🎯 **題意**

給定一個由 `'1'`（陸地）與 `'0'`（海水）組成的 2D grid，

請計算 **島嶼的數量**。

規則：

- 島嶼由 **上下左右** 相連的 `'1'` 組成
- 對角線 **不算相連**

---

## 💡 **核心觀念**

### 🔑 轉換問題

> 「數島嶼」
> ⟹ 「數有多少個 **連通區塊**」

每遇到一塊 **尚未拜訪過的陸地 **`**'1'**`，

就用 DFS / BFS 把整個島嶼走完，並計數 +1。

---

## 🧠 **你的解法（DFS + visited）**

```python
classSolution:
defnumIslands(self, grid: List[List[str]]) ->int:
self.visited = [[False] *len(grid[0])for _inrange(len(grid))]
self.grid = grid

        total =0

for iinrange(len(grid)):
for jinrange(len(grid[0])):
                total +=self.dfs(i, j)

return total

defdfs(self, x, y):
if x >=len(self.grid)or x <0or y >=len(self.grid[0])or y <0:
return0

ifself.visited[x][y]:
return0

self.visited[x][y] =True

ifself.grid[x][y] =='0':
return0

self.dfs(x-1, y)
self.dfs(x, y-1)
self.dfs(x, y+1)
self.dfs(x+1, y)

return1


```

---

## 🔍 **程式碼解析**

### 1️⃣ 外層雙迴圈

```python
for iinrange(len(grid)):
for jinrange(len(grid[0])):
        total +=self.dfs(i, j)


```

- 嘗試從每個格子出發
- `dfs()` 只會在「新島嶼起點」回傳 `1`
- 其餘情況回傳 `0`

---

### 2️⃣ DFS 的三層防護（非常重要）

```python
if out of bounds →return0
if visited →return0
if grid[x][y] =='0' →return0


```

👉 只有在「**未拜訪過的陸地**」才會往下擴散。

---

### 3️⃣ 為什麼 `return 1` 是正確的？

```python
return1


```

代表：

> 我剛剛發現了一座新島嶼的起點

之後的 DFS 只是把同一座島走完，不再重複計數。

---

## ⏱ **時間與空間複雜度**

- **時間複雜度**：`O(m × n)`
    - 每個格子最多被 DFS 一次
- **空間複雜度**：`O(m × n)`
    - `visited` 陣列 + recursion stack（最壞情況）

---

## 🔧 **可優化點（進階但常見）**

### ✅ 1️⃣ 不一定要 `visited` 陣列

可以直接 **修改原 grid**（節省空間）：

```python
grid[x][y] ='0'


```

這是 LeetCode 官方很常見的寫法。

---

### ✅ 2️⃣ 更直觀的寫法（只在遇到 '1' 時 DFS）

```python
for iinrange(m):
for jinrange(n):
if grid[i][j] =='1':
            dfs(i, j)
            count +=1


```

概念會更貼近「數島嶼」。

---

## ⚠️ **常見錯誤**

- 忘記處理邊界
- 把對角線也算成相連
- visited 設定時機錯誤
- DFS 回傳值設計不清楚

---

## 🔄 **DFS vs BFS**

| 方法 | 特點 |
| --- | --- |
| DFS | 寫法簡單、直觀 |
| BFS | 不會 stack overflow（grid 很大時） |

---

## 📘 **一句話總結（Notion 摘要）**

> Number of Islands 的本質是：
在 grid 中找有多少個「尚未拜訪過的連通陸地區塊」。
