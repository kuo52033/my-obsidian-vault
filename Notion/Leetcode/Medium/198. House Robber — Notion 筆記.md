## ⭐ **難度：Easy–Medium (4/10)**

## 📌 **面試常考程度：9/10**

這題是 **Dynamic Programming（DP）入門必考題**，

後續很多題（213、337）都是從這題延伸。

---

## 🎯 **題意**

有一排房子，每間房子有金額 `nums[i]`。

如果你偷了第 `i` 間，就不能偷相鄰的第 `i-1`、`i+1` 間。

👉 求 **最多可以偷到多少錢**。

---

## 💡 **核心思路：DP（選或不選）**

每一間房子只有兩種選擇：

1. **偷這一間** → 不能偷前一間
2. **不偷這一間** → 延續前一間的最大收益

---

## 🧠 **DP 狀態定義**

```plain text
dp[i] = 偷到第 i 間房子為止，可以拿到的最大金額
```

---

## 🔄 **狀態轉移公式（重點）**

對於第 `i` 間房子：

```plain text
dp[i] =max(
    dp[i-1],# 不偷第 i 間
    dp[i-2] + nums[i]# 偷第 i 間
)
```

---

## 🧩 **初始化**

```python
dp[0] = nums[0]
dp[1] =max(nums[0], nums[1])
```

- 第 0 間：只能偷它
- 第 1 間：偷第 0 或第 1，選大的

---

## 🧠 **你的解法（正確且標準）**

```python
classSolution:
defrob(self, nums: List[int]) ->int:
iflen(nums) ==1:
return nums[0]

        dp = [0 for _inrange(len(nums))]

        dp[0] = nums[0]
        dp[1] =max(nums[1], nums[0])

for iinrange(2,len(nums)):
            dp[i] =max(dp[i-2] + nums[i], dp[i-1])

return dp[-1]
```

---

## 🔍 **程式碼解析**

- 使用 `dp` 陣列紀錄每一步最佳解
- 每一間只依賴前兩個狀態
- 最後答案一定在 `dp[-1]`

---

## ⏱ **時間與空間複雜度**

- **時間複雜度**：`O(n)`
- **空間複雜度**：`O(n)`

---

## 🔧 **空間優化（進階）**

因為 `dp[i]` 只依賴 `i-1` 和 `i-2`，

可以用兩個變數把空間優化成 `O(1)`。

### **優化版（可背）**

```python
classSolution:
	def rob(self, nums: List[int]) ->int:
	        prev2 =0# dp[i-2]
	        prev1 =0# dp[i-1]
	
	for n in nums:
	            curr =max(prev1, prev2 + n)
	            prev2 = prev1
	            prev1 = curr

return prev1
```

---

## ⚠️ **常見錯誤**

- 忘記處理 `len(nums) == 1`
- 狀態定義不清楚（偷 or 不偷）
- 想成 greedy（其實是 DP）

> House Robber 的本質是：

DP[i] 取兩者的最大值即可。
