### 1. 難度 (最高五星)

- **難度：** ★★☆☆☆ (2/5)
- **說明：** 這是一道 "Medium" 難度的題目，但它也是「動態規劃」中最經典、最入門的 2D 矩陣問題。一旦理解了「狀態轉移」的思想，這題的程式碼就會變得很直觀。

### 2. 運用到的技巧

1. **動態規劃 (Dynamic Programming - DP):**
    - 這是你解法的核心。你不是在「窮舉」路徑，而是在「**累加**」可能性。
    - 你正確地識別了「大問題」的解可以由「小問題」的解推導出來。

### 3. 核心邏輯分析 (你的程式碼)

你的 DP 邏輯可以分為兩個關鍵部分：

**A. 基礎情況 (Base Case):**

Python

`if i == 0 or j == 0:
    dp[i][j] = 1`

- **分析：** 這是你 DP 的「**起點**」。
- `**i == 0**`** (第一行)：** 機器人要到達 `dp[0][j]` 上的任何一點 (例如 `dp[0][2]`)，都**只有**一種方法：`Start -> Right -> Right`。
- `**j == 0**`** (第一列)：** 機器人要到達 `dp[i][0]` 上的任何一點 (例如 `dp[2][0]`)，也**只有**一種方法：`Start -> Down -> Down`。
- 你把所有「邊界」都設為 1，這是完全正確的。

**B. 狀態轉移方程 (State Transition Equation):**

Python

`else:
    dp[i][j] = dp[i-1][j] + dp[i][j-1]`

- **分析：** 這是 DP 的**核心**。
- **意義：** 機器人只能「向下」或「向右」走。這意味著，要到達 `dp[i][j]` 這個格子，只有兩種可能：
    1. 從**上面**的格子 `dp[i-1][j]` **走下來**。
    2. 從**左邊**的格子 `dp[i][j-1]` **走過來**。
- **結論：** 因此，到達 `dp[i][j]` 的總方法數，就是「到達上面格子的方法數」**加上**「到達左邊格子的方法數」。

**C. 最終答案：**

Python

`return dp[m-1][n-1]`

- **分析：** `dp[m-1][n-1]` (即右下角的終點)，它所儲存的值，就是從 `(0, 0)` 走到 `(m-1, n-1)` 的所有路徑總和。

### 4. 時間與空間複雜度

- `m` = 矩陣的「行數」 (rows)
- `n` = 矩陣的「列數」 (columns)
2. **時間複雜度 (Time Complexity): $O(M \times N)$**
    - **分析：** 你使用了兩個巢狀 `for` 迴圈，遍歷了 `m x n` 網格中的**每一個**格子*一次*。
3. **空間複雜度 (Space Complexity): $O(M \times N)$**
    - **分析：** 你需要一個大小為 `m x n` 的 `dp` 陣列來儲存所有子問題的狀態。

---

### 程式碼逐行說明 (你的程式碼)

Python

```python
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        
        # 1. 建立一個 m x n 的 2D 陣列 (DP Table)
        #    dp[i][j] 將儲存到達 (i, j) 的路徑數
        dp = [[0] * n for _ in range(m)]
        
        # 2. 遍歷 m 行
        for i in range(m):
            # 3. 遍歷 n 列
            for j in range(n):
                
                # 4. [基礎情況 Base Case]
                #    如果是第一行 (i==0) 或第一列 (j==0)
                #    則到達那裡的 "唯一" 方法就是 1
                if i == 0 or j == 0:
                    dp[i][j] = 1
                
                # 5. [狀態轉移 State Transition]
                #    否則，到達 (i, j) 的方法數 =
                #    "到達 (i-1, j) (上面) 的方法數" +
                #    "到達 (i, j-1) (左邊) 的方法數"
                else:
                    dp[i][j] = dp[i-1][j] + dp[i][j-1]
        
        # 6. 返回右下角 (終點) 的值
        return dp[m-1][n-1]
```
