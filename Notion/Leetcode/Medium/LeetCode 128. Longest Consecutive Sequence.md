## 程式碼目標 🎯

給定一個未排序的整數陣列 nums，找出其中「最長連續序列」的長度。

例如：[100, 4, 200, 1, 3, 2] -> 最長連續序列是 [1, 2, 3, 4]，長度為 4。

**限制挑戰**：演算法的時間複雜度必須是 **$O(N)$**。

---

## 核心概念：Hash Set + 尋找起點

1. **捨棄排序**：排序需要 $O(N \log N)$，不符合題目要求。
2. **使用 Set**：將所有數字放入 `set`，讓查找「下一個數字是否存在」的時間從 $O(N)$ 降為 **$O(1)$**。
3. **只從「頭」開始數**：這是達到 $O(N)$ 的關鍵。我們遍歷每個數字，如果發現它有「前一個數字」(例如拿到 4，發現有 3)，我們就跳過它，直到拿到那個「沒有前一個數字」的起點 (例如 1)，才開始往後計數。

## 程式碼詳解 breakdown

Python

`class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        # 1. 建立 Hash Set
        #    這一步至關重要：
        #    a. 去除重複 (雖然題目沒說有重複，但去重不影響結果)
        #    b. 讓後續的 lookup (in) 操作變成 O(1)
        numsSet = set(nums)
        
        result = 0

        # 2. 遍歷集合中的每個數字
        for num in numsSet:
            
            # 3. 關鍵優化：尋找「序列起點」
            #    檢查 (num - 1) 是否在集合中。
            #    - 如果存在：代表 num 不是這個連續序列的第一個數字。
            #      (例如：拿到 4，發現 3 存在，那就跳過 4，等之後迴圈跑到 1 再處理)
            #    - 如果不存在：代表 num 是序列的「開頭」，可以開始往後數了！
            if (num - 1) in numsSet:
                continue
            
            # 4. 開始計數 (只會對「起點」執行)
            #    程式碼走到這裡，表示 num 是一個新的序列開頭
            current_num = num
            current_streak = 1

            # 5. 往後延伸
            #    不斷檢查 +1 的數字是否存在
            while (current_num + 1) in numsSet:
                current_num += 1
                current_streak += 1
            
            # 6. 更新最大長度
            result = max(result, current_streak)
        
        return result`

## 複雜度分析

假設陣列中有 $N$ 個數字。

### ⏰ 時間複雜度: $O(N)$

這通常是最讓人困惑的地方（看起來像雙層迴圈）。

- **外層迴圈**：遍歷集合中的每個元素，共 $N$ 次。
- **內層 **`**while**`** 迴圈**：雖然有迴圈，但因為 `if (num-1) in numsSet` 這個條件，只有當數字是「起點」時才會執行。
- **總結**：這意味著陣列中的**每一個數字**，在整個演算法的執行過程中，**最多只會被 **`**while**`** 迴圈訪問到 1 次**（就是當它被歸類在某個序列中被數到的時候）。
- 所以總操作次數大約是 $2N$ (外層掃一次 + 內層總共掃一次)，忽略常數後為 $O(N)$。

### 💾 空間複雜度: $O(N)$

- 我們使用了一個 `numsSet` 來儲存所有的數字，所以空間複雜度是 $O(N)$。
