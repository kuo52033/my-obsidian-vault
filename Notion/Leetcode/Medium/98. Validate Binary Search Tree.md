## 程式碼目標 🎯

這個程式碼的目標是判斷一個二元樹是否為一個「**有效的**」二元搜尋樹 (BST)。

有效 BST 的定義 (非常重要)：

對於樹中的任何一個節點 node：

1. `node` 的**整個**左子樹 (left subtree) 上的**所有**節點的值，都必須**嚴格小於** `node.val`。
2. `node` 的**整個**右子樹 (right subtree) 上的**所有**節點的值，都必須**嚴格大於** `node.val`。
3. 它的左、右子樹本身也必須是有效的 BST。

### ❌ 常見的陷阱

一個常見的錯誤是**只**檢查 `node.left.val < node.val` 和 `node.right.val > node.val`。這個檢查是**不夠的**，因為它無法抓到「跨層級」的錯誤，例如：

`      5
     / \
    3   7
       / \
      4   8  <-- 錯誤！ 4 雖然 > 3，但 4 沒有 < 5`

(我們之前討論的 `[5, 3, 7, 1, 6]` 也是一個很好的例子)

---

## 核心概念：$O(N)$ 解法

要用 $O(N)$ (即每個節點只拜訪一次) 完成檢查，我們必須在遍歷時「攜帶」足夠的資訊。

### 解法一：遞迴 (Top-Down) + 範圍限制 (最直觀)

核心思想：

我們從上往下 (Top-Down) 遞迴。我們不只檢查當前節點，我們還一路「收緊」有效範圍，把這個範圍當作「規則」傳遞給子節點。

我們使用 `min_val` (地板) 和 `max_val` (天花板) 來定義每個子樹必須存活的區間。

Python

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        # 1. 啟動輔助函式 (helper function)
        # 初始的 "有效範圍" 是 (負無限大, 正無限大)
        return self.check(root, float('-inf'), float('inf'))

    
    def check(self, node, min_val, max_val):
        # 2. 基礎情況 (Base Case):
        #    如果節點是空的，它沒有違反任何規則，視為合法
        if not node:
            return True
        
        # 3. 檢查當前節點
        #    當前節點的值，必須 "存活" 在 (min_val, max_val) 這個開區間內
        #    (注意：BST 不允許等於，所以是嚴格大於/小於)
        if not (min_val < node.val < max_val):
            return False
            
        # 4. 遞迴呼叫 (關鍵！)
        
        # 往「左」走：
        # - "地板" (min_val) 繼承
        # - "天花板" (max_val) 必須收緊為 "當前節點的值" (node.val)
        is_left_valid = self.check(node.left, min_val, node.val)
        
        # 往「右」走：
        # - "地板" (min_val) 必須收緊為 "當前節點的值" (node.val)
        # - "天花板" (max_val) 繼承
        is_right_valid = self.check(node.right, node.val, max_val)
        
        # 5. 返回結果
        #    必須「左邊」和「右邊」同時都合法
        return is_left_valid and is_right_valid
```

- 時間複雜度: $O(N)$
因為 check 函式對每一個節點都只拜訪一次。
- 空間複雜度: $O(H)$
$H$ 是樹的高度。這是遞迴呼叫堆疊 (call stack) 所需的空間。在最壞情況下 (一個傾斜的樹)，$H$ 可能是 $N$，空間變為 $O(N)$。

### 解法二：中序遍歷 (In-Order Traversal) (最巧妙)

核心思想：

利用 BST 的一個關鍵特性：

> 一個合法的 BST，它的「中序遍歷」(L-N-R: 左 -> 根 -> 右) 結果，必定是一個「嚴格遞增」的數列。

我們不需要把所有節點存到一個陣列再比較。我們可以在遍歷的過程中，只用一個變數 `prev_val` 記錄**「在中序順序中，上一個被拜訪的節點」**的值。

Python

```python
class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        
        # 1. 我們需要一個變數來追蹤 "中序遍歷" 中的前一個值
        #    我們使用 self. (或一個 list [float('-inf')]) 來讓它
        #    在遞迴呼叫中可以被修改並共享狀態
        self.prev_val = float('-inf')
        
        return self.in_order_check(root)

    def in_order_check(self, node):
        if not node:
            return True
        
        # 1. (L) 拜訪左子樹
        #    如果左子樹已經發現不合法，立刻返回 False
        if not self.in_order_check(node.left):
            return False
        
        # 2. (N) 拜訪 "根" 節點 (當前節點)
        #    這是「檢查」的關鍵時機！
        #    當我們從左子樹返回時，self.prev_val 已經是
        #    "當前 node" 的「中序前驅」的值
        
        # 檢查：當前節點的值是否 "嚴格大於" 前一個值
        # (不允許等於！ 5 <= 5 也是 False)
        if node.val <= self.prev_val:
            return False
            
        # 檢查通過，更新 prev_val 為 "當前節點" 的值
        # 這樣下一個節點 (node 的右子樹中最小的那個)
        # 就能和 "當前 node" 比較
        self.prev_val = node.val
        
        # 3. (R) 拜訪右子樹
        #    (如果右子樹不合法，它的呼叫會返回 False)
        return self.in_order_check(node.right)
```

- 時間複雜度: $O(N)$
標準的中序遍歷，每個節點拜訪一次 (L, N, R 各一次)。
- 空間複雜度: $O(H)$
同樣用於遞迴的呼叫堆疊。

## 總結

這兩種 $O(N)$ 解法都是驗證 BST 的標準答案：

4. **範圍限制法**：從上往下傳遞「邊界 (規則)」。
5. **中序遍歷法**：利用「中序遍歷 = 嚴格遞增」的特性來檢查。

兩種方法都非常高效且重要！
