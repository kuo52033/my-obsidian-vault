## 📌 題目重點

- 給定硬幣面額陣列 `coins`
- 給定總金額 `amount`
- **每種硬幣可使用無限次**
- 回傳湊出 `amount` 所需的 **最少硬幣數**
- 若無法湊出，回傳 `1`

---

## 🧠 解題核心思路（Unbounded Knapsack / DP）

這題本質是：

> 完全背包（Unbounded Knapsack）最小化問題

關鍵在於：

- 金額是容量
- 硬幣可以重複使用
- 目標是「最少數量」

---

## 🧩 DP 狀態定義

### 狀態

```plain text
dp[i] = 湊出金額 i 所需的最少硬幣數


```

### 初始化

```plain text
dp[0] = 0          # 金額 0 不需要任何硬幣
dp[其他] = ∞       # 先假設都湊不出來


```

---

## 🔁 狀態轉移方程

對於每一個硬幣 `coin`：

```plain text
dp[i] = min(dp[i], dp[i - coin] + 1)


```

意義：

- 若我最後使用一枚 `coin`
- 那前一個狀態一定是 `i - coin`
- 再加上這一枚 → `+1`

---

## 🧪 程式碼解析

```python
if amount <=0:
return0


```

- 邊界情況
- 金額為 0 → 不用硬幣

---

### 🔹 DP 陣列初始化

```python
dp = [float('inf')] * (amount +1)
dp[0] =0


```

- `dp[i]` 初始設為無限大
- 表示「目前還不知道怎麼湊」

---

### 🔹 雙迴圈（重點）

```python
for coinin coins:
for iinrange(coin, amount +1):
        dp[i] =min(dp[i], dp[i - coin] +1)


```

### 為什麼是「先 coin，再金額」？

- 這是 **完全背包** 的典型寫法
- 允許同一個 coin 被重複使用
- `dp[i]` 會基於 **本輪 coin 更新後的 dp[i-coin]**

👉 如果交換順序，會變成不同類型的背包問題

---

### 🔹 回傳答案

```python
return dp[-1]if dp[-1] !=float('inf')else -1


```

- 若 `dp[amount]` 仍是 ∞ → 無解
- 否則回傳最小硬幣數

---

## 🧠 用例子快速理解

`coins = [1, 2, 5], amount = 11`

DP 過程（概念）：

- 用 1：`[0,1,2,3,4,5,...]`
- 用 2：更新更小值
- 用 5：發現 `5+5+1` → 3 枚

結果：

```plain text
dp[11] = 3


```

---

## ⏱️ 時間複雜度

- 外層：硬幣數量 = n
- 內層：金額 = amount

✅ **Time Complexity：O(n × amount)**

---

## 💾 空間複雜度

- 只用一個一維 DP 陣列

✅ **Space Complexity：O(amount)**

---

## 🎯 解題技巧總結（面試必背）

- 「最少數量」→ dp 初始設 ∞
- `dp[0] = 0` 是所有 coin 題的基石
- **coin 在外層 → 完全背包**
- 金額在外層 → 組合型 DP（另一類題）

---

## 🔄 常見錯誤

- 忘記 `dp[0] = 0`
- 用 `1` 當初始值，轉移會很難寫
- 迴圈順序錯，導致硬幣不能重複使用
- 忘記最後判斷無解情況
