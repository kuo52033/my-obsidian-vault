## ⭐ **難度：Medium (5/10)**

## 📌 **面試常考程度：8/10**

這題是 **Binary Search 的經典變形題**，

常用來考：

- 對「已排序 + 旋轉」結構的理解
- Binary Search 邊界判斷
- 是否真的理解「不變量（invariant）」

---

## 🎯 **題意**

給一個 **原本遞增排序、但被旋轉過一次** 的陣列 `nums`

（**沒有重複元素**），請找出其中的 **最小值**。

例如：

```plain text
[4,5,6,7,0,1,2] →0
[1,2,3,4,5]     →1


```

---

## 💡 **核心觀念（最重要）**

### 🔑 關鍵性質

- 原陣列是 **遞增排序**
- 旋轉後會變成「兩段排序區間」
- **最小值一定在「未排序的那一半」**
- 若整段已排序 → 最小值就是 `nums[0]`

---

## 🧠 **你補到的關鍵判斷（完全正確）**

```python
if nums[-1] > nums[0]:
return nums[0]


```

### 意義：

> 如果最右邊 > 最左邊
> 👉 代表整段是已排序的
> 
> 👉 沒有發生旋轉
> 
> 👉 最小值一定在最左邊

這是很多人一開始會漏掉的 case。

---

## 🔄 **Binary Search 思路（Invariant）**

在搜尋過程中，維持一個不變量：

> 最小值一定落在 [left, right] 區間內

每次用 `mid` 與 `right` 比較來縮小區間。

---

## 🧩 **你目前的解法（完整、正確版本）**

```python
classSolution:
	deffindMin(self, nums: List[int]) ->int:
		if nums[len(nums)-1] > nums[0]:
			return nums[0]
		
		left =0
		right =len(nums)-1
		
		while left < right:
		    mid = (left + right) //2
				
				if nums[mid] > nums[right]:
				                left = mid +1
				else:
					if mid >0 and nums[mid] < nums[mid-1]:
						return nums[mid]
					else:
		        right = mid -1
		
		return nums[left]


```

---

## 🧠 **這個解法在做什麼？**

你其實是在 **找 rotation point**：

> 第一個「比前一個元素小」的位置
> 那個元素就是最小值

這個想法在邏輯上 **完全正確**。

---

## ⚠️ **設計層面的補充（不是錯）**

- `nums[mid] < nums[mid-1]` 的判斷是 **正確但可省略**
- 在這題中，只用：
```plain text
nums[mid] 與 nums[right]
```
就足以決定搜尋方向
- 標準解通常不會「提早 return」，而是讓區間自然收斂

👉 你現在的寫法是「展開版」，

👉 面試官會接受，也能清楚看出你理解旋轉結構。

---

## 🧠 **精煉心法（面試用）**

> 1️⃣ 如果整段已排序 → 回傳最左
> 2️⃣ 比較 `nums[mid]` 和 `nums[right]`
> 
> 3️⃣ `nums[mid] > nums[right]` → 最小值在右半
> 
> 4️⃣ 否則 → 最小值在左半（包含 mid）
> 
> 5️⃣ 縮到只剩一個 → 就是答案

---

## ⏱ **時間與空間複雜度**

- **時間複雜度**：`O(log n)`
- **空間複雜度**：`O(1)`

---

## 📝 **常見錯誤整理**

- 忘記處理「完全沒旋轉」的情況
- 在錯誤的分支中把 `mid` 排除掉
- 用 `right = mid - 1` 導致錯過答案
- 把這題當成一般 binary search（其實是在找 pivot）

---

## 📘 **一句話總結（Notion 摘要）**

> Find Minimum in Rotated Sorted Array 的關鍵是：
判斷哪一半是「未排序區間」，最小值一定在那一半。
