## 程式碼目標 🎯

這段程式碼的目標是判斷 `s3` 是否能由 `s1` 和 `s2` 的字元「交錯 (interleave)」組成，並且 `s1` 和 `s2` 內部的字元相對順序不能改變。

範例：

s1 = "aab"

s2 = "axy"

s3 = "aaxaby" -> True ( s1 取 "aa", s2 取 "ax", s1 取 "b", s2 取 "y" )

## 核心概念：2D 動態規劃

我們使用一個 2D 的 `dp` 陣列來解決這個問題。

### `dp[i][j]` 的定義

> dp[i][j] (布林值) = s1 的前 i 個字元 (s1[:i]) 和 s2 的前 j 個字元 (s2[:j])，能否剛好**交錯組成 s3 的前 i+j 個字元 (s3[:i+j])。

- `dp[0][0]` = `""` 和 `""` 能否組成 `""`
- `dp[1][0]` = `"a"` 和 `""` 能否組成 `"a"`
- `dp[1][1]` = `"a"` 和 `"a"` 能否組成 `"aa"`
- **最終答案** = `dp[len(s1)][len(s2)]`

## 程式碼詳解 breakdown

Python

`class Solution:
    def isInterleave(self, s1: str, s2: str, s3: str) -> bool:
        
        # 1. 預先剪枝 (Pruning)
        # 這是非常重要的優化。如果長度總和不匹配，
        # 絕對不可能組成 s3，直接返回 False。
        if len(s1) + len(s2) != len(s3):
            return False

        # 2. 建立 DP 表格
        # 大小為 (len(s1) + 1) x (len(s2) + 1)
        # +1 是為了處理 i=0 或 j=0 (空字串) 的情況
        dp = [[False] * (len(s2)+1) for _ in range(len(s1)+1)]
        
        # 3. 基礎情況 (Base Cases)
        
        # dp[0][0]：s1 的前 0 個和 s2 的前 0 個 (都是空字串)
        # 能否組成 s3 的前 0 個 (空字串)？ 可以。
        dp[0][0] = True

        # 初始化「第一列」 (j=0)
        # 這裡只用 s1 去匹配 s3
        for i in range(1, len(s1)+1):
            # 必須前一個狀態 dp[i-1][0] 成立 (s1[:i-1] 能匹配 s3[:i-1])
            # 且 s1 的新字元 s1[i-1] 必須等於 s3 的新字元 s3[i-1]
            dp[i][0] = dp[i-1][0] and s1[i-1] == s3[i-1]
        
        # 初始化「第一行」 (i=0)
        # 這裡只用 s2 去匹配 s3
        for j in range(1, len(s2)+1):
            # 必須前一個狀態 dp[0][j-1] 成立 (s2[:j-1] 能匹配 s3[:j-1])
            # 且 s2 的新字元 s2[j-1] 必須等於 s3 的新字元 s3[j-1]
            dp[0][j] = dp[0][j-1] and s2[j-1] == s3[j-1]
    
        # 4. 狀態轉移 (主體迴圈)
        # 從 (1, 1) 開始填滿剩下的表格
        for i in range(1, len(s1)+1):
            for j in range(1, len(s2)+1):
                
                # s3 的目標字元是 s3[i+j-1]
                
                # 情況一：s3 的最後這個字元是 "來自 s1"
                # 條件 1: dp[i-1][j] 必須為 True (s1[:i-1] 和 s2[:j] 已成功匹配 s3[:i+j-1])
                # 條件 2: s1 的新字元 s1[i-1] 必須等於 s3 的新字元 s3[i+j-1]
                from_s1 = dp[i-1][j] and s1[i-1] == s3[i+j-1]
                
                # 情況二：s3 的最後這個字元是 "來自 s2"
                # 條件 1: dp[i][j-1] 必須為 True (s1[:i] 和 s2[:j-1] 已成功匹配 s3[:i+j-1])
                # 條件 2: s2 的新字元 s2[j-1] 必須等於 s3 的新字元 s3[i+j-1]
                from_s2 = dp[i][j-1] and s2[j-1] == s3[i+j-1]

                # 只要上述兩種情況「任一」成立，dp[i][j] 就是 True
                dp[i][j] = from_s1 or from_s2
        
        # 5. 返回最終答案
        # (s1 的全部) 和 (s2 的全部) 能否組成 (s3 的全部)
        return dp[len(s1)][len(s2)]`

## 複雜度分析

假設 `s1` 的長度為 $M$，`s2` 的長度為 $N$。

### ⏰ 時間複雜度: $O(M \times N)$

- 我們建立了一個 $M \times N$ 大小的 DP 表格。
- 我們有三個獨立的迴圈：
    1. 初始化第一列： $O(M)$
    2. 初始化第一行： $O(N)$
    3. 主體迴圈： $O(M \times N)$
- 總時間複雜度由最大的部分決定，即 $O(M \times N)$。

### 💾 空間複雜度: $O(M \times N)$

- 我們使用了一個 `dp` 陣列來儲存 $(M+1) \times (N+1)$ 個布林值。
- 空間複雜度與 DP 表格的大小成正比，即 $O(M \times N)$。

(這題也可以像 91 題一樣，將空間優化到 $O(N)$ 或 $O(\min(M, N))$，因為 `dp[i][j]` 只依賴「上一列」和「當前列」的狀態，但 $O(M \times N)$ 的寫法已經非常清楚且標準了。)
