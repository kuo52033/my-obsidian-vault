## 程式碼目標 🎯

給定一個只包含數字的字串 `s`，我們要計算它總共有多少種解碼方法，解碼規則如下：

- 'A' -> "1"
- 'B' -> "2"
- ...
- 'Z' -> "26"

**範例：**

- `"12"` 可以解碼為 `"AB"` (1 2) 或 `"L"` (12)，所以有 **2** 種方法。
- `"226"` 可以解碼為 `"BBF"` (2 2 6), `"BZ"` (2 26), 或 `"VF"` (22 6)，所以有 **3** 種方法。

## 核心概念：動態規劃 (Dynamic Programming)

這道題有「重疊子問題」的特性：要計算 `"226"` 的方法數，你必須先知道 `"22"` 和 `"2"` 的方法數。

你的程式碼使用一個 `dp` 陣列來儲存這些子問題的答案。

### 1. `dp` 陣列的定義

這是 DP 最重要的一步。

> dp[i] = 解碼字串 s 的「前 i 個」字元 (即 s[0...i-1])，總共有多少種方法。

- `dp[0]` = 解碼空字串 `""` 的方法數。
- `dp[1]` = 解碼 `s[0]` 的方法數。
- `dp[2]` = 解碼 `s[0...1]` (例如 "22") 的方法數。
- ...
- **最終答案** = `dp[len(s)]`

## 程式碼詳解 breakdown

Python

`class Solution:
    def numDecodings(self, s: str) -> int:
        # 1. 建立 dp 陣列
        # 大小為 len(s) + 1，因為 dp[0] 要代表空字串
        dp = [0] * (len(s) + 1)
        
        # 2. 基礎情況 (Base Cases)
        
        # dp[0] = 1：解碼 "空字串" 有 1 種方法（就是 "什麼都不做"）
        # 這是 DP 的一個技巧，為了讓 i=2 時的 dp[i-2] (即 dp[0]) 計算正確
        dp[0] = 1
        
        # dp[1]：解碼第一個字元 s[0]
        # 如果 s[0] 是 '0' ("012")，無法解碼，所以 0 種方法
        # 如果 s[0] 是 '1'~'9' ("123")，可以解碼 (A~I)，所以 1 種方法
        dp[1] = 0 if s[0] == '0' else 1

        # 3. 狀態轉移迴圈
        # 我們從 i = 2 開始，一路計算到 dp[len(s)]
        for i in range(2, len(s) + 1):
            
            # --- 每次迴圈，我們有兩種方式可以 "走到" 第 i 個位置 ---
            
            # v 是當前看的最後一個字元 (s[i-1])
            v = s[i-1]

            # 情況一：單獨解碼 s[i-1] (最後一個字元)
            # [ ... s[0...i-2] ] + [ s[i-1] ]
            #
            # 只要 s[i-1] (v) 不是 '0' (即 '1'~'9')，它就可以單獨解碼
            # 這樣的話，方法數就 "繼承" 自 dp[i-1]
            if v != '0':
                dp[i] = dp[i-1]
            
            # 情況二：綁定解碼 s[i-2]s[i-1] (最後兩個字元)
            # [ ... s[0...i-3] ] + [ s[i-2]s[i-1] ]
            #
            # 我們要檢查 s[i-2]s[i-1] 是否是 "10"~"26" 之間
            #
            # s[i-2] != '0'：開頭不能是 '0' (例如 "05" 是無效的)
            # int(s[i-2]+v) <= 26：數字要在 10~26 之間
            if s[i-2] != '0' and int(s[i-2] + v) <= 26:
                # 如果成立，表示這是一組有效的雙位元解碼
                # 我們把 "繼承" 自 dp[i-2] 的方法數 "加上去"
                dp[i] += dp[i-2]
            
            # 如果兩種情況都不成立 (例如 "00" 或 "30" 或 "0")
            # dp[i] 會維持初始值 0，表示在此處中斷，無法解碼
            
        # 4. 返回最終答案
        return dp[len(s)]`

### 範例追蹤：s = "120" (n=3)

1. `dp = [0, 0, 0, 0]` (大小為 4)
2. `dp[0] = 1` -> `dp = [1, 0, 0, 0]`
3. `dp[1] = 1` (因為 `s[0] = '1'`) -> `dp = [1, 1, 0, 0]`
4. `**i = 2**`** (看 "12")**
    - `v = '2'`
    - **情況一** (`v != '0'`): `dp[2] = dp[1]` -> `dp[2] = 1`
    - **情況二** (`s[0] = '1'` 且 `int("12") <= 26`): `dp[2] += dp[0]` -> `dp[2] = 1 + 1 = 2`
    - `dp` 變為 `[1, 1, 2, 0]` ( "12" 有 2 種解法："AB" 和 "L"，正確)
5. `**i = 3**`** (看 "120")**
    - `v = '0'`
    - **情況一** (`v != '0'`): (False) `dp[3]` 保持 `0`。
    - **情況二** (`s[1] = '2'` 且 `int("20") <= 26`): (True) `dp[3] += dp[1]` -> `dp[3] = 0 + 1 = 1`
    - `dp` 變為 `[1, 1, 2, 1]`
6. 迴圈結束。
7. return dp[3] -> 1
( "120" 只有 "1" "20" -> "AT" 這一種解法，正確)

## 複雜度分析

假設 `s` 的長度為 $N$。

### ⏰ 時間複雜度: $O(N)$

- 我們建立了一個大小為 $N+1$ 的 `dp` 陣列。
- 我們的主要計算是一個 `for` 迴圈，它從 2 跑到 $N$ (總共 $N-1$ 次)。
- 在迴圈的**每一步**中，我們只執行 $O(1)$ 的常數時間操作（檢查字元、比較數字、加法）。
- 因此，總時間複雜度是 $O(N)$。

### 💾 空間複雜度: $O(N)$

- 我們使用了一個 `dp` 陣列來儲存 $N+1$ 個子問題的答案。
- 這個陣列的大小和輸入字串 `s` 的長度 $N$ 成正比。
- 因此，空間複雜度是 $O(N)$。

### 空間優化 (補充)

- 你可能已經注意到了，在計算 `dp[i]` 時，我們**只**用到了 `dp[i-1]` 和 `dp[i-2]` 這兩個值。
- 這和「費波那契數列」非常相似。
- 因此，我們可以不使用整個 `dp` 陣列，而是只用**三個變數**（例如 `a`, `b`, `c` 或 `dp_i`, `dp_i_minus_1`, `dp_i_minus_2`）來儲存前兩個狀態，從而將空間複雜度優化到 $O(1)$。
