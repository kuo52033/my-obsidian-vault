## ⭐ **難度：Medium (6/10)**

## 📌 **面試常考程度：8/10**

這題是經典 DP / Greedy 題，

常與 **53. Maximum Subarray** 一起被問，用來考「為什麼這題不能只用 Kadane」。

---

## 🎯 **題意**

給定一個整數陣列 `nums`，

找出 **乘積最大的連續子陣列**，並回傳其乘積。

---

## 💡 **核心觀念（重點）**

### ❌ 為什麼不能只記「最大乘積」？

因為 **負數 × 負數 = 正數**

「目前最小的乘積」有可能在下一步變成最大。

### ✅ 正確作法

**同時維護兩個狀態：**

- `currMax`：以目前 index 結尾的最大乘積
- `currMin`：以目前 index 結尾的最小乘積

---

## 🧠 **DP 狀態定義**

在 index `i`：

```plain text
currMax = max product subarray ending at i
currMin = min product subarray ending at i


```

每一步都只和前一步有關 → 可用 O(1) 空間。

---

## 🔄 **狀態轉移（最重要）**

對於 `nums[i]`，新的 `currMax / currMin` 只能來自三種情況：

1. 重新開始：`nums[i]`
2. 延續最大：`currMax * nums[i]`
3. 延續最小：`currMin * nums[i]`

```python
promax = currMax * nums[i]
promin = currMin * nums[i]

currMax = max(nums[i], promax, promin)
currMin = min(nums[i], promax, promin)


```

---

## 🌍 **全域答案更新**

```plain text
ans = max(ans, currMax)


```

因為最大乘積一定是某一段結尾的 `currMax`。

---

## 🧩 **完整程式碼**

```python
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        currmax = nums[0]
        currmin = nums[0]
        ans = currmax

        for i in range(1, len(nums)):
            promax = currmax * nums[i]
            promin = currmin * nums[i]

            currmax = max(nums[i], promax, promin)
            currmin = min(nums[i], promax, promin)

            ans = max(ans, currmax)

        return ans


```

---

## ⚠️ **常見陷阱**

- 只記 `currMax`（錯）
- 忽略負數翻轉
- 看到 0 沒有重置（本解法自動處理）
- 用 Kadane（只適用加法，不適用乘法）

---

## 🧪 **關鍵測資理解**

| 輸入 | 輸出 | 說明 |
| --- | --- | --- |
| `[2,3,-2,4]` | `6` | `[2,3]` |
| `[-2,0,-1]` | `0` | 單獨 0 |
| `[-2,3,-4]` | `24` | 負負得正 |

---

## ⏱ **時間與空間複雜度**

- **時間複雜度**：`O(n)`
- **空間複雜度**：`O(1)`

---

## 📘 **一句話總結（Notion 摘要）**

> 乘積題必須同時追蹤最大與最小值，
因為負數會讓最小值在下一步翻轉成最大值。
