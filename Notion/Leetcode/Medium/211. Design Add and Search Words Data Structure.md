## 📌 題目重點

設計一個資料結構，支援以下操作：

- `addWord(word)`
    - 將單字加入資料結構
- `search(word)`
    - 查詢單字是否存在
    - `word` **可能包含 **`**.**`
        - `.` 代表「任意一個字母」

---

## 🧠 解題核心思路（Trie + DFS）

### 為什麼用 Trie？

- 字串逐字比對 → Trie 非常適合
- 查詢效率高（每個字母 O(1)）
- 結構天然支援「前綴」

---

### `.` 的關鍵困難

- 一般 Trie 查詢是 **唯一路徑**
- `.` 代表：
    - 這一層 **所有 child 都可能**
    - 必須 **全部嘗試（DFS / backtracking）**

---

## 🧩 使用到的資料結構

- **Trie（Prefix Tree）**
    - `children: dict[char → TrieNode]`
    - `isEnd: bool`
- **DFS（遞迴）**
    - 用來處理 `.` 的分岔搜尋

---

## 🧱 TrieNode 設計

```python
classTrieNode:
def__init__(self):
self.children = {}
self.isEnd =False


```

- `children`
    - key：字母
    - value：下一個 TrieNode
- `isEnd`
    - 標記是否為某個單字結尾

---

## 🧪 程式碼解析

### 🔹 addWord(word)

```python
defaddWord(self, word: str) ->None:
    curr =self.root

for win word:
if wnotin curr.children:
            curr.children[w] = TrieNode()
        curr = curr.children[w]

    curr.isEnd =True


```

### 邏輯說明

- 從 root 開始逐字插入
- 不存在就建立新節點
- 最後一個字母設 `isEnd = True`

### 時間複雜度

- O(L)，L = word 長度

## 🔍 search(word) 的核心（DFS）

### DFS 函式定義

```python
defdfs(node, i):
if i ==len(word):
return node.isEnd


```

- `node`：目前所在的 TrieNode
- `i`：目前匹配到 `word[i]`
- 若 pattern 用完：
    - 必須確認目前節點是某個單字結尾

---

### 情況一：一般字母

```python
w = word[i]

if w !='.':
if wnotin node.children:
returnFalse
else:
return dfs(node.children[w], i+1)


```

- 只能走 **唯一一條路**
- 找不到字母 → 直接失敗

---

### 情況二：遇到 `.`

```python

for cin node.children.values():
```

### 為什麼要 DFS？

- `.` 可能對應多個字母
- 只要 **任一條路成功即可**
- 全部失敗才回傳 False

👉 這就是 **Backtracking**

## ⏱️ 時間複雜度

- `addWord`：**O(L)**
- `search`：
    - 沒有 `.`：O(L)
    - 有 `.`：
        - 最壞情況會嘗試多條分支
        - 約 **O(26^L)**（理論上限）

✅ 實務上通常遠小於最壞情況

---

## 💾 空間複雜度

- Trie 儲存所有字元：O(total characters)
- DFS recursion stack：O(L)

---

## 🎯 解題技巧總結（面試必背）

- **字串 + wildcard → Trie + DFS**
- `.` = branching（一定要想到 backtracking）
- base case 一定要檢查 `isEnd`
- DFS 回傳 True 就可以立刻剪枝

---

## 🔄 常見錯誤

- 忘記在 `i == len(word)` 時檢查 `isEnd`
- `.` 只試第一個 child（少了 for loop）
- 誤用 BFS（會更複雜）
