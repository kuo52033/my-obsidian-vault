### 1. 難度 (最高五星)

- **難度：** ★★★☆☆ (3/5)
- **說明：** 這是一道 "Medium" 難度的題目，但它是 Greedy / BFS 類型中非常經典的難題。它的 $O(N)$ 解法在面試中很常出現，因為它非常考驗你是否能跳出「一步一步跳」的直觀思維。

### 2. 核心技巧：貪心 (Greedy) / 廣度優先搜索 (BFS)

這題要我們求「**最小跳躍次數**」，這是 BFS 的一個經典應用場景（尋找最少步驟）。

你可以把這題想像成：

- **Level 0 (第 0 跳):** 只有 `index = 0`。
- **Level 1 (第 1 跳):** `index = 0` 能跳到的所有格子。
- **Level 2 (第 2 跳):** Level 1 的所有格子能跳到的所有格子。
- **...**

我們的目標是，找到「終點」(`n-1`) 落在第幾層 (Level)。

你的 $O(N)$ 解法，就是一個**空間複雜度為 $O(1)$ 的 BFS 演算法**。你沒有使用 `queue`，而是用了三個變數來追蹤「層」的邊界：

- `jump`：**你目前在第幾層 (Level)**。這就是答案。
- `curr_end`：**你「這一層」(Level **`**jump**`**) 的最遠邊界 (索引)**。
- `farthest`：**你「下一層」(Level **`**jump + 1**`**) 的最遠邊界 (索引)**。

### 3. 演算法邏輯拆解 (你寫法的白話文)

我們用一個 `for` 迴圈來「走」過整個陣列，模擬這個 BFS 的過程：

1. `**farthest = max(farthest, nums[i] + i)**`
    - **意義：** 這是演算法的核心。當 `i` 在「**這一層**」的範圍內行走時，我們*同時*在「**探查**」：從 `i` 這一格跳出去，最遠能到哪裡 (`nums[i] + i`)？
    - `farthest` 會不斷更新，永遠保持「**下一層**」我們所能到達的最遠潛力。
2. `**if i == curr_end:**`
    - **意義：** 這是一個「**觸發器**」。它代表「`**i**`** 已經走到了『這一層』的邊界了**」。
    - **舉例：** `[2, 3, 1, 1, 4]`
        - 一開始 `curr_end = 0`。當 `i = 0` 時，觸發此條件。
        - 這代表我們*必須*用掉一次跳躍 (`jump += 1`) 才能繼續前進。
    - **觸發後的動作：**
        - `**jump += 1**`: 我們被迫用掉了 1 跳 (進入下一層)。
        - `**curr_end = farthest**`: 我們「**更新**」『這一層』的邊界。既然我們跳了，我們的新邊界就是我們剛剛探查到的 `farthest` (下一層的最遠邊界)。
3. `**for i in range(len(nums) - 1):**`
    - **意義：** 為什麼只到 `len(nums) - 2`？
    - 因為我們在 `i` 點時，是在計算 `i` *能跳到*的未來。當 `i` 走到 `n-2` 時，`farthest` 已經更新了 `(n-2) + nums[n-2]` 的潛力。
    - 我們*不需要*在 `n-1` (終點) 上再做計算，因為我們的目標只是「**到達**」終點，而不是「*從*」終點跳出去。

### 4. 時間與空間複雜度

4. **時間複雜度 (Time Complexity): $O(N)$**
    - 我們只用了一個 `for` 迴圈遍歷了陣列一次。
5. **空間複雜度 (Space Complexity): $O(1)$**
    - 我們只使用了 `farthest`, `curr_end`, `jump` 三個常數空間的變數。

### 程式碼逐行說明 (你的程式碼)

Python

```python
class Solution:
    def jump(self, nums: List[int]) -> int:
        
        # "下一層" (next level) 潛在能到達的最遠索引
        farthest = 0
        
        # "這一層" (current level) 的最遠邊界
        curr_end = 0
        
        # 目前為止的「跳躍次數」(也就是 BFS 的 "層數")
        jump = 0

        # 遍歷陣列，但 "不需要" 訪問最後一個元素 (n-1)
        # 因為我們在 n-2 時就能決定是否能到 n-1
        for i in range(len(nums) -1):
            
            # [核心]：
            # 當 i 在「這一層」行走時，
            # 不斷 "探查" 並 "更新"「下一層」的最遠邊界
            farthest = max(farthest, nums[i] + i)
            
            # [關鍵]：
            # 當 i 走到了「這一層」的最遠邊界時...
            if i == curr_end:
                
                # 1. 我們被迫 "跳" 了 (層數 +1)
                jump += 1
                
                # 2. 更新「這一層」的邊界為我們剛剛探查到的 "farthest"
                curr_end = farthest
        
        # 迴圈結束後，jump 就會是到達終點的最小層數
        return jump
```
