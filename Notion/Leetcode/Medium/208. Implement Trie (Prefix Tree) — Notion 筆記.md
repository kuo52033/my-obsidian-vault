Trie 是字串題的**核心資料結構**，

後續大量題目都建立在 Trie 之上（211 / 212 / 336 / 648）。

---

## 🎯 **題意**

實作一個 Trie（Prefix Tree），支援三個操作：

- `insert(word)`：插入單字
- `search(word)`：查詢完整單字是否存在
- `startsWith(prefix)`：查詢是否存在某個 prefix

---

## 💡 **Trie 是什麼？（核心概念）**

> Trie 是一棵用「字元」當邊的樹，
每個節點代表一個 prefix。

- 從 root 走到某個節點
→ 就代表一個字串 prefix
- 只有在單字結尾，才標記為完整單字

---

## 🧠 **資料結構設計**

### TrieNode

```python
classTrieNode:
def__init__(self):
self.children = {}# char -> TrieNode
self.isEnd =False# 是否為完整單字結尾


```

### 為什麼 TrieNode 不存字元？

- 字元存在於 **parent.children 的 key**
- TrieNode 代表的是「prefix 的終點」
- 不重複存資料，結構更乾淨

---

## 🔄 **三個核心操作的想法**

### 1️⃣ insert(word)

- 從 root 開始
- 對每個字元：
    - 若 child 不存在 → 建立
    - 移動到 child
- 最後一個字元 → `isEnd = True`

---

### 2️⃣ search(word)

- 每個字元都要走得到
- **最後一個節點必須 **`**isEnd == True**`

---

### 3️⃣ startsWith(prefix)

- 每個字元都走得到即可
- **不需要檢查 **`**isEnd**`

---

## 🧩 **完整實作（dict 版本，標準解）**

```python
classTrieNode:
def__init__(self):
self.children = {}
self.isEnd =False

classTrie:

def__init__(self):
self.root = TrieNode()

definsert(self, word: str) ->None:
        curr =self.root

for win word:
if wnotin curr.children:
                curr.children[w] = TrieNode()
            curr = curr.children[w]

        curr.isEnd =True

defsearch(self, word: str) ->bool:
        curr =self.root

for win word:
if wnotin curr.children:
returnFalse
            curr = curr.children[w]

return curr.isEnd

defstartsWith(self, prefix: str) ->bool:
        curr =self.root

for win prefix:
if wnotin curr.children:
returnFalse
            curr = curr.children[w]

returnTrue


```

## ⏱ **時間與空間複雜度**

- **insert**：`O(L)`
- **search**：`O(L)`
- **startsWith**：`O(L)`
（L = 字串長度）
- **空間複雜度**：`O(total characters)`
→ 只為實際出現的字元建立節點

---

## ⚠️ **常見錯誤**

- insert `"apple"` 後，以為 `"app"` 也存在
- search 忘記檢查 `isEnd`
- startsWith 錯誤地檢查 `isEnd`
- 在 TrieNode 重複存字元

## 📘 **一句話總結（Notion 摘要）**

> Trie 是用來處理 prefix 的資料結構，
節點代表 prefix，字元存在於邊上，
insert / search / startsWith 都只是一路走字元。
