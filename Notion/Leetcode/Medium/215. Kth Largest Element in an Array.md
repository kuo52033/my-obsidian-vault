## 📌 題目說明

- 給定整數陣列 `nums` 與整數 `k`
- 找出 **第 k 大的元素**
- **不是第 k 個不同的元素**（重複要算）

---

## 🧠 解題核心思路（Min Heap）

### 為什麼用 Heap？

Python 的 `heapq` 是 **Min Heap**

👉 最小值永遠在 `heap[0]`

### 關鍵想法

如果我們：

1. 把整個陣列變成 **Min Heap**
2. 移除最小的 `n - k` 個元素
3. 剩下的 heap 最小值，就是 **第 k 大**

---

## 🧩 使用到的資料結構

- **Min Heap**
    - Python 實作：`heapq`
    - 支援：
        - `heapify`
        - `heappop`
        - `heap[0]` 取最小值

---

## 🧪 程式碼解析

```python
import heapq

classSolution:
deffindKthLargest(self, nums: List[int], k:int) ->int:


```

---

### 🔹 Step 1：建立 Min Heap

```python
        heapq.heapify(nums)


```

- 時間複雜度：**O(n)**
- 直接在原陣列上轉成 heap（in-place）

---

### 🔹 Step 2：移除最小的 n - k 個元素

```python
for iinrange(len(nums) - k):
            heapq.heappop(nums)


```

- 每次 `heappop` 都會移除目前最小值
- 移除完後：
    - heap 裡只剩下 **k 個最大元素**

---

### 🔹 Step 3：回傳答案

```python
return nums[0]


```

- Min Heap 的 `nums[0]`
- = 剩下元素中的最小值
- = **第 k 大元素**

---

## ⏱️ 時間複雜度

- `heapify`: O(n)
- `heappop` × (n − k): O((n − k) log n)

✅ **Time Complexity：O(n + (n − k) log n)**

➡️ 最壞情況接近 **O(n log n)**

---

## 💾 空間複雜度

- 使用原陣列建 heap（in-place）

✅ **Space Complexity：O(1)**（不算 input）

---

## 🎯 解題技巧總結

- Python 只有 **Min Heap**
- 找第 k 大：
    - 方法一：移除 `n - k` 個最小值
    - 方法二（進階）：維持大小為 k 的 Min Heap
- 如果 k 很小（例如 k ≪ n），第二種方法更快

---

## 🔄 進階 / 面試延伸

### 更優解（Heap size = k）

```python
heap = []
for nin nums:
    heapq.heappush(heap, n)
iflen(heap) > k:
        heapq.heappop(heap)
return heap[0]


```

- 時間：**O(n log k)**
- 空間：**O(k)**
- 面試常被加問：「怎麼優化？」
