## 題目精髓

給定一個樓梯,每階都有一個花費。你可以選擇爬 1 階或 2 階。一旦你踏上某一階,就需要支付該階的花費。你可以從第 0 階或第 1 階開始。目標是找到爬到「樓頂」(即超過最後一階)的最小總花費。

## 關鍵思想:動態規劃 (DP)

- **定義 DP 狀態**: 讓 `dp[i]` 代表「**到達**第 `i` 階」所需的最小花費。
- **Base Cases**: `dp[0] = 0` 和 `dp[1] = 0`。因為你可以免費從第 0 階或第 1 階開始,所以「到達」這兩階的成本是 0。
- **狀態轉移方程**: 要想「到達」第 `i` 階,你只能從第 `i-1` 階或第 `i-2` 階爬上來。
    - 從 `i-1` 爬 1 階上來:總花費 = (到達 `i-1` 的成本) + (離開 `i-1` 的成本) = `dp[i-1] + cost[i-1]`
    - 從 `i-2` 爬 2 階上來:總花費 = (到達 `i-2` 的成本) + (離開 `i-2` 的成本) = `dp[i-2] + cost[i-2]`
    - 因此,`dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])`
- **最終答案**: 我們的目標是到達「樓頂」,也就是索引 `n` (`n = len(cost)`)。所以我們需要計算 `dp[n]`。

## 演算法模式 (O(1) 空間優化 - 你的解法)

- 你的解法非常棒!你敏銳地觀察到,計算 `dp[i]` **只需要** `dp[i-1]` 和 `dp[i-2]` 這兩個值。
- 這意味著我們根本不需要一個 O(N) 的 `dp` 陣列,只需要兩個變數(你用的 `a` 和 `b`)來「滾動」儲存前兩個狀態即可。
- `a` 扮演 `dp[i-2]` 的角色。
- `b` 扮演 `dp[i-1]` 的角色。
- `temp = min(a + cost[i-2], b + cost[i-1])` 這行程式碼就是在計算 `dp[i]`。
- `a = b`, `b = temp` 則是將狀態向前滾動,為下一輪迴圈做準備。
- 迴圈結束時,`b` 儲存的就是 `dp[n]`(即 `len(cost)`)的值,也就是最終答案。

## 複雜度分析

- **時間複雜度: O(N)**
    - N 是 `cost` 列表的長度。我們只需要一個迴圈從 `2` 遍歷到 `n`。
- **空間複雜度: O(1)**
    - 你只使用了 `a`, `b`, `temp` 等固定數量的變數,這是最優的空間解法。

## Takeaway

**這題是繼「爬樓梯」(70) 之後,另一個「斐波那契數列」風格的 DP 經典題。當你發現一個 DP 問題的狀態轉移只依賴於前 $k$ 個固定數量的狀態時(如此題 $k=2$),就要立刻想到「空間優化」,使用 O(k)(此題為 O(1))的變數來取代 O(N) 的 DP 陣列。**
