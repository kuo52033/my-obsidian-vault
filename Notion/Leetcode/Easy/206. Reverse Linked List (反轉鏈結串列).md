- **題號**: 206
- **核心標籤**: `鏈結串列 (Linked List)`, `迭代 (Iteration)`, `遞迴 (Recursion)`
- **題目精髓**: 這是鏈結串列最核心的操作之一，旨在測試對指標 (pointer) 操作的熟練度。能否清晰、無誤地寫出迭代和遞迴兩種解法，是衡量鏈結串列基礎是否牢固的關鍵指標。
- **解法一：迭代法 (Iteration) - O(1) 空間**
    - **思路 (三指針交換舞步)**: 這是**最優且最常用**的解法。透過 `prev`, `curr`, `temp` 三個指針的協調，逐個節點地反轉 `next` 指標的方向。
    - **演算法模式**:
        1. 初始化 `prev = None` (這將是新鏈結串列的尾部)，`curr = head`。
        2. 進入 `while curr:` 迴圈。
        3. **在迴圈中，執行固定的四步舞步**:
            - **a. 儲存未來**: `temp = curr.next` (在斷開連接前，先把下一個節點存起來)。
            - **b. 反轉指標**: `curr.next = prev` (核心操作，將當前節點指向前一個節點)。
            - **c. **`**prev**`** 前進**: `prev = curr` (為下一輪做準備，`prev` 移動到當前位置)。
            - **d. **`**curr**`** 前進**: `curr = temp` (`curr` 移動到下一個待處理的節點)。
        4. 迴圈結束後，`curr` 為 `None`，而 `prev` 正好指向反轉後的新頭節點，返回 `prev`。
- **解法二：遞迴法 (Recursion) - O(N) 空間**
    - **思路 (信任的連鎖反應)**: 將問題分解為「反轉除了頭節點之外的其餘部分」和「將頭節點接到反轉後鏈結串列的尾部」。
    - **演算法模式**:
        1. **Base Case (遞迴出口)**: 如果 `head` 為空或只有一個節點 (`head is None or head.next is None`)，無需反轉，直接 `return head`。
        2. **遞迴深入**: `new_head = self.reverseList(head.next)`。**信任**這個呼叫會返回**已經反轉好的子鏈結串列的頭節點**。
        3. **處理當前層**:
            - `head.next.next = head`：`head.next` 此時是子鏈結串列的尾部，讓它的 `next` 指回 `head`，完成拼接。
            - `head.next = None`：**(至關重要)** 將 `head` 變為新的尾部，並切斷其原始的 `next` 連接，**防止產生循環**。
        4. **返回結果**: `return new_head`。將從最深層得到的 `new_head` 一路傳遞回頂層。
- **複雜度分析**:
    - **時間複雜度: O(N)**
        - 兩種解法都需要遍歷所有 N 個節點一次。
    - **空間複雜度**:
        - **迭代法**: **O(1)**。只使用了固定數量的額外指針。
        - **遞迴法**: **O(N)**。空間消耗在遞迴呼叫堆疊 (Call Stack)上，最深可達 N 層。
- **Takeaway**:
> 反轉鏈結串列是演算法面試中的 "Hello, World!"。
> - **迭代法**因其 O(1) 的空間複雜度，是**面試和實際應用中的首選**。務必將「三指針舞步」練成肌肉記憶。
> - **遞迴法**雖然空間效率較低，卻是**理解遞迴如何操作指標結構的絕佳練習**。其「先遞迴到底，再逐層處理」的後序遍歷思想在很多樹和圖的問題中都會出現。
