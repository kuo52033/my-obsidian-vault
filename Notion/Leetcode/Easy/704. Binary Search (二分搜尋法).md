## 題目精髓

在一個已排序的陣列中，以 $O(\log N)$ 的時間複雜度快速定位一個元素。

## 關鍵思想：分治法 (Divide and Conquer)

- 這是「分治法」最純粹的體現。二分搜尋的核心在於**「不斷縮小搜尋範圍」**。
- 我們不從頭到尾一個個檢查（O(N)），而是在每一步都**直接檢查中間的元素**。
- 透過一次比較，我們就可以**排除掉一半**不可能存在 `target` 的元素，從而使問題規模減半。

## 演算法模式 (標準的「閉區間」寫法)

這是最推薦、最不容易出錯的模板。

1. **初始化指針**: `left = 0`，`right = len(nums) - 1`。這定義了我們的搜尋範圍是一個**閉區間 **`**[left, right]**`。
2. **迴圈條件**: `while left &lt;= right:`
    - **關鍵點**: 必須包含等於 (`&lt;=`)。這確保了當 `left == right`（搜尋範圍只剩下最後一個元素）時，迴圈會**再執行一次**，對該元素進行最後的檢查。
3. **計算中間點**: `mid = (left + right) // 2` (在 Python 中，`//` 是整數除法)。
4. **比較與縮小範圍**:
    - **情況 1 (找到)**: `if nums[mid] == target:`
        - `return mid` (在迴圈內部直接返回，最簡潔)。
    - **情況 2 (目標在右側)**: `elif nums[mid] &lt; target:`
        - `mid` 已經被檢查過了，所以新的搜尋範圍是 `[mid + 1, right]`。
        - `left = mid + 1`
    - **情況 3 (目標在左側)**: `else:` (即 `nums[mid] &gt; target`)
        - `mid` 已經被檢查過了，所以新的搜尋範圍是 `[left, mid - 1]`。
        - `right = mid - 1`
5. **迴圈結束 (未找到)**: 如果迴圈自然結束（代表 `left &gt; right`，搜尋範圍為空），說明 `target` 不存在。
    - `return -1`

## 複雜度分析

- **時間複雜度: $O(\log N)$**
    - N 是陣列的長度。由於我們在每一步都將搜尋範圍縮小一半 (N -> N/2 -> N/4 -> ...)，所以最多需要 $O(\log_2 N)$ 次比較。
- **空間複雜度: $O(1)$**
    - 只使用了 `left`, `right`, `mid` 等固定數量的變數。

## Takeaway

- **二分搜尋法是所有高效演算法的基石。**
- **它的唯一前提是「資料必須是已排序的」。**
**務必將 **`**while left &lt;= right:**`** 這個「閉區間」標準模板練成肌肉記憶，它可以清晰地處理所有邊界情況，是解決二分搜尋問題最穩健的寫法。**
