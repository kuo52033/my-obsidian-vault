- **題號**: 100
- **核心標籤**: `樹 (Tree)`, `遞迴 (Recursion)`, `深度優先搜尋 (DFS)`
- **題目精髓**: 這是學習樹的遞迴遍歷的入門經典。它建立了一個解決大多數樹相關問題的通用思考框架：**將大問題分解成結構完全相同的子問題**。
- **關鍵思想：遞迴分解 (Recursive Decomposition)**
    - 判斷兩棵樹 `p` 和 `q` 是否相同，可以被完美地分解為三個子條件的**邏輯 AND** 組合：
        1. `p` 和 `q` 的**根節點值**必須相同。
        2. `p` 的**左子樹**必須和 `q` 的**左子樹**相同。
        3. `p` 的**右子樹**必須和 `q` 的**右子樹**相同。
    - 這個邏輯可以不斷地應用到更小的子樹上，直到遇到葉子節點的子節點 (`None`)。
- **演算法模式 (遞迴解題訣竅：三部曲)**
    - **1. 定義遞迴函式的「契約」**：
        - 明確 `isSameTree(p, q)` 的功能：接收兩個節點，回傳以它們為根的子樹是否相同。
    - **2. 處理基礎情況 (Base Cases) - 遞迴的出口**：
        - **成功出口**: `if p is None and q is None: return True` (兩邊都同時結束，代表此路徑匹配)。
        - **失敗出口**: `if p is None or q is None or p.val != q.val: return False` (結構不對稱，或當前節點值不匹配)。
    - **3. 處理遞迴情況 (Recursive Step) - 信任的力量**：
        - 假設 `isSameTree` 已經可以完美處理規模更小的問題。
        - 你只需要呼叫它來解決子問題，並組合結果即可：
`return self.isSameTree(p.left, q.left) and self.isSameTree(p.right, q.right)`
        - **訣竅**：不要用人腦去追蹤每一層的呼叫細節，那會讓你混亂。專注於寫好「當前層」的邏輯，並相信遞迴呼叫會正確地完成子任務。
- **複雜度分析**:
    - **時間複雜度: O(N)**
        - N 是兩棵樹中節點較少的那個的節點數。我們最多需要訪問每個節點一次。
    - **空間複雜度: O(H)**
        - 空間主要消耗在**遞迴呼叫堆疊 (Call Stack)** 上。堆疊的最大深度等於樹的高度 H。
        - **平均情況 (平衡樹)**：H ≈ log(N)，空間複雜度為 O(log N)。
        - **最壞情況 ( skewed tree)**：H = N，空間複雜度退化為 O(N)。
- **takeaway**:
> 這題是樹遞迴的「母題」。其「定義契約 -> 處理 Base Case -> 遞迴呼叫」的模式是解決幾乎所有樹問題的基礎。掌握這個模式，你就能解決大部分的樹遍歷、比較、修改等問題。
