- **題號**: 543
- **核心標籤**: `樹 (Tree)`, `遞迴 (Recursion)`, `深度優先搜尋 (DFS)`
- **題目精髓**: 這是一道經典的樹遞迴問題，它巧妙地利用「計算高度」的過程來同時解決「計算直徑」的問題。這題的**核心陷阱**在於，最長路徑（直徑）**不一定**會穿過根節點 `root`。
- **關鍵思想：直徑的三種可能性**
    - 對於任何一個節點 `node`，整棵樹的最大直徑只可能是以下三者中的最大值：
        1. **穿過 **`**node**` 的最長路徑（長度為 `height(node.left) + height(node.right)`）。
        2. **完全在 **`**node**`** 左子樹**內部的最大直徑。
        3. **完全在 **`**node**`** 右子樹**內部的最大直徑。
    - 一個低效的 O(N²) 解法是在每個節點都遞迴計算這三者。
- **演算法模式 (最優解：後序遍歷 + 全局變數)**
    - **思路**: 這是**最優的 O(N) 解法**。我們設計一個輔助函式（例如 `getHeight`），讓它在一次遞迴中**同時完成兩項任務**：
        1. **主要任務 (回傳值)**：回傳以 `node` 為根的子樹的「高度」（從 `node` 往下最長的單一路徑長度）。
        2. **次要任務 (副作用)**：在計算高度的過程中，順便計算出**穿過 **`**node**`** 的直徑** (`left_height + right_height`)，並用這個值去更新一個全局（或成員）變數 `self.max_diameter`。
    - **演算法步驟**:
        1. 在主函式中，初始化 `self.max_diameter = 0`。
        2. 呼叫輔助函式 `getHeight(root)`。
        3. `getHeight(node)` 函式的邏輯（這是一個**後序遍歷**）：
            - **Base Case**: `if node is None: return 0`。
            - **遞迴深入**: `left_height = getHeight(node.left)`，`right_height = getHeight(node.right)`。
            - **執行次要任務**: `self.max_diameter = max(self.max_diameter, left_height + right_height)`。
            - **執行主要任務**: `return 1 + max(left_height, right_height)`。
        4. 主函式最終返回 `self.max_diameter`。
- **複雜度分析**:
    - **時間複雜度: O(N)**
        - N 是樹的節點總數。我們的輔助函式只會訪問每個節點一次。
    - **空間複雜度: O(H)**
        - H 是樹的高度。空間消耗在遞迴呼叫堆疊上。平均情況（平衡樹）為 O(log N)，最壞情況（鏈結串列）為 O(N)。
- **Takeaway**:
> 這是樹的遞迴問題中一個非常強大的模式：「後序遍歷 + 全局變數」。
> 當一個問題的解（如「直徑」）依賴於子樹的計算結果（如「高度」），並且這個解可能存在於樹的任何一個局部時，這個模式是最佳解法。
> 
> **這個「遞迴函式回傳局部解，並在過程中更新全局解」的思路，是解決許多樹的 DP 問題（如 Max Path Sum）的基礎。**
