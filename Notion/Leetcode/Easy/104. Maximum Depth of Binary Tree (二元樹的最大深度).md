• **題號**: 104
• **核心標籤**: `樹 (Tree)`, `遞迴 (Recursion)`, `深度優先搜尋 (DFS)`, `廣度優先搜尋 (BFS)`
• **題目精髓**: 這是樹相關演算法的 "Hello, World!"。它旨在測試對樹的遍歷和遞迴的基本理解，是所有「Bottom-up (由下而上)」遞迴思想的基礎，也是 BFS 層序遍歷的基礎應用。
• **關鍵思想：分治法 (Divide and Conquer)**
    ◦ 一個樹的最大深度，可以被分解為 `1` (當前節點) 加上「其子樹的最大深度」。這是一個天然的遞迴結構。
• **演算法模式一：遞迴法 (DFS / Bottom-up) - (標準解法)**
    ◦ **思路**: 這就是「後序遍歷」思想的遞迴。函式的任務是計算並「回報」以當前節點為根的子樹的最大深度。
    ◦ **步驟**:
        1. **Base Case (遞迴出口)**: `if root is None: return 0`。空樹（`None` 節點）的深度為 0。
        2. **遞迴深入 (信任的力量)**:
            • `left_depth = self.maxDepth(root.left)` (獲取左子樹的深度)
            • `right_depth = self.maxDepth(root.right)` (獲取右子樹的深度)
        3. **處理當前層 (組合結果)**:
            • `return 1 + max(left_depth, right_depth)`
            • 將子問題的答案（`left_depth`, `right_depth`）組合起來，加上當前節點的 `1`，然後「回報」給上一層。
• **演算法模式二：迭代法 (BFS / 層序遍歷)**
    ◦ **思路**: 也可以用「廣度優先搜尋 (BFS)」來解。樹的最大深度，就等於 BFS 遍歷的**總層數**。
    ◦ **步驟**:
        1. 處理 `root is None` 的初始情況，返回 0。
        2. 初始化 `depth = 0` 和一個佇列 `queue` (例如 `collections.deque`)，將 `root` 加入佇列。
        3. 當 `queue` 不為空時，代表新的一層開始了：
            • `depth += 1` (層數加一)。
            • `level_size = len(queue)`。
            • 用一個 `for` 迴圈，將**這一層的所有節點** (`level_size` 個) 全部 `pop` 出來。
            • 對於 `pop` 出的每個節點，將其**下一層的非空子節點** (`.left` 和 `.right`) 加入 `queue` 中。
        4. 當外層 `while` 迴圈結束時（`queue` 為空），`depth` 就是總層數。
• **複雜度分析**:
    ◦ **時間複雜度: O(N)** (N 是節點總數)
        ▪ **解法一 (DFS)**: 每個節點被訪問一次。
        ▪ **解法二 (BFS)**: 每個節點被 `push` 和 `pop` 一次。
    ◦ **空間複雜度**:
        ▪ **解法一 (DFS)**: $O(H)$，H 是樹的高度。空間消耗在遞迴呼叫堆疊上。最壞情況（鏈結串列）為 $O(N)$，平均情況（平衡樹）為 $O(\log N)$。
        ▪ **解法二 (BFS)**: $O(W)$，W 是樹的最大寬度。在最壞情況下（滿二元樹），W $\approx$ N/2，空間為 $O(N)$。
• **Takeaway**:這題是所有樹遞迴問題的基礎模板。`return 1 + max(left, right)` 這種「先從子節點獲取答案，再組合出當前節點答案」的 Bottom-up 模式，是解決無數樹相關問題（如平衡樹、直徑、路徑和）的基石。
