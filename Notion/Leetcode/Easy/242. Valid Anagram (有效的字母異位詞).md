- **題號**: 242
- **核心標籤**: `雜湊表 (Hash Table)`, `字串 (String)`, `排序 (Sorting)`
- **題目精髓**: 這是「頻率計數 (Frequency Counting)」模式的代表性題目。旨在測試是否能用高效的資料結構來統計和比較字元出現的次數。
- **關鍵思想：字母異位詞的定義**
    - 兩個字串互為字母異位詞，若且唯若它們由**相同的字元**組成，且每個字元的**出現次數都完全相同**。
- **解法一：使用雜湊表 (Hash Table) / 頻率陣列**
    - **思路**: 這是你使用的解法，也是時間效率上的**最優解**。先統計一個字串的字元頻率，然後用另一個字串去「消耗」這個頻率計數。
    - **演算法模式**:
        1. **提前退出**: 如果兩個字串長度不同，直接返回 `False`。
        2. **建立頻率表**: 遍歷第一個字串 `s`，使用一個雜湊表 (或一個長度為 26 的陣列) 來記錄每個字元出現的次數。
        3. **驗證與消耗**: 遍歷第二個字串 `t`。對於 `t` 中的每個字元，執行以下操作：
            - 檢查該字元是否存在於頻率表中，且計數大於 0。
            - 如果不存在或計數已為 0，說明 `t` 含有多餘或 `s` 中沒有的字元，返回 `False`。
            - 如果存在，將其在頻率表中的計數減一。
        4. 如果遍歷 `t` 的迴圈順利完成，返回 `True`。
- **解法二：排序 (Sorting)**
    - **思路**: 一個非常直觀且巧妙的思路。如果兩個字串是字母異位詞，那麼將它們各自的字元排序後，得到的結果字串必然是完全相同的。
    - **一行程式碼**: `return sorted(s) == sorted(t)`
    - **優點**: 程式碼極度簡潔。
    - **缺點**: 時間複雜度較高。
- **Pythonic 解法：**`**collections.Counter**`
    - **思路**: `collections.Counter` 是一個專門用來計數的類別。它可以直接將一個序列轉換為頻率字典。
    - **一行程式碼**: `return collections.Counter(s) == collections.Counter(t)`
    - **說明**: 直接比較兩個 `Counter` 物件，Python 會自動檢查它們是否擁有相同的鍵和對應的值。
- **複雜度分析**:
    - **雜湊表解法**:
        - **時間複雜度: O(N)**，N 是字串的長度。我們只需要遍歷字串常數次。
        - **空間複雜度: O(1)**，因為英文字母的數量是有限的（26個），所以雜湊表或陣列的大小是固定的，為常數空間。
    - **排序解法**:
        - **時間複雜度: O(N log N)**，主要開銷來自排序演算法。
        - **空間複雜度: O(N)**，`sorted()` 函式通常需要額外的 O(N) 空間來儲存排序後的列表。
- **Takeaway**:
> 這是「頻率計數」的經典應用場景。雜湊表 (或固定大小的陣列) 是實現頻率計數的最佳工具，能提供 O(N) 的時間複雜度。
> **在面試中，能夠實現雜湊表解法，並能與排序解法進行時空複雜度的比較分析，可以充分展示你對基礎演算法的扎實掌握。**
