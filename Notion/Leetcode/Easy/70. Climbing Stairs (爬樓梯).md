- **題號**: 70
- **核心標籤**: `動態規劃 (Dynamic Programming)`
- **題目精髓**: 這是一道入門級的動態規劃題目，堪稱 DP 領域的 "Hello, World!"。它完美地展示了如何將一個問題分解為更小的、重複的子問題，並從子問題的解逐步構建出最終答案。
- **關鍵思想：斐波那契數列 (Fibonacci Sequence)**
    - 要到達第 `n` 階，只有兩種方法：從 `n-1` 階跨 1 步，或從 `n-2` 階跨 2 步。
    - 因此，到達第 `n` 階的方法總數，等於到達 `n-1` 階和 `n-2` 階的方法數之和。
    - 這導出了核心的**狀態轉移方程**: `f(n) = f(n-1) + f(n-2)`。
- **動態規劃識別信號 (How to Spot a DP Problem)**:
    1. **問題類型**: 題目通常要求計算「**總共有多少種方法**」、「**最大/最小值**」或「**是否存在某種可能**」。
    2. **最優子結構**: 一個大問題的解，可以由一個或多個**規模更小的、同類型問題**的解構成。 (e.g., `f(n)` 的解依賴於 `f(n-1)` 和 `f(n-2)` 的解)。
    3. **重疊子問題**: 在遞迴求解過程中，很多**相同的子問題會被重複計算**多次。DP 通過「查表」或「備忘錄」來儲存子問題的解，避免重複計算，從而提高效率。
- **演算法模式 (DP 解題標準流程)**:
    1. **定義 **`**dp**`** 陣列的含義**: `dp[i]` 表示爬到第 `i` 階總共有多少種方法。
    2. **找出遞迴關係式 (狀態轉移方程)**: `dp[i] = dp[i-1] + dp[i-2]`。
    3. **初始化 **`**dp**`** 陣列 (Base Cases)**: `dp[1] = 1` (爬到第1階只有1種方法), `dp[2] = 2` (爬到第2階有 `1+1` 和 `2` 兩種方法)。
    4. **確定遍歷順序**: 根據關係式，計算 `dp[i]` 需要先知道 `dp[i-1]` 和 `dp[i-2]`，所以遍歷順序必須是**從左到右、自底向上 (Bottom-up)**。
- **複雜度分析**:
    - **時間複雜度: O(N)**
        - 我們需要一個迴圈來計算從第 3 階到第 N 階的方法數，迴圈大小為 N。
    - **空間複雜度**:
        - **標準解法: O(N)** - 需要一個長度為 N 的 `dp` 陣列來儲存所有子問題的解。
        - **優化解法: O(1)** - 因為計算 `dp[i]` 只依賴前兩項，所以我們不需要儲存整個陣列，只需用兩個變數 (`a`, `b`) 滾動記錄前兩個狀態即可。
- **takeaway**:
> 「爬樓梯」是學習動態規劃的基石。務必掌握其核心思想：定義狀態、尋找狀態轉移方程、處理基本情況。當遇到求「方法總數」或「極值」的題目時，應主動思考是否能將問題分解為依賴於先前狀態的子問題，從而判斷是否為 DP 問題。
