- **題號**: 21
- **核心標籤**: `鏈結串列 (Linked List)`, `雙指針 (Two Pointers)`
- **題目精髓**: 如何優雅地合併兩個已排序的鏈結串列，並串接成一個新的有序鏈結串列。
- **關鍵思想：虛擬頭節點 (Dummy Head) + 迭代**
    - 直接處理鏈結串列的頭節點往往需要寫額外的 `if` 判斷，容易出錯。
    - **最佳實踐**是建立一個**虛擬頭節點 (Dummy Head 或 Sentinel Node)**，例如 `dummy = ListNode()`。這個節點本身不儲存任何有效數據，它的 `next` 指針才指向新鏈結串列的真正開頭。
    - 這樣做可以**統一所有節點的新增邏輯**，讓程式碼更簡潔、不易出錯。
- **演算法模式 (Algorithm Pattern)**:
    1. 初始化一個虛擬頭節點 `dummy` 和一個移動指針 `curr = dummy`。`curr` 負責在新鏈結串列上移動並串接節點。
    2. 使用 `while` 迴圈，當兩個鏈結串列 `list1` 和 `list2` 都**不為空**時，進行比較。
    3. 比較 `list1.val` 和 `list2.val` 的大小。
    4. 將**值較小**的那個節點接到 `curr.next` 上，然後將該鏈結串列的指針向後移動一步。
    5. **不要忘記**將 `curr` 指針也向後移動一步 (`curr = curr.next`)，使其指向新串接上的節點。
    6. 迴圈結束後，最多只會有一個鏈結串列還有剩餘的節點。直接將 `curr.next` 指向那個剩餘的鏈結串列即可。
    7. 最終返回 `dummy.next`，這才是合併後鏈結串列的真正頭部。
- **複雜度分析**:
    - **時間複雜度: O(m + n)**
        - `m` 和 `n` 分別是兩個鏈結串列的長度。
        - 演算法需要遍歷兩個鏈結串列中的每一個節點恰好一次，才能完成合併。
    - **空間複雜度: O(1) (推薦解法)**
        - 這是指**直接串接 (rewire)** 原始鏈結串列節點的解法。
        - 演算法只使用了 `dummy` 和 `curr` 等幾個額外的指針變數，這些變數佔用的空間是固定的（常數級別），與輸入鏈結串列的長度無關。
    - **空間複雜度: O(m + n) (另一種解法)**
        - 這是指**創建新節點** (`ListNode(val)`) 的解法。
        - 演算法為原始鏈結串列中的每個元素都創建了一個全新的節點，因此額外空間與輸入規模成正比。在面試中，若無特殊要求，通常 O(1) 空間的解法更受青睞。
- * takeaway**:
> 處理鏈結串列問題時，如果涉及到構建或修改一個新的鏈結串列，優先考慮使用「虛擬頭節點」技巧來簡化頭部節點的處理邏輯。這是鏈結串列題目最常用、最重要的模式之一。
