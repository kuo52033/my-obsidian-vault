- **題號**: 268
- **核心標籤**: `陣列 (Array)`, `數學 (Math)`, `位元運算 (Bit Manipulation)`
- **題目精髓**: 這是一道經典的謎題型演算法題。它旨在測試你是否能跳出常規的排序或雜湊表思路，利用問題本身的數學或位元特性，找到一個時間和空間效率都極高的解法。
- **關鍵思想：利用「完整」與「殘缺」的差異**
    - 題目的核心是，我們有一個「完整」的數字序列（從 `0` 到 `n`），和一個「殘缺」的輸入陣列 `nums`（`nums` 比完整序列正好少了一個數字）。
    - 我們的目標就是找出這兩者之間的差異。
- **解法一：數學法 (高斯求和)**
    - **思路**: 這是你使用的解法，也是最直觀的優雅解法之一。
        1. 計算出「完整」序列 `0, 1, ..., n` 的總和 `expected_sum`。
        2. 計算出「殘缺」陣列 `nums` 的實際總和 `actual_sum`。
        3. `expected_sum - actual_sum` 的差值，正好就是那個缺失的數字。
    - **優化**: 計算 `expected_sum` 時，可以使用**高斯公式 **`**n * (n + 1) / 2**` 來替代迴圈，使程式碼更為簡潔。
    - **一行程式碼**: `return (len(nums) * (len(nums) + 1) // 2) - sum(nums)`
- **解法二：位元運算法 (XOR)**
    - **思路**: 這是一個非常巧妙的解法，利用了 XOR 運算「相同為 0，相異為 1」以及「`a ^ a = 0`」的特性來「抵銷」成對出現的數字。
    - **演算法模式**:
        1. 將 `0` 到 `n` 的**完整序列**（包含索引和 `n` 本身）和**殘缺序列** `nums` 中的所有數字放在一起。
        2. 在這個混合集合中，除了那個缺失的數字外，其他所有數字都恰好出現了兩次（一次是作為索引，一次是作為 `nums` 中的值）。
        3. 將這個混合集合中的所有數字進行 XOR 運算。成對出現的數字會相互抵銷為 0，最終剩下的就是那個只出現了一次的缺失數字。
    - **參考程式碼**:Python
`missing = len(nums)
for i, num in enumerate(nums):
    missing ^= i ^ num
return missing`
- **複雜度分析**:
    - **時間複雜度: O(N)**
        - N 是 `nums` 陣列的長度。數學法和位元運算法都需要對陣列進行一次完整的遍歷。
    - **空間複雜度: O(1)**
        - 兩種最優解法都只使用了固定的額外變數，與輸入大小無關。
- **Takeaway**:
> 當問題涉及到在一個近乎完整的連續序列中尋找缺失或重複的元素時，應優先考慮「數學求和法」和「位元運算 XOR 法」。

> 這兩種方法都能在 O(N) 時間和 O(1) 空間內解決問題，是處理此類問題的標準最優解，能夠在面試中充分展示你的巧思和對不同演算法工具的掌握。
