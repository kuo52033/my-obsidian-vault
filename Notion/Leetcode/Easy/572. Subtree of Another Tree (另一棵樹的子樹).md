- **題號**: 572
- **核心標籤**: `樹 (Tree)`, `遞迴 (Recursion)`, `深度優先搜尋 (DFS)`
- **難度星星**: ★★☆☆☆ (2 / 5 顆星)
    - **評語**: 這是一道「Easy」等級的題目，但它巧妙地結合了「搜尋」和「比較」兩種遞迴，是樹遞迴中一個非常重要的進階模式。它比「相同的樹」(100) 或「反轉二元樹」(226) 要複雜一步，是練習「雙重遞迴」的絕佳題目。
- **題目精髓**:
題目要求判斷 `subRoot` 這棵樹，是否與 `root` 這棵大樹中的**任意一個節點**所形成的子樹**完全相同**。
- **關鍵思想：拆解為「搜尋」與「比較」**
    - 這是本題的最核心思想。不能用一個函式同時完成兩件事，必須將職責拆分：
    1. **任務 A (搜尋)**: `isSubtree` 函式。它的職責是在 `root` 大樹上進行**深度優先搜尋 (DFS)**，為 `subRoot` 尋找可能的「匹配起點」。
    2. **任務 B (比較)**: `isSameTree` 輔助函式。它的職責是判斷兩棵樹是否**一模一樣**（結構、值都相同）。
- **演算法模式 (雙重遞迴)**
    1. `**isSubtree(root, subRoot)**`** (搜尋函式 - OR 邏輯)**
        - **Base Case 1**: `if subRoot is None: return True` (空樹是任何樹的子樹)。
        - **Base Case 2**: `if root is None: return False` (在大樹中沒找到)。
        - **遞迴步驟 (三種可能性)**：`subRoot` 存在於 `root` 中，若且唯若以下**任一 (OR)** 條件成立：
            - **a. 當前節點就匹配**: `self.isSameTree(root, subRoot)`
            - **b. 藏在左子樹中**: `self.isSubtree(root.left, subRoot)`
            - **c. 藏在右子樹中**: `self.isSubtree(root.right, subRoot)`
        - **返回**: `return (a) or (b) or (c)`
    2. `**isSameTree(p, q)**`** (比較函式 - AND 邏輯)**
        - **Base Cases**:
            - `if p is None and q is None: return True` (兩者皆空，相同)。
            - `if p is None or q is None or p.val != q.val: return False` (結構不同或值不同，不相同)。
        - **遞迴步驟**: 必須**同時 (AND)** 滿足：
            - **a. 左子樹完全相同**: `self.isSameTree(p.left, q.left)`
            - **b. 右子樹完全相同**: `self.isSameTree(p.right, q.right)`
        - **返回**: `return (a) and (b)`
- **複雜度分析**:
    - **時間複雜度: O(M * N)**
        - M 是 `root` 的節點數，N 是 `subRoot` 的節點數。在最壞的情況下，`isSubtree` 會遍歷 `root` 的所有 M 個節點，並且在每個節點上都呼叫一次 `isSameTree`（N 次操作）。
    - **空間複雜度: O(H)**
        - H 是 `root` 樹的高度。空間主要消耗在 `isSubtree` 的遞迴呼叫堆疊上。
- **Takeaway**:
> 這是樹的「遞迴套遞迴」的經典模式。學會將一個複雜的樹問題，拆解成一個「搜尋函式 (用 OR 邏輯)」和一個「比較函式 (用 AND 邏輯)」，是解決許多樹相關問題（如子結構、路徑）的關鍵技巧。
