- **題號**: 202
- **核心標籤**: `雜湊表 (Hash Table)`, `雙指針 (Two Pointers)`, `數學 (Math)`
- **題目精髓**: 這題表面上是一個數字計算問題，但其**核心是一個循環偵測 (Cycle Detection) 問題**。一個數字的「快樂之旅」序列，要嘛最終會抵達 `1`，要嘛會陷入一個無限循環。解題的關鍵就在於如何偵測這個循環。
- **關鍵思想：識別「隱式鏈結串列」(Implicit Linked List)**
    - 我們可以把這個數字變換的過程，抽象地看作一個鏈結串列：
        - **節點 (Node)**: 當前的數字 `n`。
        - `**next**`** 指針**: 計算下一個數字的函式 `getNextNum(n)`。
    - 如此一來，這個問題就完全轉化成了我們熟悉的 **141. 環形鏈結串列** 問題。
- **解法一：使用雜湊集合 (Hash Set) - O(N) 空間**
    - **思路**: 這是最直觀的方法。我們一路計算，並把所有出現過的數字都記錄在一個集合 `seen` 中。
    - **步驟**:
        1. 建立一個 `seen` 集合。
        2. 在迴圈中，不斷計算下一個數字 `n`。
        3. 每次計算出新的 `n` 之前，先檢查它是否已經在 `seen` 集合中。
        4. 如果存在，代表出現了循環，且這個循環沒有 `1`，因此返回 `False`。
        5. 如果不存在，就將 `n` 加入 `seen` 集合，繼續計算。
        6. 如果 `n` 最終變為 `1`，則返回 `True`。
    - **優點**: 思路簡單，易於實現。
- **解法二：快慢指針法 (Floyd's Cycle-Finding) - O(1) 空間**
    - **思路**: 既然是「隱式鏈結串列」，我們就可以使用「龜兔賽跑」演算法來偵測循環，從而避免使用額外空間。
    - **步驟**:
        1. 初始化 `slow` 指針和 `fast` 指針，都指向初始數字 `n`。
        2. 在迴圈中，`slow` 每次走一步 (`slow = getNextNum(slow)`)，`fast` 每次走兩步 (`fast = getNextNum(getNextNum(fast))`)。
        3. 迴圈的終止條件有兩個：
            - `**fast == 1**`: 如果快指針先到達 `1`，證明是快樂數。
            - `**slow == fast**`: 如果快慢指針相遇，證明進入了循環，不是快樂數。
    - **優點**: 空間複雜度為 O(1)，是空間上的最優解，能體現對演算法模式的深刻理解。
- **複雜度分析**:
    - **時間複雜度: O(logN)** (或 O(k)，k 為到達循環前的路徑長度)
        - 數字 `n` 在計算各位數平方和後，其值不會無限增大，而是會收斂到一個有限的範圍內，因此路徑和循環的長度是有限的，不會造成無限迴圈。
    - **空間複雜度**:
        - **解法一**: O(N)，N 是到達循環前的路徑長度，用於儲存 `seen` 集合。
        - **解法二**: O(1)，只使用了固定的指針變數。
- **Takeaway**:
> 這題是訓練「模式識別」能力的絕佳範例。一個看似無關的數學問題，其底層結構可能是一個我們非常熟悉的演算法模式（如鏈結串列循環偵測）。能夠看穿問題的表象，將其抽象化並套用合適的演算法模型，是從「解題」到「解決問題」的關鍵一步。
