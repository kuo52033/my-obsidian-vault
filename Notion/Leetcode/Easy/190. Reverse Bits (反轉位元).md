- **題號**: 190
- **核心標籤**: `位元運算 (Bit Manipulation)`
- **題目精髓**: 這是一道純粹的位元運算題，旨在測試對 `&`, `|`, `<<`, `>>` 等基本位元運算子的熟練運用。解題的關鍵是將 32 位元的整數看作一個位元序列，而非一個數值。
- **關鍵思想：逐位取出與放入 (Bit-by-Bit Extraction and Placement)**
    - 演算法的核心是建立一個迴圈，執行 32 次，模擬位元的「搬家」過程。
    - 在每一次迴圈中，我們執行兩個主要操作：
        1. **從輸入 **`**n**`** 中**：取出其最右邊（最低位）的位元。
        2. **向結果 **`**result**`** 中**：將取出的位元「附加」到 `result` 的最前面（最高位方向）。
- **演算法模式 (標準位元反轉迴圈)**
    1. 初始化 `result = 0`。
    2. 建立一個執行 32 次的迴圈。
    3. 在迴圈的每一步：
        - **A. 左移騰出空間**: `result = result << 1`。將 `result` 的所有位元向左移動一格，這樣最右邊就會空出一個 `0`，準備接收新的位元。
        - **B. 取得 **`**n**`** 的最低位**: `bit = n & 1`。利用 AND 運算，只保留 `n` 的最低位元（結果會是 0 或 1）。
        - **C. 放入新位元**: `result = result | bit`。利用 OR 運算，將剛取出的 `bit` 填入 `result` 最右邊的空位上。
        - **D. 更新 **`**n**`: `n = n >> 1`。將 `n` 的所有位元向右移動一格，以便在下一輪迴圈中處理下一個位元。
    - **最簡潔的寫法 (合併 A, B, C)**:Python
`# 在迴圈中
result = (result << 1) | (n & 1)
n = n >> 1`
- **複雜度分析**:
    - **時間複雜度: O(1)**
        - 迴圈的執行次數是固定的 32 次，與輸入 `n` 的數值大小無關。因此，操作次數為常數。
    - **空間複雜度: O(1)**
        - 只使用了 `result`, `bit` 等固定數量的變數，空間使用為常數。
- **Takeaway**:
> 這題是位元操作的基礎模板。result = (result << 1) | (n & 1) 是一個非常強大且常見的程式碼片段 (idiom)，用於逐位構建一個新的數字。當遇到需要逐一處理位元的問題時，應熟練運用這個模式。
