- **題號**: 217
- **核心標籤**: `陣列 (Array)`, `雜湊表 (Hash Table)`, `集合 (Set)`
- **題目精髓**: 這是一道經典的「雜湊集合」入門題。其核心在於如何利用高效的資料結構來快速查找一個元素是否存在於一個集合中，從而判斷重複。
- **關鍵思想：利用 Set 的唯一性**
    - Python 中的 `set` 資料結構有一個關鍵特性：它只儲存**唯一 (unique)** 的元素。
    - 如果一個包含重複元素的列表被轉換成 `set`，所有重複的元素都會被自動去除。
    - 因此，我們可以透過比較原始列表的長度和轉換後 `set` 的長度來判斷是否存在重複。
- **解法一：利用 **`**set**`** 特性的 Pythonic 解法 (O(N) 空間)**
    - **思路**: 這也是你使用的解法，是此題最簡潔、最巧妙的寫法。
    - **一行程式碼**: `return len(nums) != len(set(nums))`
    - **邏輯**:
        - 如果 `len(nums) == len(set(nums))`，說明沒有元素在轉換過程中被移除，因此所有元素都是唯一的。
        - 如果 `len(nums) != len(set(nums))`，說明有元素被移除了，因此存在重複。
    - **優點**: 極度簡潔，可讀性高。
- **解法二：手動遍歷與雜湊集合 (O(N) 空間)**
    - **思路**: 這是更為基礎、更能體現演算法底層邏輯的解法，也是面試中常規的期望答案。
    - **演算法模式**:
        1. 建立一個空的雜湊集合 `seen = set()`。
        2. 遍歷 `nums` 列表中的每一個數字 `num`。
        3. 對於每個 `num`，檢查它是否已經在 `seen` 集合中 (`if num in seen:`)。
        4. 如果是，說明遇到了重複元素，立即 `return True`。
        5. 如果不是，將 `num` 加入 `seen` 集合中 (`seen.add(num)`)。
        6. 如果迴圈完整結束都沒有返回，說明沒有重複，`return False`。
    - **優點**: 具有**提前退出 (Early Exit)** 的潛力，一旦找到第一個重複項就結束，在某些情況下可能比解法一更快。更能展示對資料結構基本操作的理解。
- **複雜度分析**:
    - **時間複雜度: O(N)**
        - N 是 `nums` 列表的長度。無論是構建 `set` 還是手動遍歷，都需要線性時間。`set` 的 `in` 和 `add` 操作的平均時間複雜度為 O(1)。
    - **空間複雜度: O(N)**
        - 在最壞的情況下（沒有重複元素），`set` 需要儲存所有 N 個元素，因此需要線性空間。
- **Takeaway**:
> 這是「雜湊集合」的標誌性應用場景。當任何問題涉及到「去重」、「檢查元素是否存在」、「統計唯一元素個數」時，第一個應該想到的資料結構就是 set。
> **務必理解 **`**set**`** 提供的 O(1) 平均時間複雜度的查找和插入操作是解決這類問題高效的關鍵。**
