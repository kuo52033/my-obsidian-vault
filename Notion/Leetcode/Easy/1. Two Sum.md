- **核心標籤**: `陣列 (Array)`, `雜湊表 (Hash Table)`
- **題目精髓**: 在一個序列中，尋找是否存在兩個元素滿足特定關係 (如 `a + b = target`)。
- **關鍵思想：空間換時間**
    - 暴力解法是使用兩層 `for` 迴圈，時間複雜度為 O(n²)，在多數情況下會超時。
    - **最佳解法**是利用**雜湊表 (在 Python 中是 **`**dict**`**)**，將時間複雜度優化到 **O(n)**。
- **演算法模式 (Algorithm Pattern)**:
    1. 建立一個雜湊表 `mapping` (或 `seen`, `num_map`) 來儲存已遍歷過的元素及其資訊。
    2. 遍歷一次輸入陣列 `nums`。
    3. 在迴圈的每一步，計算當前元素 `num` 所需的「配對值」 `complement = target - num`。
    4. **查找 **`**mapping**`: 檢查 `complement` 是否已經存在於 `mapping` 中。
        - **如果存在**: 代表我們找到了解答，立即回傳結果。
        - **如果不存在**: 將**當前元素和它的索引**存入 `mapping` 中 (`mapping[num] = index`)，以供後續的元素進行配對。
- **Python 實用技巧**:
    - `enumerate(nums)`: 在 `for` 迴圈中同時獲取**索引 (index)** 和**值 (value)**，是處理這類需要回傳索引的問題時的最佳實踐。
    - `in` 關鍵字: `if complement in mapping:`，對字典的 `key` 進行查找，其平均時間複雜度為 **O(1)**，這是整個優化的核心。
- * takeaway**:
> 遇到需要在陣列中尋找配對或關係的題目時，第一時間就要想到「雜湊表」。它能將查找時間從 O(n) 降至 O(1)，是優化時間複雜度的首選利器。
