- **題號**: 136
- **核心標籤**: `陣列 (Array)`, `位元運算 (Bit Manipulation)`
- **題目精髓**: 在一個陣列中，除了某個元素只出現一次外，其餘每個元素都恰好出現兩次。目標是在**線性時間複雜度 O(N)** 和**常數級額外空間 O(1)** 的限制下，找出那個只出現一次的元素。
- **關鍵思想：利用位元運算 XOR 的神奇特性**
    - 這個問題的限制條件（特別是 O(1) 空間）強烈暗示了不能使用雜湊表等需要額外空間的資料結構。
    - **最佳解法**是利用 XOR（互斥或）運算子的特性：
        1. `**A ^ A = 0**`: 任何數字與其自身進行 XOR 運算，結果為 0。
        2. `**A ^ 0 = A**`: 任何數字與 0 進行 XOR 運算，結果為其自身。
        3. **交換律**: `A ^ B = B ^ A`。
    - 當我們將陣列中**所有**數字連續進行 XOR 運算時，所有成對出現的數字都會因為 `A ^ A = 0` 而兩兩相互「抵銷」為 0。最終，只剩下那個單獨的數字與 0 進行運算，根據 `A ^ 0 = A`，結果就是它自己。
- **演算法模式 (XOR 消除法)**
    1. 初始化一個變數 `result = 0`。
    2. 遍歷 `nums` 陣列中的每一個數字 `num`。
    3. 執行 `result = result ^ num`。
    4. 迴圈結束後，`result` 的值就是那個只出現一次的數字。
- **舉例**: 對 `nums = [4, 1, 2, 1, 2]`
    - `result = 0 ^ 4` -> `4`
    - `result = 4 ^ 1` -> `5`
    - `result = 5 ^ 2` -> `7`
    - `result = 7 ^ 1` -> `6`
    - `result = 6 ^ 2` -> `4`
    - (等價於 `(4) ^ (1^1) ^ (2^2) = 4 ^ 0 ^ 0 = 4`)
- **複雜度分析**:
    - **時間複雜度: O(N)**
        - N 是陣列的長度。我們只需要對陣列進行一次完整的遍歷。
    - **空間複雜度: O(1)**
        - 只使用了一個額外的變數 `result`，完全滿足題目不使用額外空間的要求。
- **Takeaway**:
> 當題目涉及到「尋找一個出現奇數次的元素，而其他都出現偶數次」或者類似的「配對與消除」場景時，要立刻想到 XOR 運算。這是位元運算在演算法中一個非常經典且強大的應用，是解決此類問題的最優解。
