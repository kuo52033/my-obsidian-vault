- **題號**: 338
- **核心標籤**: `動態規劃 (Dynamic Programming)`, `位元運算 (Bit Manipulation)`
- **題目精髓**: 題目要求為從 `0` 到 `n` 的每個數字計算其二進位表示中 `1` 的個數，並返回一個陣列。關鍵挑戰在於找到一個**線性時間 O(N)** 的解法，避免對每個數字都進行獨立的位元計數（那樣會是 O(N log N)）。
- **關鍵思想：利用已計算的結果 (DP)**
    - 動態規劃的核心在於 `dp[i]` 的值可以由 `i` 之前的某個或某些 `dp` 值推導出來。這題的巧妙之處在於，數字 `i` 的位元個數與更小的數字（如 `i/2`）的位元個數有著緊密的數學關係。
- **解法一：DP + 最低有效位 (LSB) - (你的解法)**
    - **思路**: 這是最常見且直觀的 O(N) 解法。一個數字 `i` 的位元數，與其**右移一位**後的數字 `i >> 1` (即 `i/2`) 的位元數息息相關。
    - **狀態轉移方程**: `dp[i] = dp[i >> 1] + (i & 1)`
    - **邏輯**:
        - `i >> 1`：將 `i` 的二進位表示向右移動一位，相當於去掉了 `i` 的**最低位**。
        - `i & 1`：這個操作可以取出 `i` 的**最低位**（偶數為0，奇數為1）。
        - 因此，`i` 的 `1` 的總數 = (`i` 去掉最低位後剩下的部分) 的 `1` 的總數 + (最低位本身是否為 `1`)。
- **解法二：DP + 最高有效位 (MSB)**
    - **思路**: 觀察 `dp` 陣列的規律，會發現結果是以**2的冪次方**為邊界，呈現塊狀的重複模式。
        - `dp[2..3]` = `dp[0..1]` 各自加 1。
        - `dp[4..7]` = `dp[0..3]` 各自加 1。
    - **狀態轉移方程**: `dp[i] = dp[i - offset] + 1`
    - **邏輯**:
        - `offset` 是不大於 `i` 的最大2的冪次方（例如，當 `i=5,6,7` 時，`offset=4`）。
        - `i - offset` 這個操作，相當於去掉了 `i` 的**最高有效位 (MSB)**。
        - 因此，`i` 的 `1` 的總數 = (`i` 去掉最高位後剩下的部分) 的 `1` 的總數 + (最高位那個 `1`)。
- **複雜度分析 (對兩種 DP 解法)**:
    - **時間複雜度: O(N)**
        - 我們只需要一個迴圈，從 `1` 遍歷到 `n`。在迴圈的每一步，計算 `dp[i]` 的操作都是 O(1) 的，因為它只依賴於已經計算好的 `dp` 值。
    - **空間複雜度: O(N)**
        - 需要一個長度為 `n+1` 的 `dp` 陣列來儲存結果。
- **Takeaway**:
> 這題是位元運算與動態規劃完美結合的典範。它告訴我們，在處理一系列相關的計算時，要積極尋找它們之間的遞迴關係，以避免重複勞動。
> **特別是 **`**dp[i]**`** 與 **`**dp[i >> 1]**`** 之間的關係，是位元類 DP 問題中一個極其常用且強大的模式，務必掌握。**
