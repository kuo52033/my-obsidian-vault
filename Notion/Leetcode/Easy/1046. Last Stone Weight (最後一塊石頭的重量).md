## 題目精髓

模擬一個「石頭互撞」的過程。在每一步,都需要從當前石頭堆中,選出最重的兩塊石頭進行碰撞,然後將可能的「殘骸」(新石頭)放回堆中,重複此過程直到剩下最多一塊石頭。

## 解題思路

### 關鍵思想:最大堆 (Max-Heap)

- 題目的核心需求是「**不斷地取出最大的兩個元素**」
- 這種「需要頻繁存取極值(最大或最小值)」的場景,正是**堆 (Heap) / 優先佇列 (Priority Queue)** 的經典應用場景
- 由於我們需要的是「最大值」,所以應該使用「**最大堆 (Max-Heap)**」

### 演算法模式 (用 Min-Heap 模擬 Max-Heap)

Python 的 `heapq` 模組只提供了「最小堆 (Min-Heap)」(即 `heappop` 總會彈出最小值)。

**巧妙的技巧**: 你的解法中完美地使用了這個技巧——在存入 `stones` 時,**全部存入它們的相反數(負值)**。

### 邏輯步驟

1. `max_heap = [-x for x in stones]`: 將所有石頭重量變為負數
2. `heapq.heapify(max_heap)`: O(N) 時間內建立一個最小堆。此時,堆頂 `max_heap[0]` 是最負的數,代表它對應的原始石頭是最重的
3. `while len(max_heap) &gt; 1:`: 只要堆裡還有至少兩塊石頭,就繼續碰撞
4. `a = heapq.heappop(max_heap)`: 彈出最負的數(例如 -10),代表最重的石頭 `y`
5. `b = heapq.heappop(max_heap)`: 彈出第二負的數(例如 -8),代表第二重的石頭 `x`
6. `if a == b: continue`: 如果兩者相等(例如 -8 和 -8),代表石頭一樣重,同歸於盡,不需 `push` 新石頭回去
7. `heapq.heappush(max_heap, -abs(a-b))`:
    - `a` 和 `b` 都是負數,`abs(a-b)` 等同於原始石頭的 `y - x`
    - 我們需要把這個差值(例如 `10 - 8 = 2`)的**相反數**(`-2`)`push` 回堆中,以維持 Max-Heap 的特性

### 返回結果

- `return -max_heap[0] if len(max_heap) &gt; 0 else 0`:
    - 如果堆中還剩一個元素(例如 `[-2]`),返回它的正值 `-(-2) = 2`
    - 如果堆為空,返回 `0`

## 複雜度分析

- **時間複雜度: O(N log N)**
    - `heapify` 建立堆需要 O(N)
    - 迴圈最多執行 N-1 次
    - 每次迴圈都包含 `heappop` (O(log N)) 和 `heappush` (O(log N))
    - 總時間為 O(N + N log N),簡化為 O(N log N)
- **空間複雜度: O(N)**
    - 需要 O(N) 的額外空間來儲存 `max_heap`
