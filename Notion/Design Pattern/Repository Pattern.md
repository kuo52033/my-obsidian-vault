> [!tip] 💡
> The Repository Design Pattern is a crucial framework in software development. It neatly organizes data access logic, separating it from the core application. This separation makes code more reusable, maintainable, and easier to test

- simplifying data access: an intermediarary layer between an application’s logic and data storage
-  works as a librarian between a program and data (like books in a library). Instead of the program directly looking for data, it asks for repository to find save, update, or delete the data it needs.

### Advantages

- centralized data access
- enhanced testability
- improve code maintainability
- reuseability and extensibility
