## **1. IPv4 Overview**

### **Public IPv4**

- Reachable **from the internet**
- EC2 instances can receive a public IPv4 automatically
- **Released when the EC2 instance stops**
- When the instance starts again → gets a **new** public IP
- Cost: **$0.005 per hour** (ALL public IPv4s, including Elastic IPs)

### **Private IPv4**

- Used **inside private networks** such as AWS VPC
- Not reachable from the internet
- Examples: `192.168.x.x`, `10.x.x.x`, `172.31.x.x`
- **Stays the same** for the entire instance lifetime
(even after stop/start)

---

## **2. Elastic IP (EIP)**

A **fixed public IPv4** you can attach to an EC2 instance.

### **Key Characteristics**

- Public IPv4 that **never changes** unless you release it
- Useful when:
    - You need a stable endpoint
    - You run self-managed servers
- Pricing Rule:
    - Charged the same as public IPv4 (**$0.005/hr**)
    - Extra charges apply if:
        - The Elastic IP is **allocated but not attached**
        - You have more than one EIP per instance

> AWS encourages users to avoid unnecessary use of EIPs.

---

## **3. IPv6 Overview**

### **Key Characteristics**

- Much larger address space: **3.4×10³⁸ addresses**
- All IPv6 addresses in AWS are **publicly routable**
- **No private IPv6 range**
- IPv6 addresses in AWS are **free**

### **Use Cases**

- Exposing workloads publicly without public IPv4 cost
- Modern architectures ready for future internet scale

# ⭐ **CLF-C02 Exam Tips**

### **1. Public vs Private IPv4**

| Type | Behavior | Internet Reachable | Persistent? |
| --- | --- | --- | --- |
| **Public IPv4** | Auto-assigned | Yes | ❌ Lost on stop/start |
| **Private IPv4** | VPC internal | No | ✔ Stays |
| **Elastic IP** | Fixed public IP | Yes | ✔ Stays |

---

### **2. Elastic IP Notes**

- Stick to one Elastic IP per instance (AWS best practice)
- Additional charges when:
    - EIP is not attached
    - You allocate multiple EIPs
- Encourages IPv6 adoption

---

### **3. IPv6 Key Points**

- Always **public**
- **Free** in AWS
- No concept of private IPv6
- Useful for public-facing services without public IPv4 charges

---

### **4. Typical Exam Question Patterns**

> “Which IP stays the same after stopping and starting an EC2 instance?”
> ✔ Private IPv4
> 
> ✔ Elastic IP
> 
> ❌ Public IPv4

> “Which IP type is charged in AWS?”
> ✔ Public IPv4 & Elastic IP
> 
> ❌ IPv6

> “Your instance needs a fixed public address—what should you use?”
> ✔ Elastic IP

> “You want public internet access with zero IP cost—what should you choose?”
