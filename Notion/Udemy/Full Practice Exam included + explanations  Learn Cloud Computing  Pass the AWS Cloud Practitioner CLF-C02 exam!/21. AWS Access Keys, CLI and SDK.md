## 🔑 Three Ways to Access AWS

The video explains that there are three main methods to interact with AWS services:

1. **AWS Management Console**: The web interface protected by your password and MFA (what we have used so far).
2. **AWS CLI (Command Line Interface)**: Access via the terminal/command line.
3. **AWS SDK (Software Development Kit)**: Access via application code.

*Note: Both the CLI and SDK are protected by ****Access Keys****.*

---

## 🛡️ What are Access Keys?

- **Purpose**: Access Keys are the credentials used by the **CLI** and **SDK** to authenticate your identity (instead of a username/password).
- **Components**: They consist of an **Access Key ID** (like a username) and a **Secret Access Key** (like a password).
- **Security Warning**:
    - You generate these in the Management Console.
    - You act as the owner of these keys.
    - **DO NOT SHARE THEM.** Treat them as highly confidential secrets. If you share them, others can access your account and potentially cause financial or security damage.

---

## 💻 AWS CLI (Command Line Interface)

- **Definition**: A tool that allows you to interact with AWS services using commands in your command-line shell.
- **Usage**: Commands always start with `aws` (e.g., `aws s3 cp`).
- **Benefits**:
    - Provides direct access to AWS public APIs.
    - Excellent for developing **scripts** to automate tasks.
    - It is open-source (available on GitHub).
    - Some users prefer it entirely over the Management Console.

---

## 📦 AWS SDK (Software Development Kit)

- **Definition**: A set of language-specific libraries that allow you to access and manage AWS services **programmatically** (embedded within your application code).
- **Supported Languages**: JavaScript, Python, PHP, .NET, Ruby, Java, Go, Node.js, C++, etc.
- **Special SDKs**: AWS also offers Mobile SDKs (Android, iOS) and IoT SDKs (Internet of Things).
- **Fun Fact**: The AWS CLI is actually built using the AWS SDK for Python (named **Boto**).
