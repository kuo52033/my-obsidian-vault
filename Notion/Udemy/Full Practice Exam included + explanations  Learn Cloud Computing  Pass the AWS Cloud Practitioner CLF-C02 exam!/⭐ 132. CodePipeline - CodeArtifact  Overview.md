## What is CodePipeline?

AWS CodePipeline is a **fully managed CI/CD orchestration service**

that automates the steps required to **build, test, and deploy** your application.

It connects services like:

- CodeCommit (source)
- CodeBuild (build)
- CodeDeploy / Elastic Beanstalk / CloudFormation (deploy)
- Third-party tools (GitHub, Jenkins, etc.)

---

## Why use CodePipeline?

### Key Benefits

- **Fully automated** continuous delivery workflow
- Integrates with most AWS developer tools
- Supports approval gates, manual steps, and custom actions
- Fast feedback cycle — runs automatically after code changes
- Fully managed (no servers to provision)

---

## How it works

A typical pipeline example:

1. **Source Stage**
    - Get code from CodeCommit / GitHub
2. **Build Stage**
    - Compile & test the application using CodeBuild
3. **Deploy Stage**
    - Deploy using CodeDeploy or Elastic Beanstalk
4. **Optional**
    - Manual approval
    - Integration tests
    - Deploy to multiple environments

Pipeline = a chain of stages that run automatically on each code change.

---

## When to use CodePipeline?

Any time the exam mentions:

- “Orchestrate build-test-deploy”
- “Automate software release process”
- “CI/CD pipeline”
- “Trigger deployment when code changes”

→ **Think CodePipeline**

## What is CodeArtifact?

AWS CodeArtifact is a **fully managed artifact repository service**

designed for storing **software packages and dependencies**.

It replaces the need to self-manage:

- Nexus
- Artifactory
- S3 bucket–based package storage

---

## Why use CodeArtifact?

### Key features

- Secure & scalable package storage
- IAM-based access control
- Works with major dependency managers:
    - npm / yarn (JavaScript)
    - pip / twine (Python)
    - Maven / Gradle (Java)
    - NuGet (.NET)
- Integrates with CodeBuild for dependency installation

---

## How it works

Developers can:

- **Publish** private packages to CodeArtifact
- **Download** dependencies during build
- **Share** code modules across teams safely

Example workflow:

```plain text
Developer → CodeCommit → CodeBuild → retrieves deps from CodeArtifact → build artifacts


```

---

## When to use CodeArtifact?

Use it when the question mentions:

- “centralized package repository”
- “dependency management”
- “secure internal packages”
- “avoid hosting Nexus / Artifactory”

→ **Think CodeArtifact**
