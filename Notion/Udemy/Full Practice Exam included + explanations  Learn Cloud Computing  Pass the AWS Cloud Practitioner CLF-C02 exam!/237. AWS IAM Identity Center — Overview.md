## **What is AWS IAM Identity Center?**

- **AWS IAM Identity Center** is the **successor of AWS Single Sign-On (SSO)**.
- Provides **one login (Single Sign-On)** to access:
    - **Multiple AWS accounts** in an AWS Organization
    - **Business cloud applications**
    - **SAML 2.0–enabled applications**
    - **EC2 Windows instances**

> ⚠️ Exam note: AWS SSO = IAM Identity Center (same concept)

---

## **Main Purpose (Exam Focus)**

- **Centralized user access management**
- **One set of credentials → many AWS accounts**
- Avoid managing users **account by account**

---

## **How It Works (High Level)**

1. User logs in **once** via a single URL
2. IAM Identity Center authenticates the user
3. User sees a **portal** listing:
    - AWS accounts
    - Allowed roles
    - Allowed applications
4. User clicks an account → **direct access to AWS Console**

---

## **Identity Sources (Where Users Are Stored)**

IAM Identity Center can use:

- **Built-in identity store** (native users)
- **External identity providers**, such as:
    - Microsoft Active Directory
    - Okta
    - OneLogin

---

## **What IAM Identity Center Is NOT**

- ❌ Not for application end users (that’s **Cognito**)
- ❌ Not IAM Users per account
- ❌ Not temporary credentials (that’s **STS**)

---

## **When to Use IAM Identity Center**

Use IAM Identity Center when you need:

- One login for **multiple AWS accounts**
- Centralized user & permission management
- SSO for AWS + business applications

---

## **Exam Keywords → Correct Answer**

If the question mentions:

- *Single sign-on*
- *One login for multiple AWS accounts*
- *Centralized access across an organization*
- *AWS SSO*

👉 **Answer: AWS IAM Identity Center**

## **Comparison (Very Exam-Friendly)**

| Use Case | Service |
| --- | --- |
| AWS admins (single account) | IAM |
| AWS admins (multi-account SSO) | **IAM Identity Center** |
| App users (web/mobile) | Cognito |
| Temporary credentials | STS |
| Corporate Windows login | Directory Services |
