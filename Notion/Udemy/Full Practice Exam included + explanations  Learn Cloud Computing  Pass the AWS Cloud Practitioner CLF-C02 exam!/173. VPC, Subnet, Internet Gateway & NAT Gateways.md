## **1. What is a VPC?**

A **VPC (Virtual Private Cloud)** is your own private network inside AWS.

- Region-scoped → each region has its own VPCs
- You deploy resources like EC2, RDS, etc. inside it
- You choose its **CIDR block** (IP address range)

---

## **2. Subnets**

A **subnet** is a slice of your VPC and is **mapped to exactly one Availability Zone**.

### **Types of Subnets**

### **Public Subnet**

- Has a route to the **Internet Gateway (IGW)**
- Instances can receive **public IPv4**
- Internet can reach resources directly
(e.g., EC2 web servers, load balancers)

### **Private Subnet**

- No direct route to the internet
- Common workloads:
    - Databases (RDS)
    - Backend EC2 instances
    - Internal services

---

## **3. Route Tables**

Route tables define **where traffic goes**.

Examples:

- Public subnet → route to **0.0.0.0/0 → Internet Gateway**
- Private subnet → route to **0.0.0.0/0 → NAT Gateway**
- Internal VPC traffic stays within subnets

---

## **4. Internet Gateway (IGW)**

Allows your VPC to send/receive traffic to/from the **public internet**.

A subnet becomes **public** when:

1. VPC has an Internet Gateway attached
2. Subnet’s route table contains a route:
**0.0.0.0/0 → Internet Gateway**

Use cases:

- EC2 instances that need to be accessible externally
- Public-facing load balancers

---

## **5. NAT Gateway / NAT Instance**

Used by **private subnet** instances to **access the internet** without becoming publicly reachable.

### **NAT Gateway**

- Fully managed by AWS
- Highly available within an AZ
- Scales automatically
- Requires a public subnet
- Requires an Elastic IP
- More expensive than NAT Instance

### **NAT Instance**

- Self-managed EC2 instance acting as NAT
- Must manage:
    - Patching
    - Scalability
    - Availability
- Not recommended unless special customization is needed

### **Flow**

Private Subnet → NAT Gateway → Internet Gateway → Internet

# ⭐ **CLF-C02 Exam Tips**

### **1. How to Identify Public vs Private Subnet**

| Subnet Type | Has Route to IGW? | Public IPv4 Allowed? | Internet Reachable? |
| --- | --- | --- | --- |
| **Public** | ✔ Yes | Often | ✔ Yes |
| **Private** | ❌ No | Not required | ❌ No |

---

### **2. NAT vs IGW**

| Feature | Internet Gateway (IGW) | NAT Gateway |
| --- | --- | --- |
| Who uses it? | Public subnets | Private subnets |
| Accept incoming traffic? | ✔ Yes | ❌ No |
| Provide outbound internet? | ✔ Yes | ✔ Yes (for private) |
| Requires EIP? | ❌ No | ✔ Yes |
| Managed? | Yes | Yes |
| Supports IPv6? | ✔ IGW supports | ❌ NAT Gateway is **IPv4 only** |

**Exam keyword:**

“Private instance needs access to the internet but shouldn’t be publicly reachable” → **NAT Gateway**

---

### **3. NAT Gateway Placement**

- NAT Gateway **must** be placed in a **public subnet**
- Private subnets route their default traffic to the NAT Gateway

---

### **4. VPC Structure Keywords (exam loves diagrams)**

Region → VPC (CIDR block) → AZ → Subnets (public/private) → Route Tables → IGW/NAT

---

### **Examples of Typical Exam Questions**

> “How do you give a private EC2 instance internet access?”
> ✔ NAT Gateway in a public subnet + route from private subnet

> “How do you make a subnet public?”
> ✔ Attach IGW + add route 0.0.0.0/0 → IGW

> “Which service lets instances be reachable from the internet?”
> ✔ Internet Gateway

> “Which allows outbound internet for private subnets?”
> ✔ NAT Gateway
