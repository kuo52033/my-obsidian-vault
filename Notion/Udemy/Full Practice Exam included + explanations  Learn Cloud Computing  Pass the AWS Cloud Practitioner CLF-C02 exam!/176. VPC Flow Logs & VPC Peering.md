# **1. VPC Flow Logs**

VPC Flow Logs capture **IP traffic metadata** going to and from:

- **VPC**
- **Subnet**
- **Elastic Network Interface (ENI)**
(e.g., EC2, ELB, RDS, ElastiCache)

### **Use Cases**

- Troubleshoot connectivity issues
- Diagnose why traffic is being **rejected**
- Audit traffic patterns for security teams
- Confirm whether traffic reached an instance or was blocked by SG/NACL

### **Flow Log Destinations**

You can send flow logs to:

- **Amazon S3**
- **CloudWatch Logs**
- **Kinesis Data Firehose**

### **What Flow Logs DO NOT capture**

(very common exam point)

- No packet content (metadata only)
- No DNS queries
- No Amazon DNS traffic
- No traffic to 169.254.169.254 (Instance Metadata Service)
- No traffic from management services

### **Flow Log Fields (examples)**

- Source/destination IP
- Ports
- Protocol
- Bytes/packets
- Action (ACCEPT/REJECT)
- Log status

### **Filters**

- All traffic
- Accepted traffic
- Rejected traffic

# **2. VPC Peering**

VPC Peering connects two VPCs **privately**, making them behave like one network.

### **Key Characteristics**

- Traffic stays **within AWS network** (private, not over the internet)
- Low latency, high-bandwidth connectivity
- Supports **inter-region** and **cross-account**

### **Requirements**

- **CIDR blocks must NOT overlap**
(If overlapping → cannot peer)

### **Not Transitive**

This is the most important rule.

If:

- VPC A ↔ VPC B (peered)
- VPC A ↔ VPC C (peered)

Then:

- **VPC B cannot talk to VPC C**

To enable B ↔ C communication →

You must create **another** peering connection (B ↔ C).

### **Routing**

For peering to work, each VPC must update its **route table**:

- VPC A route → VPC B CIDR → Peering connection
- VPC B route → VPC A CIDR → Peering connection

### **Peering can be:**

- **Same region or cross-region**
- **Same account or cross-account**

> “How to troubleshoot why a subnet cannot reach the internet?”
> ✔ VPC Flow Logs

> ✔ VPC Peering

> “VPC A can reach VPC B and C, but B cannot reach C. Why?”
> ✔ Peering is not transitive

> “Peering request rejected—why?”
> ✔ Overlapping CIDR ranges
