## What is API Gateway?

Amazon API Gateway is a **fully managed, serverless service** that allows developers to create, publish, maintain, monitor, and secure **HTTP APIs** in AWS.

It is commonly used when you want to expose **serverless backend logic**—for example, Lambda functions—to external clients through a public API endpoint.

---

## Why API Gateway?

### Problem:

- Lambda functions **cannot** be publicly accessed directly.
- External clients (web/mobile apps) need an accessible **HTTP endpoint**.

### Solution:

API Gateway acts as a **front door** to your serverless backend.

---

## Architecture Overview

```plain text
Client → API Gateway → Lambda → DynamoDB


```

- **Client** sends HTTP call
- **API Gateway** receives the request and routes it
- **Lambda** processes the logic (CRUD operations)
- **DynamoDB** stores the data
→ All components are **serverless** and automatically scale.

---

## Key Features of API Gateway

### ✔ Fully Serverless & Scalable

- Handles millions of requests automatically
- No infrastructure to manage

### ✔ Supports Multiple API Styles

- **RESTful APIs**
- **WebSocket APIs** (real-time communication)

### ✔ Built-in Security

- IAM authentication
- Cognito user pools
- Lambda authorizers
- API Keys
- Throttling to protect backend

### ✔ Monitoring & Logging

- Integrated with CloudWatch for logs & metrics

### ✔ Transformation & Validation

- Request/response mapping templates
- Input validation (schema-based)

---

## When to Use API Gateway?

Use it when you need:

- A **serverless HTTP API**
- Authentication/authorization for APIs
- Rate limiting / throttling
- Real-time WebSocket connections
- Easily exposing Lambda functions to the internet

**In AWS exam questions:**

> If you see “create a serverless API,” the answer is always API Gateway + Lambda.
