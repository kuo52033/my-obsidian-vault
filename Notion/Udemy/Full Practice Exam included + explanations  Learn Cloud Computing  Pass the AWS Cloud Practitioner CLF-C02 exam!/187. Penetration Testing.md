Penetration testing (pen testing) = intentionally attacking your own AWS resources to evaluate security.

AWS **allows customers** to perform security assessments **without prior approval** for a specific set of services.

---

# **1. Pen Testing Allowed Without AWS Approval**

You may test **your own infrastructure** on these services:

- **EC2** instances
- **NAT Gateways**
- **Elastic Load Balancers (ALB/NLB)**
- **Amazon RDS**
- **Amazon Aurora**
- **Amazon CloudFront**
- **API Gateway**
- **AWS Lambda & Lambda@Edge**
- **Amazon Lightsail**
- **Elastic Beanstalk environments**

> Exam does NOT require memorizing the list — just the concept:
> **Pen testing is allowed on your own resources.**

---

# **2. Activities NOT Allowed (Without Explicit Approval)**

These actions resemble real attacks and may be mistaken as attempts to harm AWS infrastructure.

### **Prohibited Without Approval**

- **DDoS or DoS attacks** (including simulated DDoS)
- **Port flooding**
- **Protocol flooding**
- **Request flooding**
- **DNS zone walking** on Route 53 Hosted Zones
- Any attempt to attack AWS-managed infrastructure directly

> AWS will block or flag these activities as malicious behavior.

---

# **3. If You Need to Perform Restricted Activities**

You must contact AWS Security / Support to request approval before performing:

- Unusual, high-volume traffic tests
- Tests that may impact AWS shared systems
- Anything resembling a real-world attack scenario

---

# ⭐ **CLF-C02 Exam Tips**

### **1. Can customers perform penetration testing?**

✔ Yes — customers can pen-test **their own** AWS resources.

### **2. Is approval required for common pen tests on EC2 or RDS?**

✔ No prior approval needed.

### **3. Are DDoS attacks allowed for testing?**

❌ Absolutely NOT.

### **4. Can you test Route 53 via DNS zone walking?**

❌ Not allowed.

### **5. General rule**

> Testing your own app? Usually allowed.
Testing AWS infrastructure? Never allowed.
