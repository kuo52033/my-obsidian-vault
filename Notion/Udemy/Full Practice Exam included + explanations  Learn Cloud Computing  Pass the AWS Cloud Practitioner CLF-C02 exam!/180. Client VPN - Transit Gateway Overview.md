AWS Client VPN allows **end users (laptops/desktops)** to securely connect to a VPC **using OpenVPN-based clients**.

---

## **1. What Client VPN Does**

- Lets your computer connect securely to an AWS VPC
- After connecting, your laptop behaves **as if it were inside the VPC**
- You can access private resources (e.g., EC2 private IPs, RDS private endpoints)
- Connection travels over the **public internet**, but encrypted

---

## **2. Why Use Client VPN**

- Need secure access from **remote workforce** to AWS private networks
- Developers/admins need to SSH/RDP into **private EC2 instances**
- Secure access to internal apps (bastion-less model)

---

## **3. Client VPN + On-Premises**

If your VPC is connected to on-premises via:

- Site-to-Site VPN, or
- Direct Connect

Then your laptop → Client VPN → VPC → on-premises

→ You gain private access to both AWS AND on-prem systems.

---

# ⭐ **CLF-C02 Exam Tips – Client VPN**

- User/laptop connecting to VPC? → **Client VPN**
- Need secure access to EC2 **private IPs**? → **Client VPN**
- Remote workforce connectivity? → **Client VPN**
- Not used for app-to-app connections (that's Site-to-Site VPN or PrivateLink)

# **181. Transit Gateway Overview**

Managing many VPC connections individually quickly becomes complex (peering, VPN, DX, etc.).

**Transit Gateway (TGW)** solves this.

---

## **1. What Transit Gateway Does**

Transit Gateway provides a **central hub** to connect:

- Multiple VPCs
- Site-to-Site VPN connections
- Direct Connect gateways
- On-premises data centers

This forms a **hub-and-spoke** (star) network architecture.

---

## **2. Why Use Transit Gateway**

Without TGW:

- VPC Peering requires **one-to-one connection**
- Peering is **not transitive**
- Complex routing when many VPCs exist

With TGW:

- All networks attach to ONE hub
- Routing becomes simplified
- It **scales to thousands of VPCs**

---

## **3. Supported Connections**

Transit Gateway supports:

- **Amazon VPCs**
- **Site-to-Site VPN**
- **Direct Connect Gateway**
- **Inter-region peering (TGW ↔ TGW)**

TGW is the **enterprise-grade** network backbone for AWS.

---

# ⭐ **CLF-C02 Exam Tips – Transit Gateway**

### **1. When to pick Transit Gateway**

If you see:

- “Connect **many** VPCs together”
- “Multi-account AWS Organization”
- “Hybrid environment (VPN + Direct Connect + VPCs)”
- “Scalable network architecture”
- “Hub-and-spoke design”

→ Correct answer = **Transit Gateway**

---

### **2. When NOT to use TGW**

- Connecting **only 2 VPCs** → use VPC Peering
- Connecting **one on-prem site** quickly → Site-to-Site VPN
- Need dedicated private connection → Direct Connect
- Need service-level private access → PrivateLink

---

### **3. Exam trick**

VPC Peering is **NOT transitive**,

but Transit Gateway **IS transitive** (via routing attachments).
