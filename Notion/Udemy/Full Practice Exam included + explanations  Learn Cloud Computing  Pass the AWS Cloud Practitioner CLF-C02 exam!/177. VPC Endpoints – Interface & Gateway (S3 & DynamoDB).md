VPC Endpoints allow AWS services to be accessed **privately** from within a VPC **without using the public internet**.

Benefits:

- Higher security (no public exposure)
- Lower latency
- Data stays inside AWS private network

---

# **1. Two Types of VPC Endpoints**

## **A. Gateway Endpoint**

- **Only for 2 AWS services:**
    - **Amazon S3**
    - **DynamoDB**
- Added in the **Route Table** of subnets
- Traffic stays inside AWS network

### **When to use**

- Your EC2 in a **private subnet** needs to access:
    - S3 (e.g., read/write objects)
    - DynamoDB (e.g., queries, scans)
- Cheapest option (no per-hour charge)

### **Exam must-remember**

> Only S3 and DynamoDB support GETWAY endpoints.

This is one of the *top 10 most-tested networking questions* on CLF-C02.

---

## **B. Interface Endpoint (AWS PrivateLink)**

- Works for **most AWS services** (hundreds)
- Implements a network interface (ENI) inside your VPC
- Uses **PrivateLink**
- Requires Security Groups attached to the endpoint
- Paid per hour + data processing costs

### **When to use**

- Access ANY AWS service privately (except S3/DynamoDB):
    - CloudWatch
    - SNS
    - SQS
    - Secrets Manager
    - API Gateway private APIs
    - EC2, ECR, KMS… etc.

### Visual summary

Private Subnet EC2 → Interface Endpoint → AWS Service (privately)

---

# ⭐ **3. Gateway vs Interface — Quick Table**

| Feature | Gateway Endpoint | Interface Endpoint |
| --- | --- | --- |
| Services supported | **S3, DynamoDB ONLY** | Almost all AWS services |
| Cost | Free | Hourly + data charges |
| Implementation | Route table entry | ENI inside your subnet |
| Security Group | ❌ No | ✔ Required |
| Uses PrivateLink | ❌ No | ✔ Yes |
| Best for | Big data to S3/DynamoDB | Fine-grained private access to other AWS services |

---

# ⭐ **CLF-C02 Exam Tips**

### **1. Memorize this:**

> Gateway Endpoints = ONLY S3 + DynamoDB
> Everything else = **Interface Endpoint**

This shows up extremely frequently.

---

### **2. Private Access Needs**

💬 *“EC2 in a private subnet needs to access CloudWatch (or SQS, SNS, KMS, ECR…) without public internet.”*

✔ Use **Interface Endpoint**

💬 *“Private EC2 needs to access S3 without internet.”*

✔ Use **Gateway Endpoint (S3)**

---

### **3. Endpoint Reduces the Need for NAT Gateway**

If a private subnet uses:

- S3 Gateway Endpoint → no NAT needed for S3 traffic
- DynamoDB Gateway Endpoint → no NAT needed for DynamoDB traffic

This can save a lot of cost.

---

### **4. Mixed Support on S3 & DynamoDB**

S3 and DynamoDB technically support *both*:

- Interface Endpoint
- Gateway Endpoint

But the **exam wants you to identify Gateway Endpoint as the recommended + classic option.**

---

### **5. Typical Exam Questions**

> “Which VPC Endpoint type is used for S3 and DynamoDB?”
> ✔ Gateway Endpoint

> “Which VPC Endpoint type provides ENIs in your VPC?”
> ✔ Interface Endpoint

> “Your private EC2 needs access to SQS without internet. What do you use?”
> ✔ Interface Endpoint

> “How to reduce NAT Gateway traffic cost for S3 access?”
> ✔ S3 Gateway Endpoint
