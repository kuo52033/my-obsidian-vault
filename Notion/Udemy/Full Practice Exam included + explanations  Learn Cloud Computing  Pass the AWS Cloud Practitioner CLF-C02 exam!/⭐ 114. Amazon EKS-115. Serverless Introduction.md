## **What is EKS?**

Amazon **EKS (Elastic Kubernetes Service)** is a **managed Kubernetes service** on AWS.

Kubernetes = open-source system to **deploy, scale, and manage containerized applications** (usually Docker).

### **Why EKS?**

- Running Kubernetes yourself is difficult (installation, security, upgrades).
- EKS manages the **control plane**, making Kubernetes easier and more reliable.
- Kubernetes is **cloud-agnostic**, so workloads can move between:
    - AWS
    - Azure
    - GCP
    - On-premises

Exam keyword:

> If you see “Kubernetes” → answer is Amazon EKS.

---

## **How EKS Works**

EKS can run containers using:

- **EC2 instances (self-managed nodes)**
- **Fargate (serverless pods)**

### **Architecture Example**

```plain text
EKS Cluster
 ├─ Control Plane (managed by AWS)
 ├─ Worker Nodes:
 │     ├─ EC2 Node → runs Pods (containers)
 │     └─ Fargate Node → runs Pods serverlessly
 └─ Apps packaged as containers & pods


```

### **Key Benefits**

- Fully managed Kubernetes control plane
- Auto-scaling and self-healing features
- Run containers consistently across clouds & on-prem
- Works with ECR (for container images)

## **What is Serverless?**

A cloud model where **you do NOT manage servers**.

AWS handles:

- provisioning
- scaling
- patching
- fault tolerance

> “Serverless” ≠ “No servers”
> → Servers exist, but **you don’t manage them**.

---

## **Early Meaning vs Modern Meaning**

### **Old definition**

Serverless = **Function as a Service (FaaS)** → AWS Lambda

You upload **code**, AWS runs it on demand.

### **Modern meaning**

Any AWS service where **you don’t manage servers**, such as:

- **S3** (infinite storage, no servers)
- **DynamoDB** (serverless NoSQL DB)
- **API Gateway**
- **SNS / SQS**
- **Athena**
- **Fargate** (serverless containers)
- **Lambda** (serverless compute)

Serverless =

➡️ No provisioning

➡️ Automatic scaling

➡️ Pay-per-use

➡️ No patching

➡️ No OS management

---

## **Examples of Serverless in AWS**

| Service | Why it's Serverless |
| --- | --- |
| **S3** | Upload files, no servers to manage |
| **DynamoDB** | No DB servers; auto-scaling tables |
| **Fargate** | Run containers without EC2 nodes |
| **Lambda** | Run functions without managing compute |
| **Athena** | Query S3 with SQL, no infrastructure |

## **Why Serverless Matters**

- Reduces operational overhead
- Ultra-scalable automatically
- Cost-efficient (pay only for usage)
- Ideal for modern cloud-native architectures
