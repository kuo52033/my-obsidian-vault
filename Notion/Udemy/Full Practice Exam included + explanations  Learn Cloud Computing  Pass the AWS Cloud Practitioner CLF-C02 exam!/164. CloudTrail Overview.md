## **1. What is AWS CloudTrail?**

AWS CloudTrail is a service that provides:

- **Governance**
- **Compliance**
- **Operational auditing**
- **Security auditing**

CloudTrail records **every API call and event** that happens inside your AWS account.

CloudTrail is **enabled by default** for every AWS account.

---

## **2. What CloudTrail Records**

CloudTrail logs *who did what, when, and from where* in your AWS environment.

This includes:

- **Console actions** (AWS Management Console)
- **CLI commands**
- **SDK/API calls**
- **Service-to-service actions**
- **IAM user and IAM role activity**

Examples of CloudTrail logs:

- EC2 instance creation
- S3 bucket deletion
- IAM policy change
- Lambda invocation through AWS API

→ If something happens in AWS, **CloudTrail knows**.

---

## **3. Where CloudTrail Stores Logs**

CloudTrail stores events for **90 days** in its console.

For long-term retention, you must configure a **Trail**.

Trails can deliver logs to:

- **Amazon S3** (common for long-term archiving)
- **CloudWatch Logs** (for real-time analysis and monitoring)

---

## **4. Trails and Multi-Region Tracking**

### **Trail Options**

You can create a trail that:

- Applies to **one region**, or
- Applies to **all regions** (recommended for security)

Why Multi-Region?

- Malicious activity may occur in unused regions
- Ensures complete visibility across your AWS account

---

## **5. Why CloudTrail is Important**

CloudTrail allows you to answer questions such as:

- *“Who deleted this S3 bucket?”*
- *“Who stopped my EC2 instance?”*
- *“When was MFA disabled?”*
- *“What API call caused this security group change?”*

It gives a full **audit history** of AWS API activity.

---

## **6. Common Use Cases**

- Security forensics & incident investigation
- Compliance audits (HIPAA, PCI, SOC)
- Monitoring IAM user/role activity
- Troubleshooting infrastructure changes
- Tracking resource modifications

# ⭐ **CLF-C02 Exam Tips (Must Remember)**

### **1. CloudTrail vs CloudWatch**

| CloudTrail | CloudWatch |
| --- | --- |
| Logs **API calls** (who did what) | Logs **performance metrics** (CPU, network) |
| Audit, security, compliance | Monitoring and alarms |
| Default enabled | Must configure services to send metrics |
| Data stored 90 days | Metrics depend on service (5 min/1 min) |

> If question asks: “Which service tells you who did an action?” → CloudTrail

---

### **2. CloudTrail Is Enabled by Default**

Even without creating a trail, CloudTrail logs events for the **last 90 days**.

→ **Exam loves this fact.**

---

### **3. For Long-Term Storage → Create a Trail**

Trails send logs to:

- **S3** → long-term retention
- **CloudWatch Logs** → near real-time monitoring

→ Without a trail, you cannot keep logs beyond 90 days.

---

### **4. Multi-Region Trail Is Recommended**

If the exam refers to:

- Full visibility
- Detecting activity in all regions
- Security best practices

Choose:

✔ **Enable Multi-Region Trail**

---

### **5. CloudTrail Records:**

CloudTrail logs:

- Console logins
- API calls
- IAM activity
- AWS service actions
- Changes to resources

CloudTrail **does NOT** monitor:

- System metrics (that’s CloudWatch)
- Application logs (that’s CloudWatch Logs)
