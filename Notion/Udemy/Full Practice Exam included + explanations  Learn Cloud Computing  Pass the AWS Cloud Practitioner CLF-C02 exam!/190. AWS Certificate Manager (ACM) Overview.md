AWS Certificate Manager (ACM) simplifies **provisioning, managing, and deploying** SSL/TLS certificates for secure HTTPS connections.

---

# **1. What ACM Does**

ACM manages certificates used for **in-flight encryption**, ensuring secure communication between clients and AWS services.

### ACM handles:

- Generating certificates
- Validating domain ownership
- Automatic renewal
- Deploying certificates to supported AWS services

This eliminates the need for manual certificate management.

---

# **2. Where ACM Certificates Are Used**

ACM integrates with major AWS services, including:

- **Elastic Load Balancers (ALB/NLB/CLB)**
- **CloudFront**
- **API Gateway**
- **AppSync**
- **Elastic Beanstalk environments**

Use ACM whenever you need HTTPS endpoints.

---

# **3. How ACM Works — Example**

### Scenario:

User → HTTPS → Application Load Balancer → HTTP → EC2 ASG

### Flow:

1. ALB presents the **ACM TLS certificate** to the client
2. Client connects securely over HTTPS
3. ALB communicates with backend EC2 over HTTP (optional)

This provides **in-transit encryption** on the public-facing connection.

---

# **4. Certificate Types**

ACM supports:

- **Public certificates**
    - Free
    - Issued and renewed automatically
- **Private certificates**
    - Via ACM Private CA (paid add-on)
    - Used for internal corporate systems

---

# **5. Key Features**

### ✔ **Automatic renewal**

ACM renews certificates before expiration — extremely useful for production environments.

### ✔ **Free public certificates**

ACM is one of the easiest free ways to enable HTTPS globally.

### ✔ **Deep integration with AWS services**

Certificates can be deployed with a few clicks.
