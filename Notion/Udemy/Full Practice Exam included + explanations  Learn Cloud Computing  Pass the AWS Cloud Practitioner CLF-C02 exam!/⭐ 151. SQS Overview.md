## What is SQS?

Amazon SQS (Simple Queue Service) is a **fully managed message queue** used to **decouple** application components.

It allows producers to send messages into a queue, and consumers to read & process them asynchronously.

---

## How SQS Works

- **Producers** send messages to the queue.
- **Queue** stores unlimited messages (up to retention time).
- **Consumers** poll messages, process them, and **delete** them after successful processing.
- Enables components to operate **independently**, at different speeds.

---

## Key Characteristics

- **Fully managed & serverless** (no servers to provision).
- **Decouples applications** — major exam keyword.
- Scales automatically from **1 msg/sec → tens of thousands msg/sec**.
- Message retention:
    - **Default:** 4 days
    - **Max:** 14 days
- **Low latency** (<10 ms for publish & receive).
- Unlimited number of messages.
- Consumers can scale horizontally.

---

## Typical Architecture Use Case (Decoupling)

Example: Video processing workflow

- Web servers → send “video processing job” message → SQS
- A separate EC2 Auto Scaling Group reads jobs from SQS and processes them
- Each layer scales independently
- This improves:
    - reliability
    - performance
    - cost efficiency

---

## FIFO Queues

SQS FIFO (First-In-First-Out) queues provide:

- **Guaranteed message order**
- **Exactly-once processing**
- Useful when order matters (e.g., financial transactions, event sequencing)

Standard queues do **not** guarantee order.
