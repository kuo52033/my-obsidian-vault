## **What is Amazon Cognito?**

- **Amazon Cognito** provides **identity management** for **web and mobile application users**.
- Designed to support **millions of users**.
- Used for **end-users of applications**, **NOT** for AWS administrators.

> ❗ Do NOT create IAM users for app users
> IAM users are only for people who need to access AWS directly.

---

## **Why Cognito Exists**

- Web & mobile apps often have:
    - Millions of users
    - Sign-up / sign-in requirements
    - Social login (Google, Facebook, etc.)
- IAM **does not scale** for this use case.
- Cognito is built specifically for **application authentication & authorization**.

---

## **High-Level Architecture**

1. Users access your **Web or Mobile Application**
2. App integrates with **Amazon Cognito**
3. Cognito:
    - Manages user identities
    - Handles authentication
    - Issues tokens
4. App uses those tokens to access:
    - Backend services
    - AWS resources (optionally via IAM roles)

---

## **Key Features**

- **User sign-up & sign-in**
- **User database** (can store millions of users)
- **Authentication & authorization**
- **Federated identities**
    - Login with **Google, Facebook, Apple, Twitter**
- Secure token-based authentication

---

## **Federated Login (Social Login)**

- Users can log in using:
    - Google
    - Facebook
    - Twitter
    - Other identity providers
- Cognito handles the redirection and authentication flow.

---

## **What Cognito Is NOT**

- ❌ Not for AWS console access
- ❌ Not a replacement for IAM users
- ❌ Not for DevOps / Admin authentication

---

## **Typical Use Cases**

- Web applications
- Mobile applications
- Consumer-facing apps
- Apps with large user bases
- Apps requiring social login

---
