**Amazon Kendra** is a *fully managed, ML-powered enterprise search service*.

It indexes documents from many data sources and provides **natural language search** (Google-like search experience).

---

## **✨ What Kendra Does**

- Extracts answers from documents such as:
    - PDFs
    - Word / PowerPoint
    - HTML
    - FAQs
    - Plain text
- Builds an intelligent **search index** using ML

---

## **🔍 Search Capabilities**

- Users can ask questions in natural language
→ *“Where is the IT support desk?”* → **“1st floor.”**
- Kendra returns *precise answers*, not just keyword matches.
- Uses **incremental learning** to improve search relevance over time.

---

## **🎯 Use Cases**

- Enterprise search portals
- Knowledge base search
- Help desks / Employee onboarding portals
- Document-heavy industries (legal, healthcare, finance)

---

## **💡 Exam Tips**

- “Document search service” → **Kendra**
- Used when you need Q&A-style search from large repositories of documents.

# **211. Amazon Personalize — Overview**

**Amazon Personalize** is a fully managed ML service for **real-time personalized recommendations**.

It uses the same technology behind **Amazon.com** recommendations.

---

## **✨ What Personalize Does**

- Creates personalized product or content recommendations
- Real-time user-specific ranking
- Custom marketing (emails, SMS, app notifications)

---

## **🧩 How It Works**

1. Import interaction data (from S3 or API)
2. Personalize trains a recommendation model
3. Apps call Personalize API for real-time recommendations

---

## **🎯 Use Cases**

- Retail (product recommendations)
- Media / entertainment (movie or article suggestions)
- E-commerce personalization
- Marketing automation

---

## **💡 Exam Tips**

- “Recommendation engine” or “personalized suggestions” → **Amazon Personalize**
- You do **not** build ML models yourself — fully managed.

# **212. Amazon Textract — Overview**

**Amazon Textract** extracts **text, handwriting, forms, and tables** from scanned documents using ML.

It is more advanced than OCR because it understands **document structure**.

---

## **✨ What Textract Can Extract**

- Printed text
- Handwriting
- Key-value pairs in forms
- Tables and structured data
- Images, PDFs, scanned documents

---

## **📄 Typical Input Examples**

- Driver licenses
- Tax forms
- Invoices
- Medical records
- Claims and contracts

---

## **🎯 Use Cases**

- Financial services → invoice processing
- Healthcare → extracting patient data
- Government → ID verification, form processing
- Back-office automation

---

## **💡 Exam Tips**

- “Extract text/handwriting from scanned documents” → **Textract**
- For image labeling → Rekognition
- For NLP analysis → Comprehend
- For translation → Translate
- For speech → Transcribe/Polly
