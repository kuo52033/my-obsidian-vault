## **What is AWS Backup?**

- **Fully managed backup service**
- Centralized management of backups across AWS services
- Automates **backup creation**, **retention**, and **lifecycle**
- Eliminates the need to manage backups per service

---

## **Key Features**

- **Centralized backup management**
- **On-demand** and **scheduled backups**
- **Point-in-Time Recovery (PITR)**
- Backup **retention policies**
- **Lifecycle management** (move to cold storage, delete after X days)
- **Backup policies** applied at scale
- **Cross-Region backups**
- **Cross-Account backups** (via AWS Organizations)

---

## **How AWS Backup Works**

1. Create a **Backup Plan**
    - Backup frequency (hourly, daily, weekly, etc.)
    - Retention period
    - Lifecycle rules
2. Assign **resources** to the backup plan
3. AWS Backup automatically performs backups
4. Backups are stored in **Amazon S3 (AWS-managed backup vaults)**

---

## **Supported AWS Services**

AWS Backup supports:

- **Amazon EC2**
- **Amazon EBS**
- **Amazon RDS**
- **Amazon Aurora**
- **Amazon DynamoDB**
- **Amazon EFS**
- **Amazon FSx**
- **AWS Storage Gateway**

---

## **Security & Governance**

- Integrated with **AWS Organizations**
- Supports **cross-account backup policies**
- IAM-based access control
- Encrypted backups (KMS integration)

---

## **What AWS Backup Is NOT**

- ❌ Not manual snapshot management
- ❌ Not limited to a single AWS service
- ❌ Not region-locked (supports cross-region)

---

## **Common Exam Scenarios**

AWS Backup is the **correct answer** when you see:

- Centralized backup solution
- Automated backups across multiple AWS services
- Backup policies across accounts
- Cross-region or cross-account backups
- Point-in-time recovery requirement
