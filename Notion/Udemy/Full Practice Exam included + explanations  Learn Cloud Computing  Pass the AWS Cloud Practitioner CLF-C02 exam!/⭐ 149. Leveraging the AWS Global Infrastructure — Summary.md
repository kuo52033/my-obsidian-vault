## Overview

This section covers how AWS global features help you build **low-latency**, **high-availability**, and **globally distributed** applications.

Each service contributes differently to global performance and resilience.

---

## Route 53 — Global DNS

- AWS’s global, highly available **managed DNS** service.
- Routes users to the **closest** or **best-performing** endpoint.
- Supports multiple routing policies:
    - Simple
    - Weighted
    - Latency-based
    - Failover
- Essential for multi-region and disaster recovery strategies.

---

## CloudFront — Global Content Delivery Network (CDN)

- Caches content at **edge locations** worldwide.
- Reduces latency for static and dynamic content.
- Often used with **S3**, **ALB**, **EC2**, or **custom origins**.
- Provides additional protection via:
    - AWS Shield
    - AWS WAF

---

## S3 Transfer Acceleration

- Speeds up uploads/downloads to S3 buckets across long distances.
- Uses AWS edge locations + internal AWS backbone network.
- Good for global applications needing fast file transfers.

---

## AWS Global Accelerator

- Improves global application performance by routing traffic through AWS’s private backbone network.
- Uses **static Anycast IPs**.
- Optimizes TCP/UDP performance and offers deterministic failover.
- No caching (unlike CloudFront); focuses on network acceleration.

---

## AWS Outposts (Hybrid Cloud)

- Brings AWS hardware and services **into your on-premises data center**.
- Lets you run AWS services locally (EC2, EBS, RDS, EKS, etc.).
- Designed for:
    - ultra-low latency
    - data residency
    - on-prem + cloud hybrid models

---

## AWS Wavelength (5G Edge)

- Embeds AWS infrastructure directly inside **telecom 5G networks**.
- Ultra-low latency for mobile/edge use cases:
    - AR/VR
    - gaming
    - IoT
    - connected vehicles
- Traffic stays within the telecom network before reaching AWS.

---

## AWS Local Zones

- Extend your AWS Region into **specific metropolitan areas**.
- Provide compute, storage, and database closer to local users.
- Reduce latency for workloads sensitive to geographic distance.
- Example cities: Boston, Chicago, Dallas, Miami, etc. (US-focused for now)

---

## Big Picture Summary

| Service | Purpose | Key Benefit |
| --- | --- | --- |
| **Route 53** | Global DNS | Direct users to best endpoint |
| **CloudFront** | CDN | Cache content globally for speed |
| **S3 Transfer Acceleration** | Accelerated S3 upload/download | Faster transfers over long distances |
| **Global Accelerator** | Global network routing | Faster TCP/UDP performance + failover |
| **Outposts** | Hybrid AWS on-prem | Local AWS services with low latency |
| **Wavelength** | 5G edge computing | Ultra-low latency near mobile users |
| **Local Zones** | Local AWS compute/storage | Low latency in local metro areas |
