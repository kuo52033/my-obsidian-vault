## **1. What is CloudWatch Logs?**

CloudWatch Logs is a service used to **collect, store, monitor, and analyze log files** from various AWS and on-premise sources.

### **Why logs matter?**

Applications generate log messages such as:

- User actions
- Background jobs
- Errors or exceptions
- Cleanup tasks
- Internal process status

Logs help developers troubleshoot and understand system behavior.

---

## **2. Log Sources in AWS**

CloudWatch Logs can collect logs from many AWS services:

- **Elastic Beanstalk** – application logs
- **ECS** – container logs
- **Lambda** – function logs automatically sent
- **CloudTrail** – audit event logs
- **Route 53** – DNS query logs
- **CloudWatch Logs Agent** – installed on:
    - **EC2 instances**
    - **On-premises servers**

Using the log agent allows hybrid log collection from any server.

---

## **3. Log Retention**

- Logs do **not need to be stored forever**
- You can set retention policies such as:
    - 1 day
    - 1 week
    - 30 days
    - 1 year
    - Never expire

Retention settings help manage storage cost.

---

## **4. How CloudWatch Logs Works on EC2**

### **Important**

EC2 **does NOT send logs to CloudWatch Logs by default**.

To enable logging:

1. Install the **CloudWatch Logs Agent** or the **CloudWatch Unified Agent** on the EC2 instance
2. Configure which log files to push
3. Ensure the EC2 instance has an **IAM Role** with permissions:
    - `logs:CreateLogGroup`
    - `logs:CreateLogStream`
    - `logs:PutLogEvents`

### **Architecture**

- Log agent → CloudWatch Logs service
- Works for **EC2** and **on-premises machines (hybrid)**

# ⭐ **CLF-C02 Exam Tips (Very Important)**

### **CloudWatch Logs Basics**

- Used to **centralize log collection** from AWS and on-premise sources
- Log retention is configurable
- CloudWatch Logs supports **real-time log monitoring**

### **Log Delivery**

- **EC2 does not send logs automatically** → needs CloudWatch Logs Agent
- **Lambda logs are automatically sent to CloudWatch Logs without agent**

### **IAM Requirement**

- EC2 needs IAM Role permissions to push logs
- Very common exam question:
> “Your EC2 instance cannot push logs to CloudWatch Logs. What is missing?”
> ✔ **Instance IAM role with the correct CloudWatch Logs permissions**

### **CloudTrail vs CloudWatch Logs**

- CloudTrail = records API activity
- CloudWatch Logs = stores the logs and lets you analyze/search them
- CloudTrail logs can be delivered **to CloudWatch Logs** for deeper analysis
→ test point for CLF-C02

### **Hybrid Logging**

- CloudWatch Logs Agent can run on **on-premise** servers
- Very common exam keyword: *hybrid monitoring* → CloudWatch Logs is involved
