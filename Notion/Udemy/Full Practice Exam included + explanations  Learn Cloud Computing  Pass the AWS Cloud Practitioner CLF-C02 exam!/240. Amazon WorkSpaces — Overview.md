## **What is Amazon WorkSpaces?**

- **Managed Desktop as a Service (DaaS)**
- Provides **Windows or Linux virtual desktops** in the cloud
- Fully managed by AWS

---

## **Main Purpose**

- Replace **on-premises VDI (Virtual Desktop Infrastructure)**
- Allow users to access a **secure cloud desktop** from anywhere

---

## **Key Characteristics**

- **Fully managed** (no desktop infrastructure to maintain)
- **Fast to provision** and **scales to thousands of users**
- **Secure** (integrated with AWS KMS encryption)
- **Pay-as-you-go pricing** (pay only for usage)

---

## **How It Works (High Level)**

1. Admin provisions a WorkSpace (Windows/Linux)
2. User connects from any device (home, office, laptop)
3. User accesses:
    - Corporate applications
    - Cloud resources
    - Internal data
4. Desktop runs securely inside AWS (usually in a VPC)

---

## **Common Use Cases**

- Remote work environments
- Secure corporate desktops
- Temporary or contract workers
- Companies eliminating on-prem VDI

---

## **Latency & Best Practices (Exam Tip)**

- **Deploy WorkSpaces close to end users**
- Example:
    - US users → WorkSpaces in US region
    - EU users → WorkSpaces in Europe region

> 🔑 Rule of thumb: Minimize latency by deploying close to users

---

## **When to Use Amazon WorkSpaces**

Use WorkSpaces when you need:

- Virtual desktops in the cloud
- Managed desktop infrastructure
- Secure access to corporate environments
- No physical laptops or on-prem VDI

---

## **What WorkSpaces Is NOT**

- ❌ Not application streaming (that’s **AppStream 2.0**)
- ❌ Not server hosting
- ❌ Not a laptop replacement (it’s a virtual desktop)
