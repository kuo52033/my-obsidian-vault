AWS Secrets Manager is used to **securely store, retrieve, and automatically rotate secrets** such as:

- Database credentials
- API keys
- Application passwords
- OAuth tokens

---

# **1. What Secrets Manager Does**

### ✔ Secure Secret Storage

All secrets are **encrypted with KMS keys** (automatic and mandatory).

### ✔ Automatic Rotation

Secrets Manager can **automatically rotate** credentials on a schedule (e.g., every 30 or 90 days).

This is one of its primary advantages over alternatives.

### ✔ Automated Credential Generation

Can use **AWS Lambda** to generate and rotate secrets dynamically.

### ✔ Tight Integration with RDS

Secrets Manager can automatically:

- Create RDS passwords
- Rotate DB credentials without downtime

This makes it extremely valuable for database authentication.

---

# **2. When to Use Secrets Manager**

Use it when you need any of these:

- Securely store application secrets
- Automatically rotate secrets
- Rotate **RDS credentials**
- Enforce compliance policies (password aging, auditability)
- Simplify secret access from Lambda, EC2, ECS, EKS, etc.

---

# **3. Key Features**

| Feature | Description |
| --- | --- |
| **KMS encryption** | All stored secrets are encrypted with KMS |
| **Auto rotation** | Rotate DB passwords, API keys, etc. |
| **Lambda integration** | Custom rotation logic |
| **RDS integration** | One-click rotation for RDS MySQL, PostgreSQL, etc. |
| **Fine-grained IAM access** | Control which resource can read which secret |

---

# **4. Cost**

Secrets Manager is **not free** — billed per secret and per 10K API calls

(important because SSM Parameter Store is cheaper).

# **Secrets Manager vs Parameter Store (High-Level)**

(This comparison appears often on CLF exams.)

| Feature | Secrets Manager | SSM Parameter Store |
| --- | --- | --- |
| Auto rotation | ✔ Yes | ❌ No |
| Cost | Paid | Free (standard) |
| KMS encryption | ✔ Yes | ✔ Optional |
| Intended use | Sensitive secrets | Config parameters / simple strings |
