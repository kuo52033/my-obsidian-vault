The **root user** is the original account owner in AWS.

It has **full, unrestricted access** to all AWS services and resources.

⚠️ **Never use the root user for daily tasks.**

Create an admin IAM user instead.

---

# **1. When Should You Use the Root User?**

Most AWS tasks can be done by IAM users — **except a few special actions**

that *only the root user* can perform.

These are frequently tested in CLF-C02.

---

# ⭐ **Root-Only Actions (Must Memorize)**

### **Identity & Account Management**

- Change **account settings**
(account name, root email, root password)
- Change or delete **root access keys**
- **Close** the AWS account
- **Restore IAM user permissions** (if mistakenly removed)
- Sign up for **AWS GovCloud**

---

### **Billing & Support**

- View **certain tax invoices**
- Change or cancel the **AWS Support Plan**
(e.g., upgrade to Business/Enterprise, downgrade, cancel)

---

### **Marketplace & Reserved Instances**

- **Register as a seller** on the Reserved Instance Marketplace
(required to sell unused Reserved Instances)

---

### **S3-Specific Root Privileges**

- **Enable MFA Delete** on an S3 bucket
(Only the root user can enable/disable it)
- Edit/delete an S3 bucket policy that has:
    - an *invalid VPC ID*
    - an *invalid VPC endpoint ID*

---

# **2. Exam Tips (CLF-C02)**

### **Q: Who can close an AWS account?**

✔ Root user only

### **Q: Who can change the support plan?**

✔ Root user only

### **Q: Who can enable MFA Delete on an S3 bucket?**

✔ Root user only

### **Q: Who can sell Reserved Instances on the marketplace?**

✔ Root user only

### **Q: Should you use the root user for everyday admin tasks?**

❌ No → create an IAM admin user

---

# **3. Best Practices**

- Lock the root credentials in a secure place
- Enable **MFA** on the root account
- Never create root access keys unless absolutely necessary
- Use IAM roles/users for daily work
