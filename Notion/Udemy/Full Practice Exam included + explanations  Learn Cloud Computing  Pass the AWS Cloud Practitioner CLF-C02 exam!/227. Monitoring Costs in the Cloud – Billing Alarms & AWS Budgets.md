## **1. Billing Alarms (CloudWatch Billing Metric)**

### **Key Facts**

- Billing metrics exist **only in us-east-1 (N. Virginia)**.
- Metric represents **actual cost** (not forecasted cost).
- Aggregates cost **across all regions**.
- You can create a CloudWatch alarm to notify you via SNS when cost exceeds a threshold.

### **Use Cases**

- Simple cost monitoring
- Get an email when your monthly bill exceeds a certain amount

### **Limitations**

- Cannot trigger advanced automation
- No forecast-based alerts
- Only gives **actual cost** data, not predicted spending

---

## **2. AWS Budgets**

A more powerful and flexible way to track and control costs.

### **What Budgets Can Monitor**

1. **Cost Budgets** (most common)
2. **Usage Budgets** (GB-hours, request count, etc.)
3. **Savings Plans Budgets**
4. **Reservation (RI) Budgets**
    - Tracks RI **utilization** and **coverage**
    - Supports EC2, RDS, ElastiCache, Redshift

---

## **3. Why AWS Budgets Is Stronger than Billing Alarms**

| Billing Alarm | AWS Budgets |
| --- | --- |
| Actual cost only | Actual + **Forecasted cost** |
| Single alarm | Up to **5 SNS notifications** |
| Basic | Filters by service, region, tags, usage type, AZ, API operation, account, etc. |
| Cannot automate | Can trigger **SNS**, Lambda workflows |
| No forecast | Forecast up to end of period |

---

## **4. Budget Notifications**

A budget can alert when:

- **Actual cost** exceeds a threshold (e.g., 80% of budget)
- **Forecasted cost** is expected to exceed budget
→ *This is a common exam keyword: forecast alert.*

### **Supported Notification Methods**

- SNS email
- SNS → Lambda automation (e.g., shutdown resources)
- Up to **5 notifications per budget**

---

## **5. Budget Configuration Options**

### **Period Types**

- Monthly
- Quarterly
- Annually
- Daily
- Recurring or fixed date range

### **Filtering (very important for exam)**

You can filter budgets by:

- Service (e.g., EC2 only)
- Region
- Availability Zone
- Linked Account (Org)
- Usage type
- Purchase option (On-Demand / RI / SP)
- Tags

*Budget filtering works similarly to Cost Explorer.*

---

## **6. Example Workflow**

5. Create a **Cost Budget** with a $10 limit
6. Filter only **EC2 + KMS** cost
7. Set alerts:
    - **80% Actual cost** → send email
    - **80% Forecasted cost** → send email
8. Optionally trigger Lambda via SNS for automated remediation

---

## **7. Exam Tips**

- Billing alarms are **simple** and use CloudWatch metrics in **us-east-1** only.
- AWS Budgets is needed when:
    - You want **forecast** alerts
    - You need more than one notification
    - You need filtering based on tags / service / region
    - You want to take automated actions via SNS/Lambda
- Budgets can monitor **RI utilization** and **Savings Plans** usage.
