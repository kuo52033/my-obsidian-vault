## **What is AWS Infrastructure Composer?**

- **Visual tool** to design AWS infrastructure
- Focused on **serverless applications**
- Uses **drag-and-drop** interface
- Automatically generates **Infrastructure as Code (IaC)**

---

## **Key Capabilities**

- Visual design of serverless architectures
- No deep CloudFormation knowledge required
- Define how AWS services interact with each other
- Generates **CloudFormation-compatible templates**
- Supports **AWS SAM templates**

---

## **Bidirectional Workflow**

Infrastructure Composer supports:

- **Design → Code**
    - Drag & drop resources
    - Auto-generate CloudFormation / SAM templates
- **Code → Visual**
    - Import existing CloudFormation or SAM templates
    - Visualize architecture graphically

---

## **Main Use Cases**

- Rapid serverless architecture design
- Learning AWS infrastructure visually
- Prototyping applications quickly
- Understanding existing CloudFormation stacks

---

## **What It Is NOT**

- ❌ Not a deployment service by itself
- ❌ Not a replacement for CloudFormation
- ❌ Not limited to runtime execution

(It **generates** IaC; deployment still uses CloudFormation)

---

## **Exam Keywords → Correct Answer**

If the question mentions:

- Visual infrastructure design
- Drag-and-drop AWS architecture
- Generate CloudFormation templates
- Visualize existing IaC

👉 **Answer: AWS Infrastructure Composer**

# **246. AWS Device Farm — Overview**

## **What is AWS Device Farm?**

- **Fully managed application testing service**
- Tests **web & mobile apps** on **real devices**
- Supports:
    - Desktop browsers
    - Mobile phones
    - Tablets

---

## **Key Features**

- Uses **real physical devices** (not emulators)
- Run tests **in parallel** across many devices
- Custom device configurations:
    - GPS
    - Language
    - Wi-Fi
    - Bluetooth
- Interactive testing available
- Provides:
    - Logs
    - Screenshots
    - Videos
    - Test reports

---

## **Supported Applications**

- Android apps
- iOS apps
- Web applications
- Cross-platform apps

---

## **Main Use Cases**

- Cross-device compatibility testing
- UI/UX testing
- Regression testing
- Catch device-specific bugs early

---

## **What Device Farm Is NOT**

- ❌ Not a CI/CD tool (though it integrates with them)
- ❌ Not an emulator-only testing service
- ❌ Not used to host applications

---

## **Exam Keywords → Correct Answer**

If the question mentions:

- Testing mobile apps on many devices
- Real phones and tablets
- Browser compatibility testing
- Automated or interactive testing

👉 **Answer: AWS Device Farm**

---
