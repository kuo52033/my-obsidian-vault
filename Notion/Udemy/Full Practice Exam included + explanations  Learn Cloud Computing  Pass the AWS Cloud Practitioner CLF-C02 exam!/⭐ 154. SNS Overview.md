## What is SNS?

Amazon SNS (Simple Notification Service) is a **Pub/Sub (publish–subscribe)** messaging service used to **send one message to many subscribers**.

當你看到以下關鍵字時 → **SNS**：

- *Notification*
- *Publish/Subscribe*
- *One-to-many messaging*
- *Fan-out*

---

## Why SNS?

Without SNS, your service must build **direct integrations** to every receiver:

```plain text
Buying Service → Email Service
Buying Service → Fraud Service
Buying Service → Shipping Service
Buying Service → SQS


```

This creates tight coupling and duplicate effort.

---

## SNS Pub/Sub Pattern

SNS simplifies everything:

```plain text
Buying Service → SNS Topic → (Email, Fraud, Shipping, SQS...)


```

- Publisher sends a message **once**.
- SNS automatically delivers it to **all** subscribers.
- Subscribers do **not** share messages; each gets a full copy
(unlike SQS where messages are load-balanced).

---

## Key Features

### ✔ Massive Fan-out

- 12 million subscriptions per topic
- 100,000 topics per account (soft limit)

### ✔ Multi-protocol Delivery

SNS can deliver to:

- **Amazon SQS**
- **AWS Lambda**
- **Kinesis Data Firehose**
- **Email**
- **SMS / Mobile push**
- **HTTP/HTTPS endpoints**

### ✔ Fully managed & serverless

No servers, auto-scaling, highly reliable.

---

## SNS vs SQS

| Feature | SNS | SQS |
| --- | --- | --- |
| Pattern | Pub/Sub (1 → many) | Queue (1 → 1 per message) |
| Delivery | Every subscriber gets **all** messages | Consumers **share** messages |
| Use case | Notifications, fan-out | Decoupling workers, async jobs |
| Ordering | No strict ordering | FIFO option available |

---

## Exam Tips

- “Notify multiple systems at once” → **SNS**
- “Fan-out to SQS queues + Lambda + email” → **SNS**
- “Pub/Sub model” → **SNS**
- “One message → many receivers” → **SNS**
