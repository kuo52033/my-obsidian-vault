Amazon Detective is a **security investigation service** that helps you **find the root cause** of suspicious activities and security findings.

---

# **1. Purpose of Amazon Detective**

Services like:

- GuardDuty (threat detection)
- Macie (sensitive data detection)
- Security Hub (security findings)

…can tell you **there is a problem**,

but they **don’t tell you why it happened**.

**Amazon Detective = Finds the root cause**

It provides deep investigation tools to analyze security events.

---

# **2. How Detective Works**

Detective automatically **collects and correlates data** from:

- **VPC Flow Logs**
- **CloudTrail logs**
- **GuardDuty findings**

Then it builds a **graph-based model** to link events across users, IPs, resources, and timelines.

---

# **3. What Detective Provides**

- Visual timelines of suspicious activity
- Detailed resource interaction graphs
- Easy navigation through related events
- Faster investigation workflow for security analysts

It transforms raw logs → visual, connected insights.

---

# ⭐ **CLF-C02 Exam Tips — Amazon Detective**

### **1. Need to investigate a security incident root cause?**

✔ Amazon Detective

### **2. Need visualization of CloudTrail / GuardDuty / VPC flow logs?**

✔ Detective

### **3. Does Detective detect threats?**

❌ No — GuardDuty detects threats

✔ Detective **investigates** after detection

### **4. Does Detective require installing agents?**

❌ No — automatically processes logs

---

# **Detective vs Security Hub vs GuardDuty (Quick Overview)**

| Service | Purpose |
| --- | --- |
| **GuardDuty** | Detect threats & anomalies |
| **Security Hub** | Aggregate + centralize findings |
| **Detective** | Investigate & find root cause |

# **199. AWS Abuse Overview**

AWS Abuse is the team you contact when you suspect **illegal, abusive, or malicious activity** originating from AWS resources.

---

# **1. What Counts as AWS Abuse**

Examples of prohibited activities:

- **Spam** emails coming from AWS IPs
- **Port scanning** from EC2 instances
- **DDoS attacks** sourcing from AWS
- **Hacking attempts / intrusion attempts**
- Hosting illegal, copyrighted, or harmful content
- **Malware distribution**

These violate the AWS Acceptable Use Policy.

---

# **2. What To Do If You Detect Abuse**

If you detect suspicious activity:

📌 **Report it to the AWS Abuse Team**

Two methods:

- Online abuse report form
- Email: **abuse@amazonaws.com**

Include:

- Source IP
- Logs
- Time of incident
- Description of suspicious activity

---

# ⭐ **CLF-C02 Exam Tips — AWS Abuse**

### **1. Someone is performing port scans or DDoS from AWS IPs. What do you do?**

✔ Contact **AWS Abuse Team**

### **2. Can you solve it by shutting down the instance?**

❌ No — you don't control other users’ resources

### **3. What is not allowed on AWS?**

✔ Hosting malware

✔ Intrusion attempts

✔ Spam or attack traffic
