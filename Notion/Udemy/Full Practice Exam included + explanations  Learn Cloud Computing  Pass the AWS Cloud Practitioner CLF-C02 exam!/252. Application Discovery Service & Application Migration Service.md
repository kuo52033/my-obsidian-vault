When migrating **from on-premises to AWS**, there are **two key phases**:

1. **Discover & Plan** → *What do I have? How are systems connected?*
2. **Migrate** → *Move workloads to AWS*

AWS provides **two core services** for this process.

---

## **1. AWS Application Discovery Service (ADS)**

### **Purpose: Plan the migration**

### **What it does**

- Discovers **on-premises servers**
- Collects **utilization data** and **dependency mappings**
- Helps you decide:
    - What to migrate
    - In which order
    - With minimal risk

### **Key Outputs**

- CPU, memory, disk usage
- Running processes
- Network connections
- Application dependencies

---

### **Discovery Methods**

### **1) Agentless Discovery**

- Uses a **Discovery Connector**
- No agent installed inside servers

**Collects**

- VM configuration
- Performance history (CPU, memory, disk)

**Best for**

- Quick, low-impact discovery
- Virtualized environments

---

### **2) Agent-Based Discovery**

- Install **Application Discovery Agent** on servers

**Collects (more detailed)**

- OS & system configuration
- Running processes
- Network connections (dependency mapping)

**Best for**

- Complex environments
- Detailed application dependency analysis

---

### **Where the data goes**

- Results are viewed in **AWS Migration Hub**
- Migration Hub provides:
    - Central visibility
    - Migration progress tracking

---

### **Exam Clue**

> “Need to understand server dependencies before migration”
> ➡️ **Application Discovery Service**

---

## **2. AWS Application Migration Service (MGN)**

*(formerly CloudEndure Migration)*

### **Purpose: Execute the migration**

### **What it does**

- Performs **Rehost (Lift-and-Shift)** migrations
- Converts physical, virtual, or cloud servers
→ **Native AWS EC2 instances**

---

### **How it works**

3. Install **Replication Agent** on source servers
4. Continuous **block-level disk replication**
5. Data replicated to:
    - Low-cost EC2
    - EBS volumes (staging area)
6. When ready → **Cutover**
7. Launch **production-sized EC2 instances**

---

### **Key Benefits**

- Minimal downtime
- Automated migration
- No application changes
- Reduced migration cost & complexity

---

### **Supports**

- Physical servers
- Virtual machines
- Other cloud platforms
- Multiple OS and databases

---

### **Exam Clue**

> “Lift-and-shift on-prem servers to AWS with minimal downtime”
> ➡️ **Application Migration Service (MGN)**

---

## **ADS vs MGN – Exam Comparison Table**

| Service | Purpose | Phase |
| --- | --- | --- |
| **Application Discovery Service** | Discover servers & dependencies | Planning |
| **Application Migration Service (MGN)** | Lift-and-shift workloads | Migration |

---

## **Typical Exam Flow Question**

> “Company wants to migrate on-prem servers to AWS. They first need to understand dependencies, then move servers with minimal downtime.”

**Correct Answer**

8. **Application Discovery Service**
9. **Application Migration Service (MGN)**

- **Discover first → ADS**
- **Move servers → MGN**
