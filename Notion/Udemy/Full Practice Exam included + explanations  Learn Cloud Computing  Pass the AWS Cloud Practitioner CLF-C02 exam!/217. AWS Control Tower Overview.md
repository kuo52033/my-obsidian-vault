## **1. What is AWS Control Tower?**

AWS Control Tower is the **easiest, fastest**, and **best-practice-aligned** way to set up and govern a **secure, compliant multi-account AWS environment**.

它是建在 Organizations、SCP、Config、CloudTrail 等服務之上的「自動化 Landing Zone」。

---

## **2. What Control Tower Provides**

### **✔ Automated Multi-Account Setup**

- One-click creation of a secure baseline environment
- Automatically sets up:
    - **AWS Organizations**
    - **Organizational Units (OUs)**
    - **Centralized logging**
    - **Baseline IAM roles**
    - **Network configuration**

### **✔ Guardrails (High-Yield Exam Concept)**

Guardrails are **prebuilt governance rules** that help enforce best practices.

Two types:

| Guardrail Type | Function |
| --- | --- |
| **Preventive** | Uses **SCPs** to *block* prohibited actions (e.g., disallow public S3 buckets) |
| **Detective** | Uses **AWS Config rules** to *identify* violations |

Control Tower applies these guardrails automatically.

### **✔ Continuous Compliance**

- Detects policy violations
- Can auto-remediate some issues
- Provides compliance visibility through a dashboard

### **✔ Lifecycle Management**

- Easily provision new accounts following the same compliant baseline
(via Account Factory)

---

## **3. How Control Tower Works (Simplified)**

1. **Control Tower enabled** → It automatically:
    - Creates an AWS Organization
    - Creates core OUs (e.g., Security, Sandbox)
    - Sets up mandatory security logging (CloudTrail, Config)
2. **Guardrails applied**
    - SCPs for preventive governance
    - Config rules for detective compliance
3. **Account Factory**
    - Users can create new accounts that comply with governance standards

---

## **4. Exam Tips（高機率出題）**

- Control Tower = fastest way to set up **secure, compliant multi-account architecture**.
- Built **on top of AWS Organizations**.
- Uses **SCPs + AWS Config** to enforce guardrails.
- Provides **central dashboard** for governance.
- Great for enterprises starting AWS at scale.
