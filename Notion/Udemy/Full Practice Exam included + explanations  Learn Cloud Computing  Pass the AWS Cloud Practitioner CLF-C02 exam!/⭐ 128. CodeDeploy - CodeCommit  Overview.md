## What is CodeDeploy?

AWS CodeDeploy is a **deployment automation service**

that lets you update applications **on EC2 instances or on-premises servers**.

It supports:

- EC2 instances
- On-premises servers (hybrid)
- Lambda functions (not in CCP exam)

Its main purpose is **automated, repeatable, controlled deployments**.

---

## Why use CodeDeploy?

Because it helps you move from **V1 → V2** of your app *without manual updates*.

### Key Features

- Works on **any server you provision** (EC2 or on-prem)
- Handles deployment logic automatically
- Supports various deployment strategies (rolling, blue/green, etc.)
- Requires installing the **CodeDeploy Agent** on your servers
- Not tied to Beanstalk or CloudFormation — completely independent

### Hybrid Deployment

Since CodeDeploy works on on-prem servers,

it’s great for companies migrating gradually to AWS.

---

## How it works

1. You prepare your application package
2. Your servers (EC2/on-prem) run the **CodeDeploy agent**
3. CodeDeploy orchestrates upgrading from V1 → V2
4. It ensures each server updates safely and consistently

# 130. CodeCommit Overview

## What is CodeCommit?

AWS CodeCommit is a **managed Git repository service**

similar to GitHub or GitLab.

It lets teams store code **privately inside AWS**.

---

## Why use CodeCommit?

- Fully-managed & scalable
- Encrypted in transit & at rest
- Integrates with IAM for fine-grained permissions
- Integrated easily with AWS dev tools (CodeBuild, CodePipeline, CodeDeploy)
- Ideal for teams that already operate inside AWS and want private repos

---

## Benefits

- Works with standard Git CLI
- Unlimited repos
- Version control + collaboration
- High availability (stored across multiple AZs)
