A DDoS (Distributed Denial of Service) attack overwhelms your application with massive traffic, making it unreachable.

AWS provides multiple services to protect against these attacks.

---

# **1. AWS Shield**

## **Shield Standard**

- **Enabled automatically** for all AWS customers
- **Free**
- Protects against common Layer 3 & 4 attacks:
    - SYN Floods
    - UDP Reflection
    - Other volumetric attacks

### **Exam keyword**

> “Free default DDoS protection” → Shield Standard

---

## **Shield Advanced**

- Paid service (~$3000/month per org)
- Protection for:
    - EC2
    - Elastic Load Balancer (ALB/NLB)
    - CloudFront
    - Route 53
    - Global Accelerator
- Includes:
    - 24/7 **AWS DDoS Response Team (DRT)**
    - Real-time attack visibility
    - Cost protection (AWS covers scaling costs during DDoS)

### **Exam keyword**

> “Need expert support / advanced DDoS protection” → Shield Advanced

---

# **2. AWS WAF (Web Application Firewall)**

Protects **Layer 7 (HTTP/HTTPS)** applications from web exploits.

### **Supported Services**

- Application Load Balancer (ALB)
- CloudFront
- API Gateway (not needed in exam, but real)

### **WAF Features**

You create **Web ACLs** with rules such as:

- Block/allow based on IP
- Header inspection
- Body inspection
- Pattern matching (SQL injection, XSS)
- Request size limits
- Geo-blocking (block by country)
- **Rate-based rules** (e.g., block > 1000 req/min)

### **Exam keyword**

> “Protect against SQL injection or XSS” → AWS WAF
> “Layer 7 protection” → **AWS WAF**

---

# **3. CloudFront & Route 53**

Help mitigate DDoS via AWS global edge locations.

- Route 53:
    - Highly available DNS
    - Protected by Shield
- CloudFront:
    - Absorbs traffic at global edge
    - Reduces load on your origin servers

> Best practice: put CloudFront in front of your ALB/EC2.

---

# **4. Auto Scaling**

Even with protection, scaling capacity helps absorb unexpected spikes.

> “Absorb traffic surges during DDoS” → Auto Scaling

---

# **5. DDoS Protection Reference Architecture**

User → **Route 53** (Shield protected)

→ **CloudFront** (Shield protected, caching)

→ **AWS WAF** (Layer 7 filtering)

→ **Load Balancer (ALB/NLB)**

→ **EC2 Auto Scaling Group**

This layered model provides defense-in-depth.

# ⭐ **CLF-C02 Exam Tips**

### **1. Shield Standard vs Shield Advanced**

| Feature | Standard | Advanced |
| --- | --- | --- |
| Cost | Free | ~$3000/mo |
| Coverage | Common L3/L4 attacks | Sophisticated attacks |
| Services Protected | Automatically on AWS | Specific services (EC2, ELB, CF, R53) |
| DDoS Response Team | ❌ No | ✔ Yes |
| Cost Protection | ❌ No | ✔ Yes |

---

### **2. WAF Uses**

- SQL injection / XSS filtering → **WAF**
- Rate limiting → **WAF**
- Geo-blocking → **WAF**
- HTTP layer protection → **WAF**

---

### **3. Shield Uses**

- DNS-level DDoS protection → **Route 53 + Shield**
- Global edge mitigation → **CloudFront + Shield**

---
