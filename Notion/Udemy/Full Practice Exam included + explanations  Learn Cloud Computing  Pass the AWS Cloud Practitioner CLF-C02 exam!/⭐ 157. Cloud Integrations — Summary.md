## 📨 Amazon SQS (Simple Queue Service)

**Purpose:** Queueing & application decoupling

- Supports **multiple producers** sending messages into a queue
- Messages persist **up to 14 days**, then expire
- **Consumers share the workload** (each consumes different messages)
- After a consumer processes a message → **the message is deleted**
- Ideal for:
    - Decoupling microservices
    - Asynchronous processing
    - Worker-based architectures

**Exam keywords:**

➡ *decouple*, *queue*, *workers*, *polling* → **Think SQS**

---

## 📢 Amazon SNS (Simple Notification Service)

**Purpose:** Publish/Subscribe notifications

- Producer publishes to **one SNS topic**
- SNS pushes the message to **all subscribers**
    - Email
    - Lambda
    - SQS queue
    - HTTPS endpoint
    - Mobile push
- **No message retention** (not a durable store)
- Every subscriber receives **all messages**

**Exam keywords:**

➡ *fan-out*, *notifications*, *publish-subscribe*, *multiple subscribers* → **Think SNS**

## ⚡ Amazon Kinesis

**Purpose:** Real-time streaming data

- Designed for **real-time ingestion** of large, fast data streams
- Has **data persistence** (retention hours to days)
- Can run **real-time analytics**
- Common data sources:
    - Logs
    - Clickstreams
    - IoT device streams
    - Metrics

**Exam keywords:**

➡ *real-time*, *streaming*, *analytics* → **Think Kinesis Data Streams**

---

## 🏛 Amazon MQ

**Purpose:** Managed message broker for legacy protocols

Supports open protocols used on-premises:

- **MQTT**, **AMQP**, **STOMP**, **OpenWire**, **WSS**

Based on:

- **ActiveMQ**
- **RabbitMQ**

Used when migrating traditional systems to AWS **without rewriting to SQS/SNS APIs**.

**Limitations:**

- Broker-based → **does not scale like SQS/SNS**
- Requires multi-AZ for high availability

**Exam keywords:**

➡ *legacy migration*, *MQTT/AMQP/STOMP/OpenWire*, *ActiveMQ*, *RabbitMQ* → **Think Amazon MQ**

## 🧠 Quick Comparison Table

| Service | Model | Message Retention | Scaling | Best For |
| --- | --- | --- | --- | --- |
| **SQS** | Queue | Yes (up to 14 days) | Very high | Decoupling workers |
| **SNS** | Pub/Sub | No | Very high | Fan-out notifications |
| **Kinesis** | Real-time stream | Yes | High | Real-time analytics & events |
| **Amazon MQ** | Broker (queue + topic) | Yes | Limited | Legacy protocol migration |
