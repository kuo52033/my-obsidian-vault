AWS defines **7 common cloud migration strategies**, known as the **7Rs**.

From an exam perspective, you must understand **what each R means** and **when to use it**.

---

## **1. Retire**

### **What it means**

- **Shut down** applications or services that are no longer needed

### **Why use it**

- Reduce attack surface
- Save costs (often **10–20%** savings)
- Focus effort on critical systems only

### **Exam clue**

> “Application is no longer needed”
> ➡️ **Retire**

---

## **2. Retain**

### **What it means**

- **Keep the application on-premises**
- No migration to the cloud

### **Common reasons**

- Security or compliance requirements
- Performance constraints
- Unresolved dependencies
- No business value in migrating

### **Exam clue**

> “Cannot migrate due to compliance or dependencies”
> ➡️ **Retain**

---

## **3. Relocate**

### **What it means**

- Move workloads **without changing the platform**
- Typically used with **VMware environments**

### **Example**

- VMware on-prem → **VMware Cloud on AWS**
- EC2 moved to another VPC, account, or region

### **Key point**

- **No application changes**
- Infrastructure location changes only

### **Exam clue**

> “Move VMware workloads to AWS without refactoring”
> ➡️ **Relocate**

---

## **4. Rehost (Lift and Shift)**

### **What it means**

- Move applications **as-is** to AWS
- No code or architecture changes

### **Benefits**

- Fast migration
- Low risk
- Can save ~**30%** in infrastructure cost

### **Typical tools**

- **AWS Application Migration Service (MGN)**

### **Exam clue**

> “Fast migration, minimal changes”
> ➡️ **Rehost**

---

## **5. Replatform (Lift and Reshape)**

### **What it means**

- Make **small optimizations**
- Use **managed AWS services**
- Core architecture remains the same

### **Examples**

- Database → **Amazon RDS**
- App → **Elastic Beanstalk**

### **Benefits**

- Less operational overhead
- Built-in backups, HA, scaling

### **Exam clue**

> “Move database to RDS without changing the app”
> ➡️ **Replatform**

---

## **6. Repurchase (Drop and Shop)**

### **What it means**

- Replace the application with a **SaaS product**

### **Examples**

- CRM → **Salesforce**
- HR → **Workday**
- CMS → **Drupal SaaS**

### **Trade-off**

- Higher short-term cost
- Very fast deployment
- No infrastructure management

### **Exam clue**

> “Replace existing app with SaaS”
> ➡️ **Repurchase**

---

## **7. Refactor / Re-architect**

### **What it means**

- **Redesign the application** using cloud-native services

### **Why do it**

- Improve scalability
- Improve performance & security
- Increase agility
- Break monoliths into microservices

### **Examples**

- Move to **serverless**
- Use **S3, Lambda, DynamoDB**
- Microservices architecture

### **Cost & effort**

- Highest effort
- Highest long-term payoff

### **Exam clue**

> “Redesign application for cloud-native architecture”
> ➡️ **Refactor**

---

## **7Rs Summary Table (Exam Friendly)**

| Strategy | Key Idea |
| --- | --- |
| **Retire** | Shut it down |
| **Retain** | Keep on-prem |
| **Relocate** | Move VMware / location only |
| **Rehost** | Lift & Shift |
| **Replatform** | Lift & Reshape |
| **Repurchase** | Move to SaaS |
| **Refactor** | Cloud-native redesign |

---

## **One-Line Exam Memory Tip**

> Fastest → Rehost
> **Cheapest** → Retire
> 
> **Most Cloud-Native** → Refactor
> 
> **SaaS Replacement** → Repurchase
