## What is CloudFront?

- CloudFront is AWS’s **Content Delivery Network (CDN)**.
- Caches your content across **global edge locations** to reduce latency.
- Delivers faster read performance to users around the world.
- Provides **built-in DDoS protection** (via CloudFront + AWS Shield + WAF).

---

## How CloudFront Works (High-Level)

1. User sends a request to the nearest **edge location**.
2. If the content **exists in the edge cache**, it returns immediately (low latency).
3. If content is **not cached**, CloudFront fetches it from the **origin**:
    - S3 bucket
    - ALB / NLB
    - EC2 instance (private via VPC origin)
    - Any public HTTP server
4. The edge location **stores a cached copy** for future requests.

---

## CloudFront Origins

### **1. S3 Origin**

- Used for static files (HTML / CSS / JS / images / videos).
- Secured by **Origin Access Control (OAC)**.
- CloudFront → AWS private network → S3

### **2. VPC Origin**

- For servers running inside private subnets:
    - ALB, NLB
    - EC2 instances
- CloudFront connects **privately** to your VPC.

### **3. Custom HTTP Origin**

- Any public HTTP endpoint:
    - S3 website hosting (static website mode)
    - On-premises servers
    - Non-AWS servers

---

## Why CloudFront Improves Latency

- There are **hundreds of Points of Presence (PoPs)** globally.
- Users connect to the **closest edge** instead of the origin (which may be far away).
- Cached content gives **millisecond-level** response times.

Example:

Origin S3 bucket in **Australia** → User in **US**

→ First request fetches from AU → US edge caches it

→ Subsequent US users get **instant** responses from local edge.

---

## CloudFront vs S3 Cross-Region Replication (CRR)

| Feature | CloudFront | S3 Cross-Region Replication |
| --- | --- | --- |
| Purpose | Global caching (CDN) | Replicating bucket across regions |
| Locations | ~216 PoPs worldwide | Only to the regions you configure |
| Update speed | Cached for hours/days | Near real-time replication |
| Content type | Best for **static, read-heavy** content | Best for **dynamic** and replicated data |
| Write support | Read-only delivery | Replication supports writes (in origin bucket) |

### Exam Shortcut:

- **CDN? Performance for global users? → CloudFront**
- **Replicate S3 bucket to another AWS region? → S3 CRR**
