AWS Network Firewall provides **centralized, VPC-wide network protection** across Layers 3–7.

---

## **1. What AWS Network Firewall Does**

- Protects **entire VPC traffic**, not just subnets
- Inspects:
    - Inbound traffic from the internet
    - Outbound traffic to the internet
    - Traffic over Direct Connect
    - Traffic over Site-to-Site VPN
    - Peered VPC traffic
- Works at **VPC Scale**, unlike NACL (subnet-level)

---

## **2. Features**

- Stateful & stateless rule inspection
- Deep packet inspection (Layer 7)
- Domain filtering
- Intrusion prevention features
- Central deployment at the **VPC edge** (via dedicated firewall endpoints)

---

## **When to Use AWS Network Firewall**

- When asked to “protect **an entire VPC**”
- When centralized inspection of all inbound/outbound/through traffic is needed
- When NACLs or SGs aren’t sufficient

---

# ⭐ **CLF-C02 Exam Tips**

- **NACL = subnet firewall**
- **Security Group = instance firewall**
- **Network Firewall = VPC-wide firewall** (the “big shield”)
- If question says “Layer 3–7 protection for entire VPC,” → **AWS Network Firewall**

# **186. AWS Firewall Manager**

A service that lets you centrally manage **security policies across all AWS accounts** in an AWS Organization.

---

## **1. What Firewall Manager Manages**

Firewall Manager centrally applies rules for:

- **Security Groups** (MOST TESTED)
- **AWS WAF rules**
- **AWS Shield Advanced protections**
- **AWS Network Firewall**
- **Other policy types across your org**

---

## **2. How It Works**

- Policies are set at the **Organization root** or an OU
- Automatically enforced across:
    - **All current accounts**
    - **All future accounts created later**
- Ensures standardized security posture across the org

---

# ⭐ **CLF-C02 Exam Tips**

### **1. Most common scenario**

> “Manage VPC Security Groups across multiple AWS accounts.”
> ✔ **AWS Firewall Manager**

### **2. Additional scenarios**

> “Enforce WAF or Shield Advanced policies org-wide.”
> ✔ Firewall Manager

> “Ensure all accounts use consistent Network Firewall rules.”
> ✔ Firewall Manager

### **3. Requirement**

Firewall Manager requires:

- **AWS Organizations**
- **All features enabled**

---

# **Summary: Network Firewall vs Firewall Manager**

| Feature | AWS Network Firewall | Firewall Manager |
| --- | --- | --- |
| Purpose | Protect a single **VPC** | Manage security **across multiple accounts** |
| Scope | Network-level filtering | Organization-wide policy enforcement |
| Works with | VPC traffic (L3–L7) | SG, WAF, Shield, Network Firewall, etc. |
| Exam keyword | “Protect entire VPC” | “Manage rules across accounts” |
