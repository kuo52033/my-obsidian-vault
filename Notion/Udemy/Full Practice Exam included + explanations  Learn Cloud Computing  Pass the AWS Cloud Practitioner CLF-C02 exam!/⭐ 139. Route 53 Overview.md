## What is Route 53?

- **Managed DNS (Domain Name System)** service on AWS.
- Works like a phonebook: maps **domain names → IP addresses / AWS resources**.
- Highly available and scalable.

---

## DNS Record Types

### **A Record**

- Maps hostname → IPv4
e.g., `www.example.com → 192.0.2.10`

### **AAAA Record**

- Maps hostname → IPv6

### **CNAME Record**

- Maps hostname → another hostname
e.g., `api.example.com → backend.example.com`

### **Alias Record (AWS special)**

- Maps hostname → AWS resource (no cost for DNS queries)
- Works for:
    - Elastic Load Balancer (ALB/NLB)
    - CloudFront
    - S3 website endpoints
    - API Gateway
    - Route 53 record sets
- Alias ≠ CNAME
Alias works at root domain & integrates directly with AWS resources.

---

## How DNS Resolution Works (Simple Example)

1. Client requests: `myapp.mydomain.com`
2. Route 53 looks up the DNS record.
3. Returns the corresponding IP (e.g., EC2 public IP).
4. Browser connects to server using that IP.

## Route 53 Routing Policies (Exam-critical)

### **1. Simple Routing Policy**

- Single record → single resource.
- **No health checks**.
- Used for straightforward DNS mapping.

---

### **2. Weighted Routing Policy**

- Distribute traffic across multiple resources.
- Assign weights (e.g., 70%, 20%, 10%).
- **Health checks supported**.
- Use case: A/B testing, gradual traffic shifting, load distribution.

---

### **3. Latency Routing Policy**

- Routes user to the region with the **lowest latency**.
- e.g., users near US → US servers; users near AUS → Australia servers.
- **Health checks supported**.
- Use case: Global applications.

---

### **4. Failover Routing Policy**

- Primary + secondary (DR) setup.
- Health check monitors primary.
- If primary fails → Route 53 auto-redirects to secondary.
- Use case: **Disaster Recovery (DR)**, high availability.
