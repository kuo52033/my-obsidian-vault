## **Overview**

AWS Cost Anomaly Detection uses **machine learning** to continuously analyze your historical spend patterns and detect **unexpected cost spikes** or unusual usage increases.

You **do not need to set thresholds manually** — the ML model learns what “normal” looks like and alerts when something deviates.

---

## **Key Capabilities**

### **✔ ML-based anomaly detection**

- Learns past spending behavior
- Detects both:
    - One-time spikes
    - Gradual abnormal increases

### **✔ Monitors multiple dimensions**

- AWS services
- Member accounts (Organizations)
- Cost allocation tags
- Cost categories

### **✔ Automated reporting**

Provides:

- Anomaly report
- Root cause analysis (example: which service/region/account caused spike)

---

## **Notifications**

You can receive alerts via:

- **Individual alerts** (event-by-event)
- **Daily or weekly summary**
- SNS integration (email or automation)

---

## **Use Cases**

- Unexpected EC2 launch
- Incorrect autoscaling behavior
- Misconfigured Lambda causing high invocation count
- Rogue resources created accidentally
- Abnormal data transfer cost

---

## **Exam Tips**

- AWS Cost Anomaly Detection = **ML-based anomaly detection** for billing
- No need to configure thresholds (the model learns automatically)
- Provides **root cause analysis**
- Sends alerts via **SNS**, daily/weekly summaries

# **229. AWS Service Quotas**

## **Overview**

AWS imposes service limits (called **quotas**) for safety and capacity planning.

**Service Quotas** centralizes the management and monitoring of all these limits.

Examples:

- EC2 instances per region
- Lambda concurrent executions
- VPCs per region
- EBS volume limits

---

## **Key Features**

### **✔ Centralized quota dashboard**

View service limits across all AWS services in one place.

### **✔ CloudWatch alarms for quotas**

You can create alarms directly from the Service Quotas console to notify when usage approaches a limit.

Example:

- Alert if Lambda concurrent executions reach 80% of quota

### **✔ Request quota increases**

Many quotas are **adjustable**.

You can request increases directly from the console—no support ticket required.

### **✔ Helps prevent failures**

Useful for preventing:

- Throttling
- Deployment failures
- Scaling failures
- Unexpected outages

---

## **Workflow Example**

1. Go to **AWS Service Quotas**
2. Select “Lambda → Concurrent executions”
3. View current usage
4. Create a **CloudWatch alarm** if usage exceeds a threshold
5. If needed, click **Request quota increase**

---

## **Exam Tips**

- Service Quotas monitors AWS resource limits
- You can create **CloudWatch alarms** on quota usage
- You can request **quota increases** directly
- Helps avoid hitting capacity limits during scaling events
