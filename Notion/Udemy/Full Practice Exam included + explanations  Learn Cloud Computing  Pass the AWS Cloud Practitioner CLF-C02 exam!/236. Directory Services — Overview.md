## **What is Microsoft Active Directory (AD)?**

- **Microsoft Active Directory (AD)** is a **centralized directory service**.
- Runs on **Windows Server with AD Domain Services**.
- Stores and manages **directory objects**, such as:
    - Users
    - Computers
    - Printers
    - File shares
    - Security groups

---

## **What Problem Does Active Directory Solve?**

- **Centralized identity & access management**
- One username & password can be used to:
    - Log into multiple computers
    - Access internal systems
- Common in **on-premises corporate environments**

---

## **Classic AD Example (On-Prem)**

- User logs into a **Windows laptop**
- Laptop is joined to a **Domain Controller**
- Authentication is validated against **Active Directory**
- Same credentials work across company resources

---

## **Active Directory in AWS**

AWS does **NOT** natively run AD by default,

but it provides **AWS Directory Services** to integrate or host AD.

---

## **AWS Directory Services (High Level)**

> ⚠️ Details are NOT deeply tested in CLF-C02

### **1. AWS Managed Microsoft AD**

- Fully managed **Microsoft Active Directory** in AWS
- AWS manages:
    - Patching
    - Availability
- Supports:
    - **MFA**
    - **Trust relationships** with on-prem AD

### **2. AD Connector**

- **Proxy service**
- Redirects authentication requests from AWS to **on-premises AD**
- Users remain in on-prem AD
- Supports **MFA**

### **3. Simple AD**

- Lightweight, **AD-compatible**
- **NOT full Microsoft AD**
- **Cannot** establish trust with on-prem AD
- Used for simple directory needs only
