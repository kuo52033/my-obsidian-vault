## 🎭 What is an IAM Role?

- An **IAM Role** is an identity that you create in your account that has specific permissions.
- Unlike IAM Users, which are intended for physical people, **IAM Roles are intended for AWS Services** (machines).

## ❓ Why do we need them?

- Many AWS services (like EC2 instances) need to perform actions on your behalf within your AWS account.
- Just like users, these services require **permissions** to access other AWS resources.
- Instead of sharing passwords or keys with these services, we assign them a Role.

## ⚙️ How it works (Example)

The video uses an **EC2 Instance** (a virtual server) as the primary example:

1. You create an EC2 Instance.
2. This instance needs to access information or perform actions in AWS.
3. You create an **IAM Role** with the necessary permissions.
4. You assign this Role to the EC2 Instance.
5. The EC2 Instance now "assumes" this role to perform the allowed actions securely.
