## **1. What Are Savings Plans?**

A **flexible pricing model** that gives major discounts in exchange for committing to a **specific $/hour spend** for **1 or 3 years**.

- You commit to **dollars**, not specific instance types
- Much simpler than Reserved Instances
- Discount applies automatically to eligible compute usage

---

# **2. Types of Savings Plans**

---

## **A. EC2 Savings Plan**

### **Discount: Up to 72% off On-Demand**

### **Scope / Flexibility**

- Applies **only to EC2**
- Limited to a **specific instance family** in a **specific region**
    - e.g., "C5 family in us-east-1"
- Flexible across:
    - Instance **size** (C5.large → C5.12xlarge)
    - **AZ** (any AZ within the same region)
    - **OS** (Linux / Windows)
    - **Tenancy** (shared / dedicated)

### **Payment options**

- All upfront (biggest discount)
- Partial upfront
- No upfront

📌 **Exam Tip:**

EC2 Savings Plan = best if you know the instance family you will always use (like consistent workloads).

---

## **B. Compute Savings Plan**

### **Discount: Up to 66% off On-Demand**

### **Most Flexible Savings Plan**

Applies across:

- **Any EC2 instance family**
- **Any region**
- **Any size**
- **Any OS**
- **Any tenancy**
- **Other compute services**, including:
    - **Fargate**
    - **Lambda**

📌 **Exam Tip:**

Compute Savings Plan = **maximum flexibility**; best when architectures change frequently or use mixed compute.

---

## **C. Machine Learning Savings Plan**

### Supports:

- **SageMaker** workloads
- Up to **64% savings** (varies by plan)

Useful for long-running ML training or inference.

---

# **3. How to Choose the Right Plan**

AWS **Cost Explorer** provides recommendations based on:

- Past usage
- Expected commitment levels
- Historical cost patterns

Exam keyword: **Cost Explorer recommends Savings Plans**

---

# **4. Savings Plans vs Reserved Instances (RI)**

| Feature | Savings Plans | Reserved Instances |
| --- | --- | --- |
| Commitment | $/hour | Specific instance attributes |
| Covers | EC2, Fargate, Lambda (Compute Plan) | EC2 / RDS / Others depending on RI |
| Flexibility | High (Compute Plan) | Low—must match type/region/engine |
| Common Use | Dynamic workloads | Predictable stable workloads |
| Discount | 66–72% | Up to 75%+ |

📌 **Exam trick:**

If the question emphasizes **flexibility**, answer = **Compute Savings Plan**.

If the question emphasizes **EC2 family**, answer = **EC2 Savings Plan**.

---

# **5. Exam-Critical Quick Summary**

- Savings Plans require **1 or 3 year** commitment
- Commitment is in **$/hour**, not instance type
- Two main types:
    - **EC2 Savings Plans** → up to 72% discount
    - **Compute Savings Plans** → up to 66% & applies to EC2, Fargate, Lambda
- Best replacement for many Reserved Instance use cases
- Recommended via **Cost Explorer**
