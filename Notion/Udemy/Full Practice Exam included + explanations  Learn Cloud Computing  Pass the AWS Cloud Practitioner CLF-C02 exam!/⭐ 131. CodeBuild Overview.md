## What is CodeBuild?

AWS CodeBuild is a **fully managed, serverless build service** that compiles your source code, runs tests, and produces deployable artifacts.

It is the “build” stage in a CI/CD pipeline.

---

## How CodeBuild Works

Typical workflow:

1. **Source**
    - CodeBuild pulls source code from a repository
(CodeCommit, GitHub, S3, etc.)
2. **Build**
    - Executes build commands defined in a **buildspec.yml**
    - Compiles application
    - Runs tests
    - Packages artifacts
3. **Output**
    - Generates a ready-to-deploy package
    - Can be consumed by CodeDeploy, Elastic Beanstalk, or other services

Diagram:

```plain text
CodeCommit → CodeBuild → build/test → artifact → (CodeDeploy)


```

---

## Key Features

### ✔ Fully Managed & Serverless

- No servers to provision or manage
- Automatically scales to meet build volume

### ✔ Pay-as-you-go

- You only pay for **build minutes**
- No idle cost

### ✔ Supports Many Languages

- Java, Node.js, Python, Go, .NET, Ruby, Android, Docker, etc.
- Custom environments supported via Docker images

### ✔ Secure

- Integrated with IAM
- Builds run inside isolated environments

---

## Why Use CodeBuild?

Use CodeBuild when you need to:

- Compile code in the cloud
- Run automated tests
- Generate deployment-ready artifacts
- Integrate into a CI/CD pipeline (with CodePipeline)
