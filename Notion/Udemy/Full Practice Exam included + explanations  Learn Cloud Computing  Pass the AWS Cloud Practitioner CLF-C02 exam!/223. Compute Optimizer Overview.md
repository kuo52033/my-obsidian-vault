## **1. What Is AWS Compute Optimizer?**

A service that uses **machine learning** to analyze resource usage and recommend **cost-efficient** and **performance-optimized** AWS resource configurations.

### **Primary Goals**

- **Reduce costs** (up to ~25%)
- **Improve performance**
- **Eliminate over-provisioning / under-provisioning**

---

## **2. How Compute Optimizer Works**

Compute Optimizer automatically:

1. Analyzes your **resource configuration**
2. Monitors **CloudWatch metrics**
3. Uses ML models trained on AWS best practices
4. Provides *right-sizing* recommendations

---

## **3. Supported Resource Types**

Compute Optimizer supports optimization for:

- **EC2 Instances**
    - Right-sizing recommendations (over-provisioned / under-provisioned)
- **Auto Scaling Groups**
    - Optimized instance types and desired capacity
- **EBS Volumes**
    - Suggests GP2 → GP3 or right-size adjustments
- **Lambda Functions**
    - Memory size right-sizing based on execution behavior

---

## **4. Types of Recommendations**

Compute Optimizer generates suggestions such as:

- Smaller / larger EC2 instance type
- Switching instance families (e.g., m5 → m6i)
- Changing EBS volume type (e.g., gp2 → gp3)
- Increasing Lambda memory for better performance
- Reducing ASG capacity to match load

Each recommendation includes:

- Performance risk score
- Price comparison
- Projected savings
- Utilization metrics

---

## **5. Exporting Recommendations**

You can export results to:

- **Amazon S3**
- For further analysis (CSV)
- Or integrate into reporting workflows

---

## **6. Exam Tips**

- Compute Optimizer = **right-sizing recommendations**
- Uses **ML + CloudWatch metrics**
- Supports EC2, ASG, EBS, Lambda
- Helps reduce cost up to **25%**
- Results can be **exported to S3**

📌 If a question says: *“Which AWS service helps reduce cost by identifying over-provisioned EC2 or Lambda resources?”*

👉 **Answer: AWS Compute Optimizer**
