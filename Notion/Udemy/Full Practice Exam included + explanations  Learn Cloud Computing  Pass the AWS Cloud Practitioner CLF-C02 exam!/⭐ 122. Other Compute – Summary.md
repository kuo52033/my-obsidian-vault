## Docker (Container Technology)

- Docker packages applications into containers for consistent execution.
- Containers can run on any machine without compatibility issues.
- Lightweight and fast to scale.

---

## ECS (Elastic Container Service)

| Feature | Description |
| --- | --- |
| Infrastructure | Must provision EC2 instances yourself |
| Container placement | ECS schedules containers onto your EC2 instances |
| Use case | Run Docker containers with full control over compute resources |

---

## Fargate

| Feature | Description |
| --- | --- |
| Infrastructure | **No EC2 provisioning** (serverless) |
| Scaling | Automatically runs containers on AWS-managed servers |
| Use case | Simplest way to run containers without managing servers |

**Fargate = serverless container compute.**

---

## ECR (Elastic Container Registry)

- Private Docker image registry on AWS.
- Stores container images for ECS and Fargate.
- Fully integrated with IAM for security.

---

## AWS Batch

| Feature | Description |
| --- | --- |
| Purpose | Run batch processing jobs at any scale |
| How it works | Dynamically provisions EC2 or Spot Instances |
| Under the hood | Uses ECS as the container compute engine |
| Use case | Long-running jobs, data processing, simulations |

**Batch ≠ serverless** → relies on EC2 compute managed by AWS.

## Lightsail

Simplified compute platform with:

- Virtual servers
- Databases
- Networking
- Simple pricing model

Designed for:

- Beginners with no cloud experience
- Simple web apps (WordPress, LAMP, Node.js, etc.)

**Often a distractor in exams unless simplicity is required.**

---

## Lambda (Serverless Compute)

- Completely serverless function execution.
- Automatically scales from 1 request to thousands per second.
- Event-driven (invoked only when needed).
- Supports many languages: Node.js, Python, Java, C#, Ruby, etc.
- Can run container images **only if** they implement the Lambda Runtime API.

### Pricing

- Billed by:
    1. **Function duration** (time × memory)
    2. **Number of invocations**

### Limitations

- Max runtime: **15 minutes**
- Not suitable for large container workloads → use ECS/Fargate instead.

---

## API Gateway

- Fully managed service to expose REST or WebSocket APIs.
- Commonly paired with Lambda for serverless APIs.
- Supports:
    - Authentication
    - Throttling
    - API keys
    - Monitoring
    - Versioning
