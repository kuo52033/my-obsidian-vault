This section summarizes all AWS database and analytics services and the scenarios where each one is the correct choice in the exam.

Use this as your quick-reference guide to match **use case → correct AWS service**.

---

## Relational Databases (SQL / OLTP)

### **RDS (Relational Database Service)**

- Managed relational databases (MySQL, PostgreSQL, MariaDB, Oracle, SQL Server)
- Best for **OLTP** workloads (transactional applications)
- Supports SQL queries

### **Aurora**

- AWS cloud-optimized relational database
- Compatible with MySQL & PostgreSQL
- Higher performance and auto-scaling storage
- Same relational/OLTP use cases as RDS

### **RDS Deployment Options**

| Feature | Purpose |
| --- | --- |
| **Read Replicas** | Scale READ queries |
| **Multi-AZ** | High availability & failover |
| **Multi-Region Read Replicas** | Global reads + disaster recovery |

---

## In-Memory Databases

### **ElastiCache (Redis & Memcached)**

- Fully managed in-memory cache
- Ultra-low latency
- Reduces read pressure on RDS

Used for:

- Caching
- Session stores
- Leaderboards

---

## Key-Value Database (NoSQL)

### **DynamoDB**

- Fully managed, **serverless** NoSQL key-value database
- Single-digit millisecond performance
- Scales to **millions of requests per second**

### **DAX (DynamoDB Accelerator)**

- In-memory cache **specifically for DynamoDB**
- Microsecond read latency

---

## Analytics & Data Warehousing (OLAP)

### **Redshift**

- Columnar storage data warehouse
- Optimized for analytics (OLAP)
- Integrates with BI tools

### **EMR (Elastic MapReduce)**

- Managed Hadoop/Spark cluster
- Used for **big data processing** (ETL pipelines, ML, batch jobs)

### **Athena**

- Serverless SQL queries **directly on S3 data**
- Great for log analysis, BI queries, ad-hoc analytics

### **QuickSight**

- Serverless BI dashboarding tool
- Used to create visualizations and business reports

---

## Document, Graph, and Time-Series Databases

### **DocumentDB**

- MongoDB-compatible NoSQL document database
- Stores and queries **JSON-like documents**
- Use when you see “MongoDB” in exam questions

### **Neptune**

- Fully managed **graph database**
- Best for:
    - Social networks
    - Knowledge graphs
    - Recommendation engines
    - Fraud detection

### **Timestream**

- Serverless **time-series** database
- Designed for metrics, IoT telemetry, events changing over time
- Fast and cost-efficient for time-series analytics

---

## Other Database-Related Services

### **Glue**

- Serverless **ETL service**
- Extract → Transform → Load for analytics
- Includes the **Glue Data Catalog** (metadata store)

### **DMS (Database Migration Service)**

- Migrates databases with **minimal downtime**
- Supports:
    - Homogeneous migrations (Oracle → Oracle)
    - Heterogeneous migrations (SQL Server → Aurora)

### **Managed Blockchain**

- Managed Hyperledger Fabric & Ethereum networks
- Used when exam mentions **blockchain**, **decentralized applications**, **ledger technology**
