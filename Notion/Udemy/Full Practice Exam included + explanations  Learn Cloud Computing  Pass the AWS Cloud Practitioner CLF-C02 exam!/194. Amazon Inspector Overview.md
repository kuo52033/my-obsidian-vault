Amazon Inspector is an **automated vulnerability scanning service** for AWS workloads.

It detects security issues such as **CVE vulnerabilities** and **network exposure risks**.

---

# **1. What Inspector Scans**

Inspector automatically and continuously analyzes **three types of resources:**

## **A. EC2 Instances**

- Uses **SSM Agent** on EC2
- Scans for:
    - Known OS vulnerabilities (CVE database)
    - Unintended network exposure (reachable ports, misconfigurations)

## **B. Container Images in Amazon ECR**

- Scans Docker/container images **upon push** or periodically
- Detects:
    - Package vulnerabilities
    - Dependency vulnerabilities
    - CVEs in image layers

## **C. Lambda Functions**

- Scans Lambda code + package dependencies
- Runs analysis when new versions are deployed
- Identifies:
    - Vulnerable libraries
    - Insecure dependencies

---

# **2. How Inspector Works**

- Continuously monitors workloads
- Scans automatically when:
    - A new EC2 instance launches
    - A new ECR image is pushed
    - A new Lambda version is deployed
- Uses the **CVE vulnerability database**
- Assigns **risk scores** to prioritize remediation

---

# **3. Integrations**

Inspector findings can be sent to:

### **A. AWS Security Hub**

- Centralized security findings dashboard

### **B. Amazon EventBridge**

- Trigger notifications (SNS)
- Trigger automated remediation (Lambda)
- Integrate with SIEM tools

---

# **4. Key Characteristics**

- Completely **automated**
- No manual scans required
- Continuously updated CVE data
- Scales automatically across workloads
- No agent required for Lambda or ECR (agentless scanning)
- Requires SSM Agent for EC2 scanning
