## What is Amazon Redshift?

Amazon Redshift is a **fully managed, petabyte-scale data warehouse** service optimized for **analytics (OLAP)** rather than transactional workloads (OLTP).

### Key characteristics

- Based on **PostgreSQL**, but heavily modified for analytics
- Designed for **Online Analytical Processing (OLAP)**
- Columnar storage format (store data by column, not row)
- Uses **MPP (Massively Parallel Processing)** for fast queries
- Scales to **petabytes** of structured data
- Pay-as-you-go based on compute nodes
- Works well with BI tools: QuickSight, Tableau, Power BI

---

## When to Use Redshift?

Redshift is ideal when the scenario mentions:

- **Data warehouse**
- **Analytics / BI / dashboards**
- **Large datasets (TB to PB)**
- **Columnar storage**
- **Complex aggregations or data transformations**

---

## Redshift vs RDS (Quick Comparison)

| Feature | RDS | Redshift |
| --- | --- | --- |
| Purpose | OLTP (transactions) | OLAP (analytics) |
| Query type | Simple queries, frequent writes | Complex analytical queries |
| Data size | GB–TB | TB–PB |
| Storage model | Row-oriented | Columnar |
| Performance | Optimized for transactions | Optimized for aggregations & scans |
| Integration | App databases | BI tools, dashboards |

---

## Redshift Serverless

A newer mode where:

- No need to provision clusters
- Auto-scales based on query load
- Pay only for compute time + storage
- Ideal for **unpredictable** or **intermittent** analytics workloads

# 102. EMR Overview

## What is EMR?

EMR = **Elastic MapReduce**, a managed service to run **Hadoop & big data workloads** on AWS.

### Key characteristics

- Not a database → it is a **big data processing framework**
- Can run:
    - Hadoop
    - Spark
    - HBase
    - Flink
    - Presto
- Can scale to **hundreds of EC2 nodes**
- Integrates with Spot instances for major cost savings
- Ideal for **batch processing** and **large-scale analytics**

---

## When to Use EMR?

Look for keywords:

- **Big Data**
- **Hadoop cluster**
- **Spark jobs**
- **Batch processing**
- **ETL pipelines**
- **Machine Learning with big data**

---

## Redshift vs EMR (Quick Comparison)

| Feature | Redshift | EMR |
| --- | --- | --- |
| Type | Data warehouse | Big data processing cluster |
| Storage | Columnar | Depends (HDFS, S3, etc.) |
| Use case | BI, dashboards | ETL, ML pipelines, transformations |
| Query languages | SQL | Spark, Hive, Presto, etc. |
| Scaling | Cluster or serverless | Dynamically add EC2 nodes |

# 103. Athena Overview

## What is Athena?

Athena is a **serverless SQL query engine** that runs analytics directly on **data stored in S3**.

### Key characteristics

- **No servers**, no cluster to manage
- SQL query engine built on **Presto**
- Supports file formats:
    - CSV, JSON, Parquet, ORC, Avro
- Pay **only for data scanned**
    - ~$5 per TB scanned
- Cost can be reduced by using **columnar formats** (Parquet/ORC)

---

## When to Use Athena?

Use Athena when:

- You need **serverless analytics**
- Your data is **already in S3**
- You want **ad-hoc queries** without setting up a database
- You need to analyze AWS logs:
    - VPC Flow Logs
    - CloudTrail Logs
    - ELB Logs

---

## Athena vs Redshift vs EMR

| Feature | Athena | Redshift | EMR |
| --- | --- | --- | --- |
| Type | Query engine | Data warehouse | Big data cluster |
| Storage | S3 | Local cluster storage | HDFS/S3 |
| Setup | No setup | Need cluster (or serverless) | Need cluster |
| Ideal for | Quick SQL on S3 | BI & dashboards | ETL, ML, batch processing |
| Pricing | Per TB scanned | Node-hours | EC2-hours |

# 104. QuickSight Overview

## What is QuickSight?

QuickSight is a **serverless BI (Business Intelligence)** service for creating interactive dashboards.

### Key characteristics

- No servers to manage
- Auto-scales to thousands of users
- ML-powered insights (e.g., anomaly detection)
- Supports embedded analytics in applications
- Pay-per-session pricing available

---

## Integrations

QuickSight can visualize data from:

- RDS
- Aurora
- Redshift
- S3 (via Athena)
- DynamoDB
- EMR
- Many external databases

---

## When to Use QuickSight?

Look for keywords:

- **Dashboards**
- **Visualizations**
- **Business intelligence**
- **Charts, graphs**
- **Analytics for business users**
