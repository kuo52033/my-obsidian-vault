## What is Hybrid Cloud?

A **hybrid cloud** setup means an organization uses:

- **On-premises infrastructure**, and
- **Cloud infrastructure (AWS)**
at the same time.

This usually requires **two sets of tools, APIs, and skillsets**, making operations complex.

---

## What is AWS Outposts?

AWS Outposts brings **AWS hardware** (server racks) **into your on-premises data center**.

You get:

- The **same AWS infrastructure**
- The **same APIs**
- The **same console & tools**
- The **same services** (EC2, EBS, S3, RDS, ECS/EKS, EMR…)

But running **physically inside your building**.

AWS delivers, installs, monitors, and maintains the Outpost hardware.

You now operate on-prem resources *as if they were AWS cloud resources*.

---

## Why Outposts?

Because some workloads:

- Require **ultra-low latency** to on-prem systems
- Have strict **data residency requirements**
- Need **local processing**
- Or want **gradual migration** from data center → cloud

Outposts lets companies stay hybrid while using AWS APIs everywhere.

---

## Key Benefits

- **Low latency** to local systems
(since compute is physically in your data center)
- **Local data processing**
(data can stay on-prem; doesn’t need to go to AWS region)
- **Data residency / compliance**
- **Unified AWS API + console**
→ same tools for on-prem & cloud
- **Fully managed hardware**
(AWS installs + maintains the Outposts rack)
- **Easy migration path**
On-prem → Outposts → AWS Cloud

---

## What Services Run on Outposts?

Outposts currently supports major AWS services, including:

- **EC2** (virtual machines)
- **EBS** (block storage)
- **S3 (Outposts)** (local object storage)
- **ECS & EKS** (containers)
- **RDS** (databases)
- **EMR** (big data)

These run *locally* on Outposts, but controlled through the AWS console.

---

## Important Responsibility Difference

In the cloud:

- AWS handles **physical security**

With Outposts (installed in customer datacenter):

- **Customer** is responsible for physical security
(the rack is on-site)
- AWS still manages the hardware and software

---

## When to Use AWS Outposts? (Exam-style)

Use Outposts when:

- You must run AWS services **on-prem**
- You need **consistent AWS APIs** across cloud & data center
- You need **sub-millisecond latency** to local equipment
- Applications cannot send data to public cloud due to regulation
- You want a **hybrid migration strategy**
