## **1. What is AWS RAM?**

AWS Resource Access Manager (RAM) allows you to **share AWS resources across accounts** without duplication.

You can share with:

- Other AWS accounts
- AWS Organizations OUs or entire org

---

## **2. Why Use RAM?**

✔ Avoid resource duplication

✔ Improve network architecture efficiency

✔ Centralize resource ownership while allowing cross-account usage

---

## **3. Common RAM-Sharable Resources**

(記考試：記概念，不用背全部種類)

- **VPC Subnets**
- **Transit Gateway**
- **Route53 Resolver rules**
- **Aurora / RDS Clusters**
- **License Manager configurations**
- **Outposts**
- …and many more

---

## **4. How RAM Works (Example → High Yield)**

### **Scenario:**

- Account A owns a **VPC + Private Subnets**
- Account B and C need to launch EC2 inside those subnets

### **Using RAM:**

1. Account A shares VPC Subnets via RAM
2. Account B / C can **launch EC2 instances** into Account A’s VPC
3. All resources now **communicate over the same network**

👉 **Result:**

Cross-account workloads run inside the same VPC → no VPC peering, no transit gateway, full network connectivity.

---

## **5. Exam Tips**

- RAM = **share AWS resources across accounts**
- Often used for **Subnet Sharing** → multiple accounts deploy workloads in one centralized VPC
- Helps **avoid duplication** and **simplify multi-account networking**

# **220. AWS Service Catalog – Overview**

## **1. What Problem Does It Solve?**

New or non-expert AWS users may:

- Create misconfigured resources
- Break organizational standards
- Deploy insecure or expensive setups

Service Catalog solves this by giving them **only approved, preconfigured options**.

---

## **2. What Is Service Catalog?**

A service that lets administrators create **pre-approved, centrally managed products** for internal users.

Users get a **self-service portal** with only what they are allowed to deploy.

---

## **3. Key Concepts**

### **✔ Products**

- A product = a **CloudFormation template**
- Defines allowed resources + configurations

### **✔ Portfolios**

- A collection of products
- Admins decide which users/roles can access which portfolios

### **✔ Users**

- Can launch only the products they have permissions for
- Deployments occur automatically using **CloudFormation**

---

## **4. Benefits**

- Ensures **consistency** (resources follow organizational standards)
- Enforces **tagging**, **networking**, **security**, and **cost policies**
- Provides a simple **self-service catalog** for internal teams
- Prevents misconfigurations by inexperienced AWS users

---

## **5. Example Use Case**

A user wants to launch an RDS database but doesn’t know:

- VPC setup
- Security groups
- Parameter groups
- Backup retention

Admin provides a **“RDS-Standard-MySQL-Template” product**.

User selects it → CloudFormation handles everything → compliant by design.

---

## **6. Exam Tips**

- Service Catalog = **self-service portal for approved AWS resources**
- Used to enforce **governance, consistency, compliance**
- Products = CloudFormation templates
- Portfolios = group of products
