## **What is AWS DataSync?**

AWS DataSync is a **managed data transfer service** used to **move large amounts of data** from **on-premises environments to AWS** (or between AWS services).

---

## **Supported Destinations**

DataSync can synchronize data into:

- **Amazon S3** (any storage class)
- **Amazon EFS**
- **Amazon FSx for Windows File Server**

---

## **How AWS DataSync Works**

1. Deploy a **DataSync Agent** on-premises
2. Agent connects securely to **AWS DataSync service**
3. Data is transferred to AWS storage (S3 / EFS / FSx)
4. First transfer = **full data copy**
5. Subsequent transfers = **incremental updates only**

---

## **Key Feature (VERY IMPORTANT FOR EXAM)**

### ✅ **Incremental Transfers**

- After the initial sync, **only changed data is transferred**
- Reduces bandwidth usage and transfer time

**Exam Keyword**

> If you see “incremental sync”, think AWS DataSync

---

## **Scheduling Options**

- One-time migration
- Scheduled sync:
    - Hourly
    - Daily
    - Weekly

---

## **Primary Use Cases**

- Migrate on-premises file servers to AWS
- Keep on-prem and AWS storage **in sync**
- Periodic data replication to AWS
- Large-scale data transfer

---

## **What AWS DataSync Is NOT**

- ❌ Not real-time streaming
- ❌ Not database replication
- ❌ Not backup replacement (use **AWS Backup** instead)

---

## **AWS DataSync vs Similar Services (Exam Tip)**

| Service | Purpose |
| --- | --- |
| **AWS DataSync** | Large-scale, **scheduled & incremental** data transfer |
| AWS Snowball | Offline, physical data transfer |
| AWS Backup | Backup & restore |
| AWS DRS | Disaster recovery (block-level replication) |

---

## **One-Line Exam Summary**

> AWS DataSync = large-scale, scheduled, incremental data transfer from on-premises to AWS storage
