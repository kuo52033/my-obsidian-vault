## **What is AWS FIS?**

AWS Fault Injection Simulator (FIS) is a **managed service for running fault injection experiments** on AWS workloads.

It is based on **Chaos Engineering**.

---

## **Chaos Engineering (Concept)**

Chaos Engineering means **intentionally creating failures** in your system to:

- Test resilience
- Validate recovery mechanisms
- Discover hidden bugs and bottlenecks

Instead of waiting for real failures, you **simulate them safely**.

---

## **Why Use FIS?**

- Improve **application resilience**
- Validate **high availability & fault tolerance**
- Identify **hidden bugs**
- Detect **performance bottlenecks**
- Test **monitoring & observability**

---

## **What Kind of Failures Can Be Simulated?**

Examples of chaos events:

- EC2 instances termination
- CPU or memory stress
- ECS tasks stopping
- EKS pod disruptions
- RDS failures

⚠️ You **do NOT need to memorize specific actions for the exam**.

---

## **How AWS FIS Works**

### **1. Create an Experiment**

- Use **pre-built templates**
- Define:
    - Target resources (EC2, ECS, EKS, RDS, etc.)
    - Fault actions
    - Duration & scope

---

### **2. Run the Experiment**

- FIS injects faults into selected resources
- Simulates real production failures

---

### **3. Observe System Behavior**

Use observability tools:

- Amazon CloudWatch
- Amazon EventBridge
- AWS X-Ray

---

### **4. Analyze Results**

- Did the app recover?
- Were there outages?
- Were alerts triggered correctly?
- Any performance degradation?

---

### **5. Improve the System**

- Fix bottlenecks
- Improve failover
- Strengthen monitoring
- Increase resilience

---

## **Key Characteristics**

- Integrates with monitoring tools
