## What is Elastic Beanstalk?

Elastic Beanstalk is a **Platform as a Service (PaaS)** on AWS.

It lets developers **deploy applications without managing infrastructure**.

AWS handles:

- EC2 instances
- Auto Scaling Group
- Load Balancer
- Launch configurations
- OS patching
- Deployment versions
- Health monitoring

You only provide **your application code**.

---

## Why use Beanstalk?

Beanstalk simplifies the typical **3-tier web architecture**:

```plain text
Users → Load Balancer → EC2 (ASG) → RDS / ElastiCache
```

Instead of configuring these one by one, Beanstalk automatically sets them up.

### Benefits

- Developer-friendly
- Automated capacity provisioning
- Automated load balancing
- Automated scaling
- Built-in health monitoring
- Supports multiple environments (dev/test/prod)
- Free to use — only pay for underlying resources

---

## Deployment Models

Beanstalk supports 3 architecture patterns:

1. **Single Instance**
    - For development or testing
    - One EC2 instance, no load balancer
2. **Load Balanced + Auto Scaling**
    - For production web apps
    - LB + ASG
3. **Worker Environment**
    - For background jobs
    - Only an ASG (no LB)
    - Typically consumes SQS messages

Beanstalk supports many languages and runtimes:

- Node.js
- Python
- Java
- Ruby
- Go
- .NET
- PHP
- Docker (single or multi-container)

---

## Health Monitoring

- Each EC2 instance runs a **Beanstalk health agent**
- Pushes metrics to CloudWatch
- Reports health states (OK / Warning / Degraded / Severe)
- You can view metrics, logs, and perform troubleshooting directly within the Beanstalk console

---

## Summary

Elastic Beanstalk = simple application deployment with AWS-managed infrastructure.

It’s ideal when developers want to **deploy code**, not manage servers, networks, or scaling.
