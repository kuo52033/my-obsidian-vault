## What is AWS Systems Manager (SSM)?

AWS Systems Manager (SSM) is a **hybrid operations management service** used to manage:

- EC2 instances
- On-premises servers/VMs
- Across Linux, Windows, macOS, and even Raspberry Pi

It provides operational insights and allows centralized management at scale.

---

## Key Capabilities

### ✔ Fleet Management

SSM lets you manage large fleets of instances consistently across AWS and on-premises.

### ✔ Automatic Patching

Patch Manager provides automated OS patching for compliance.

### ✔ Run Commands (Remote Execution)

Run Command allows you to safely execute commands across **multiple servers at once**:

- Install software
- Update configuration
- Restart services
- Collect logs

All without SSH or RDP.

### ✔ Parameter Store

Secure and scalable storage for:

- Configuration values
- Secrets (API keys, passwords)
- Environment variables

Often used by applications and CI/CD pipelines.

---

## Hybrid Architecture (EC2 + On-premises)

SSM is hybrid. It works for:

- AWS EC2 instances
- On-premises VMs/servers
- Edge devices

This allows consistent operations across environments.

---

## How SSM Works

### 1. SSM Agent

Each managed server must run the **SSM Agent**.

- Preinstalled in Amazon Linux and many Ubuntu AMIs
- Must be manually installed for on-prem systems

The agent communicates with the Systems Manager service.

### 2. SSM Service

Once the agent is connected, SSM can:

- Run commands
- Patch servers
- Apply configurations
- Retrieve parameters

---

## When to Think of SSM (Exam Tips)

If the question describes:

- Managing EC2 and on-premises servers **together**
- Automating OS patching
- Running the same command on many servers
- Storing configuration or secrets
- A “hybrid operations” requirement

→ The answer is **AWS Systems Manager (SSM)**.
