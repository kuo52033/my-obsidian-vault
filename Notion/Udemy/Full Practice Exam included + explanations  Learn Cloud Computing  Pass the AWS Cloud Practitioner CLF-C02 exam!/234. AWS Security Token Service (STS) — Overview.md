## **What is AWS STS?**

- **AWS STS (Security Token Service)** allows you to create **temporary, limited-privilege credentials** to access AWS resources.
- These credentials are **short-lived** and automatically expire.
- STS is a **core AWS service** used behind the scenes in many AWS features.

---

## **How STS Works (Conceptually)**

1. A user or service **assumes an IAM role**.
2. An **STS API call** (e.g., `AssumeRole`) is made.
3. STS returns **temporary security credentials**, including:
    - Access Key ID
    - Secret Access Key
    - Session Token
4. These credentials are valid for a **limited duration**.
5. The user/service accesses AWS resources using the **permissions of the assumed role**.

---

## **Key Characteristics**

- Credentials are:
    - **Temporary**
    - **Time-limited**
    - **Automatically rotated**
- Reduces the need for long-term credentials.
- Improves overall security posture.

---

## **Common Use Cases (EXAM-FOCUSED)**

### **1. Identity Federation**

- Users authenticate using **external identity providers** (e.g., corporate directory, SSO).
- STS issues temporary credentials to access AWS.
- No need to create IAM users for every external user.

---

### **2. IAM Roles (Cross-Account or Same-Account Access)**

- A user or service **assumes a role** in:
    - The same AWS account
    - A different AWS account
- Enables secure **cross-account access**.

---

### **3. EC2 Instance Roles (Behind the Scenes)**

- When you attach an **IAM role to an EC2 instance**:
    - STS automatically provides temporary credentials.
    - Credentials are refreshed automatically.
- No access keys stored on the instance.

---

## **Why STS Is Important**

- Eliminates long-term credentials.
- Enforces **least privilege**.
- Enables secure access patterns (SSO, cross-account, service-to-service).

---

## **Exam Tip (VERY IMPORTANT)**

> If a question mentions temporary, time-limited, or short-term credentials,
> the correct answer is **AWS STS**.
