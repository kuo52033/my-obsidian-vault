## What is Session Manager?

**SSM Session Manager** is a Systems Manager feature that lets you start a **secure shell session** on:

- EC2 instances
- On-premises servers

— **without SSH**, no port 22, no SSH keys, no bastion host.

This significantly increases security by removing the need to expose SSH ports or manage SSH key rotation.

---

## How It Works

### Architecture

- Each managed instance runs the **SSM Agent**.
- The agent maintains a secure connection to the **Session Manager service**.
- Users connect to the instance **via the AWS Console or CLI**, not through SSH.

Flow:

`User → SSM Session Manager → SSM Agent → EC2 instance`

### Supported OS

- Linux
- Windows
- macOS

### Logging Integration

Session logs can be sent to:

- Amazon S3
- CloudWatch Logs

This is useful for auditing and compliance.

---

## Benefits (Exam Tips)

- No need to open port 22 → **improved security**
- No SSH keys required
- Centralized access control via IAM
- Session activity logging
- Works for both EC2 and on-prem servers

**If the question asks about connecting to EC2 with no SSH / no open ports, the answer is Session Manager.**

# 136. SSM Parameter Store

## What is Parameter Store?

A **serverless, scalable, secure** key/value storage service for:

- Application configuration
- Secrets (API keys, passwords)
- Environment variables

No infrastructure to manage; it scales automatically.

---

## Key Features

### ✔ Store Configurations & Secrets

Two parameter types:

- **String / StringList** – plain text
- **SecureString** – encrypted using **KMS**

### ✔ Versioning

Each update to a parameter creates a new version.

### ✔ IAM Access Control

Fine-grained permissions:

- Per parameter
- Per path (`/app/dev/db/password`)

### ✔ High Availability & Durable

Delivered as a fully managed AWS service.

---

## Usage (Common Patterns)

Applications can pull configuration from Parameter Store:

- During startup
- At runtime
- Through Lambda, EC2, ECS, EKS, etc.

---

## When to Choose Parameter Store (Exam Tips)

If the question mentions:

- Storing secrets or configuration
- Needing encryption (KMS)
- Centralized config for EC2 / Lambda / containers
- Want serverless, simple key/value management

→ Think **SSM Parameter Store**.
