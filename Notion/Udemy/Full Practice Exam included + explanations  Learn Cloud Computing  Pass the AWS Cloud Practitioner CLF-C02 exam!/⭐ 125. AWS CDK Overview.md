## What is AWS CDK?

AWS CDK (Cloud Development Kit) lets you **define AWS infrastructure using real programming languages**, instead of writing YAML/JSON CloudFormation templates manually.

You can use:

- TypeScript / JavaScript
- Python
- Java
- C# (.NET)
- Go

CDK converts your code into **CloudFormation templates**, then deploys them.

---

## Why use CDK instead of CloudFormation?

CloudFormation is powerful, but writing YAML/JSON can be verbose and repetitive.

CDK gives you:

- **Familiar programming languages**
- **Type safety**
- **Reusable components**
- **Loops, conditions, classes, functions**
- **Share app code + infrastructure code (IaC) in same language**

CDK → generates a CloudFormation template → CloudFormation deploys it.

---

## How CDK works (flow)

1️⃣ You write code in Python/TS/etc.

2️⃣ CDK synthesizes it into CloudFormation JSON/YAML

3️⃣ CloudFormation deploys the infrastructure

```plain text
Your Code (e.g., Python)
       ↓ CDK CLI (cdk synth)
Generated CloudFormation
       ↓
AWS Infrastructure
```

---

## Example use case

Define:

- VPC
- ECS Cluster
- Application Load Balancer
- Fargate Service

All written in TypeScript/Python instead of YAML.

### Example code (simplified TypeScript)

```typescript
const vpc = new ec2.Vpc(this, 'MyVpc');
const cluster = new ecs.Cluster(this, 'MyCluster', { vpc });
const alb = new elbv2.ApplicationLoadBalancer(this, 'LB', { vpc });
```

CDK turns this into a CloudFormation template for you.

---

## When to use CDK?

Use CDK when:

- You prefer coding over YAML
- You want reusable infra modules
- You want strong typing and IDE auto-completion
- You deploy infrastructure and application code together (e.g., Lambda, ECS)
