This section summarizes all AWS monitoring & governance services and when to use each.

---

## **1. CloudWatch**

### **CloudWatch Metrics**

- Monitors **performance metrics** of AWS services (CPU, Network, S3 requests, EBS I/O)
- Includes **Billing metrics** (us-east-1 only)

### **CloudWatch Alarms**

- Trigger actions when metrics cross a threshold
→ EC2 actions (reboot/stop), Auto Scaling, SNS notifications

### **CloudWatch Logs**

- Centralized log collection from:
    - EC2 (via CloudWatch Agent)
    - Lambda (automatic)
    - On-premise servers
- Allows retention control + real-time log analysis

### **CloudWatch Events / EventBridge**

- React to AWS service events
- Create **scheduled rules** (cron jobs)
- Route events to Lambda, SNS, SQS, Step Functions, etc.

---

## **2. CloudTrail**

- Logs **all API calls** across console, CLI, SDK, services
- Enabled by default (90 days history)
- Create Trails for:
    - Long-term retention in S3
    - Real-time analysis in CloudWatch Logs
- **CloudTrail Insights** → detects unusual API activity

---

## **3. AWS X-Ray**

- **Distributed tracing** for microservice/system analysis
- Visualizes request flow & service interactions
- Identifies bottlenecks, errors, latency issues

---

## **4. AWS Health Dashboard**

### **Service Health Dashboard**

- Public AWS-wide service status
- Not specific to your account

### **Account Health Dashboard**

- Personalized alerts for **your resources only**
- Shows scheduled maintenance or incidents affecting your workloads
- Can aggregate across an AWS Organization

---

## **5. Amazon CodeGuru**

### **CodeGuru Reviewer**

- ML-powered **static code review**
- Detects bugs, security issues, resource leaks

### **CodeGuru Profiler**

- Optimizes runtime performance (CPU, latency, memory)
- Helps reduce compute costs in production or pre-prod

---

# ⭐ **CLF-C02 Exam High-Level Guide**

### **If you need to monitor performance** → **CloudWatch Metrics**

### **If you need alerts / automatic actions** → **CloudWatch Alarms**

### **If you need log collection & search** → **CloudWatch Logs**

### **If you need cron jobs or event routing** → **EventBridge**

### **If you need to know *****who did what*** → **CloudTrail**

### **If you need distributed tracing** → **X-Ray**

### **If you need AWS service health** → **Service Health Dashboard**

### **If you need resource-specific impact alerts** → **Account Health Dashboard**

### **If you want automated code review** → **CodeGuru Reviewer**

### **If you want performance optimization** → **CodeGuru Profiler**
