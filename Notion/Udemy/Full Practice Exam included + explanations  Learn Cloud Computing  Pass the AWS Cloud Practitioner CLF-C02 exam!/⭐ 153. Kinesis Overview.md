## What is Kinesis?

Amazon Kinesis is AWS’s **real-time big data streaming** platform.

If the exam mentions:

- **real-time data**
- **streaming**
- **clickstream**
- **IoT telemetry**
- **logs or metrics ingestion**
→ Think **Kinesis**.

---

## Kinesis Data Streams (KDS)

Kinesis Data Streams lets you **collect, process, and analyze real-time streaming data at any scale**.

### Typical data sources:

- Website clickstreams
- Mobile apps
- IoT sensors
- Application logs
- Metrics, events

### Why use Kinesis Data Streams?

- Real-time ingestion (sub-second latency)
- Scales massively
- Multiple consumers can read the same stream
- Good for analytics, dashboards, ML, monitoring

---

## Kinesis Data Firehose

A second service often paired with Data Streams.

### Purpose:

**Automatically delivers streaming data into destinations**, such as:

- Amazon S3
- Amazon Redshift
- Amazon OpenSearch
- 3rd-party endpoints (e.g., Datadog, Splunk)

### Key points:

- Fully managed (no servers)
- Auto-scales
- Can batch, compress, and transform data before delivery

---

## Typical Architecture

**Fast data sources → Kinesis Data Streams → Kinesis Firehose → Storage/Analytics**

Examples:

- User clicks → KDS → Firehose → S3 (for Athena queries)
- IoT telemetry → KDS → Firehose → Redshift (for dashboards)
- App logs → KDS → Lambda processing → S3
