AWS Config is a **configuration tracking and compliance auditing service**.

It monitors how AWS resources change over time and evaluates whether they comply with defined rules.

---

# **1. What AWS Config Does**

### ✔ Tracks configuration **changes** to AWS resources

Every modification—security group changes, bucket policy updates, subnet route changes—is recorded.

### ✔ Provides **history** of resource configurations

You can see:

- What changed
- When it changed
- How it changed
- Who changed it (via CloudTrail integration)

### ✔ Evaluates resource **compliance**

Using **Config Rules**, Config checks whether resources follow your security and governance policies.

Examples:

- “Is SSH open to the world (0.0.0.0/0)?”
- “Are S3 buckets publicly accessible?”
- “Is encryption enabled on EBS volumes?”
- “Has my ALB config changed unexpectedly?”

### ✔ Sends notifications

SNS alerts when resources become **noncompliant** or configurations change.

### ✔ Stores data in S3

All configuration history can be written to S3, where it can be analyzed with Athena.

---

# **2. AWS Config Rules**

Two types:

- **AWS Managed Rules**
(e.g., s3-bucket-public-read-prohibited)
- **Custom Rules** using Lambda

Rules evaluate whether resources meet defined compliance requirements.

---

# **3. AWS Config Architecture Characteristics**

- **Regional service**, but supports **multi-account aggregation**
- Aggregators combine compliance data across:
    - Multiple accounts
    - Multiple regions

Useful for organizations with many environments.

---

# **4. Use Cases**

AWS Config helps answer questions like:

- “Which security groups allow unrestricted SSH?”
- “Did someone open a port on my ALB or change routing tables?”
- “Are all my S3 buckets encrypted?”
- “What was the configuration of this resource last week?”
- "Who changed this setting?"

Ideal for:

- Compliance auditing
- Change management
- Security posture monitoring

### **1. Need to detect configuration changes?**

✔ AWS Config

### **2. Need continuous compliance monitoring?**

✔ AWS Config

### **3. Need historical resource configuration?**

✔ AWS Config

### **4. Need notifications of configuration drift?**

✔ AWS Config (with SNS)

### **5. Need to evaluate S3 bucket access or SG settings?**

✔ AWS Config Rule

### **6. What AWS Config does *****not***** do**

- It does **not** block changes
- It does **not** perform threat detection (GuardDuty does)
- It does **not** patch or remediate resources (SSM does that)

It only **records**, **evaluates**, and **reports**.
