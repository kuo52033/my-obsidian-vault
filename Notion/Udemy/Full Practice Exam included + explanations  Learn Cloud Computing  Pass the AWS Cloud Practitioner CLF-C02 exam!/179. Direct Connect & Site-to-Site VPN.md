Hybrid connectivity allows your **on-premises data center** to connect to your **AWS VPC**.

There are two main options:

---

# **1. Site-to-Site VPN**

A Site-to-Site VPN creates an **encrypted connection over the public internet** between your on-prem network and AWS.

### **Characteristics**

- Encrypted IPsec VPN
- Travels over **public internet**
- Bandwidth depends on internet quality
- Possible latency fluctuations
- **Fast to set up (minutes)**
- Low cost
- Good for:
    - Quick connectivity
    - Backup/DR link
    - Smaller workloads

### **Architecture**

To set up:

| Component | Location | Description |
| --- | --- | --- |
| **Customer Gateway (CGW)** | On-premises | Your hardware/software VPN device |
| **Virtual Private Gateway (VGW)** | AWS side | AWS endpoint for VPN |

Connection established between **CGW ↔ VGW**.

> CLF MUST KNOW: Site-to-Site VPN = CGW + VGW

---

# **2. AWS Direct Connect (DX)**

A dedicated **private physical connection** between your on-prem infrastructure and AWS.

### **Characteristics**

- Private network (does **not** use internet)
- Stable, consistent bandwidth
- Supports 1 Gbps / 10 Gbps / 100 Gbps links
- Secure and predictable performance
- Takes **weeks to months** to provision
- More expensive
- Best for:
    - Large-scale enterprise workloads
    - Big data migrations
    - Latency-sensitive applications

### **Architecture**

On-premises → Direct Connect Partner → AWS DC location → Your VPC

DX can optionally be combined with VPN for extra encryption (DX + VPN).

---

# **3. VPN vs Direct Connect — Key Comparison**

| Feature | Site-to-Site VPN | Direct Connect |
| --- | --- | --- |
| Path | Public internet | Private dedicated circuit |
| Encryption | ✔ Built-in | ❌ Not by default (add VPN if needed) |
| Setup time | Minutes | Weeks–Months |
| Cost | Low | High |
| Reliability | Varies (depends on ISP) | Very stable |
| Bandwidth | Limited | High capacity |
| Use case | Quick setup, backup | Enterprise workloads, stable traffic |

---

# ⭐ **CLF-C02 Exam Tips**

### **1. Fast setup needed?**

✔ Site-to-Site VPN

### **2. Private, stable, high-bandwidth connection needed?**

✔ Direct Connect

### **4. Required components for Site-to-Site VPN**

- **Customer Gateway (CGW)** → on-prem device
- **Virtual Private Gateway (VGW)** → AWS endpoint

Expect exam questions like:

> “What two components are required to set up a Site-to-Site VPN?”
> ✔ **CGW + VGW**

---

# **4. Bonus: Hybrid Best Practices (not heavy but helpful for exam)**

- Many customers start with VPN → later upgrade to DX
- DX can be paired with a VPN for **encryption + failover**
- DX does not provide encryption by default
