## **Overview**

AWS Trusted Advisor is a **fully managed best-practice checker** for your AWS account.

No installation is required. It analyzes your environment and provides recommendations across several categories.

It helps answer questions like:

- Do you have **public EBS snapshots**?
- Are **RDS snapshots** publicly accessible?
- Is the **root user** being used?
- Are you close to **service limits**?

---

## **Recommendation Categories (6 Pillars)**

Trusted Advisor checks are grouped into:

1. **Cost Optimization**
    - Unused EC2 instances
    - Underutilized RDS
    - Idle load balancers
    - Unassociated Elastic IPs
2. **Performance**
    - High utilization resources
    - EC2 instance type mismatches
    - Service bottlenecks
3. **Security**
    - MFA on root
    - Public S3 buckets
    - Public snapshots (EC2/RDS)
    - IAM password policy checks
4. **Fault Tolerance**
    - EBS snapshot presence
    - RDS Multi-AZ
    - Auto Scaling configuration
5. **Service Limits**
    - Alerts when usage approaches regional AWS quotas
    - Example: EC2 instance limit reaching 80%
6. **Operational Excellence**
    - Follows AWS Well-Architected best practices
    - Checks for operational readiness

| Support Plan | Checks Available |
| --- | --- |
| **Basic Support (free)** | **Core Checks only** (limited set) |
| **Developer Support** | Core checks only |
| **Business Support** | **Full set of Trusted Advisor checks** |
| **Enterprise Support** | Full checks + API access |

### ✔ Full set includes:

- All cost optimization checks
- All security checks
- Service limit checks
- Additional performance/fault tolerance checks

→ Automate compliance workflows

## **Exam Tips**

- Trusted Advisor provides **best practice recommendations**.
- Categories: **Cost, Performance, Security, Fault Tolerance, Service Limits, Operational Excellence**.
- **Full checks require Business or Enterprise Support**.
- Basic/Developer support = **Core checks only**.
- API access = **Business/Enterprise support** only.
