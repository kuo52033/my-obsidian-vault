## **1. What is CloudWatch Metrics?**

CloudWatch Metrics helps you monitor the performance and health of your AWS resources.

- A **metric** is a variable you want to monitor
e.g., `CPUUtilization`, `NetworkIn`, `NetworkOut`, etc.
- Metrics include **timestamps** and change over time.
- You can visualize metrics with **CloudWatch Dashboards**.

### **Common AWS Metrics**

- **EC2 Metrics**
    - `CPUUtilization` — how busy your CPU is
    - `StatusCheckFailed` — health of EC2 instance
    - `NetworkIn` / `NetworkOut` — traffic going in/out
    - **Note:** **RAM usage is NOT available** by default (must install CloudWatch Agent)
- **EBS Metrics**
    - Disk read/write throughput & operations
- **S3 Metrics**
    - Bucket size (bytes)
    - Number of objects
    - Number of requests
- **Billing Metric**
    - Available ONLY in **us-east-1**
    - Shows the **estimated total AWS charges**, resets monthly
- **Service Limits Metric**
    - Shows API usage vs. service quotas
- **Custom Metrics**
    - You can push your own metrics using CloudWatch API.

### **Standard vs Detailed Monitoring**

| Type | Frequency | Cost |
| --- | --- | --- |
| **Standard Monitoring** | Every **5 minutes** | Free |
| **Detailed Monitoring** | Every **1 minute** | Additional cost |

> EC2 Auto Scaling needs faster metrics → Detailed monitoring commonly used.

## **2. What are CloudWatch Alarms?**

CloudWatch Alarms let you trigger actions when a metric crosses a threshold.

### **Alarm Actions**

- **Auto Scaling Actions**
Automatically increase or decrease instance count.
- **EC2 Actions**
    - `Stop`
    - `Terminate`
    - `Reboot`
    - `Recover` (only on supported instance types)
- **SNS Notifications**
Send email, SMS, HTTP messages, etc.

### **Examples**

- If EC2 CPU > **90%**, send email via SNS
- If Billing > **$20**, send SNS notification
- If disk space low, trigger recovery action

### **Alarm States**

| State | Meaning |
| --- | --- |
| **OK** | Everything normal |
| **ALARM** | Threshold breached |
| **INSUFFICIENT_DATA** | Not enough datapoints yet |

### **Evaluation Parameters**

- You can configure:
    - Period (e.g., 1 min, 5 min, 1 hour)
    - Statistic type (sum, avg, min, max, percentile)
    - Sampling window

# ⭐ **CLF-C02 Exam Tips (Important!)**

### **CloudWatch Metrics**

- Every AWS service publishes metrics to CloudWatch automatically
- EC2 RAM is *not available* by default → **must use CloudWatch Agent**
- Billing metric is **only in us-east-1**
- Standard monitoring = **5-min** interval
- Detailed monitoring = **1-min** interval (costs more)

### **CloudWatch Alarms**

- Can trigger **Auto Scaling**, **EC2 actions**, or **SNS notifications**
- Billing alarms don’t stop resources — they only *notify*
- Alarm states: **OK**, **ALARM**, **INSUFFICIENT_DATA**
- Custom metrics cost extra because CloudWatch charges per API request

### **Common exam pattern:**

> “Which AWS service is used to monitor metrics and trigger notifications when they cross a threshold?”
> **Answer: CloudWatch Alarms**

> “Where can you find the total estimate of your AWS usage cost?”
> **Answer: Billing metric in CloudWatch (region: us-east-1)**

> “How do you monitor CPU, network traffic, and status checks of an EC2 instance?”
> **Answer: CloudWatch Metrics**
