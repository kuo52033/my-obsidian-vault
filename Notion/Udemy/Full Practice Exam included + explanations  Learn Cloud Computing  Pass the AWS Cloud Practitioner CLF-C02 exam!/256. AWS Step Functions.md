## **What is AWS Step Functions?**

AWS Step Functions is a **serverless orchestration service** used to build **visual workflows** that coordinate multiple AWS services.

Most commonly used to orchestrate **AWS Lambda functions**, but it supports **many other services**.

---

## **Key Idea**

> Step Functions = workflow orchestration + state management

Instead of writing complex logic in code, you **design workflows visually**.

---

## **How It Works**

- You define a **state machine** (workflow)
- Each step represents a task
- You define:
    - What happens on **success**
    - What happens on **failure**
- AWS handles execution, retries, and state tracking

---

## **Core Features**

Step Functions natively supports:

- **Sequencing** (step A → step B → step C)
- **Parallel execution**
- **Conditional logic (if / else)**
- **Timeouts**
- **Retries & error handling**
- **State tracking**
- **Human approval steps**

---

## **Supported Integrations**

Step Functions can orchestrate:

- AWS Lambda
- Amazon EC2
- Amazon ECS tasks
- API Gateway
- Amazon SQS
- On-premises servers
- Many other AWS services

⚠️ **Exam tip:** You do NOT need to memorize all integrations.

---

## **Human Approval Workflow**

Step Functions can pause execution and wait for **manual approval**.

Example:

- Workflow reaches approval step
- Human reviews results
- Decision:
    - **Approve → continue**
    - **Reject → fail workflow**

---

## **Common Use Cases**

- Order fulfillment
- Data processing pipelines
- Web application workflows
- Multi-step business processes
- Complex serverless workflows

---

## **Why Use Step Functions?**

- Avoid complex Lambda code
- Clear visualization of workflows
- Built-in reliability
- Easier debugging & monitoring
- Fully serverless
