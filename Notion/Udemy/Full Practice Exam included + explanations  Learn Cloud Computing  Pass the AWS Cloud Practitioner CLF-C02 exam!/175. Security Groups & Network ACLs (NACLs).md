## **1. Network Security Layers in a VPC**

AWS provides two firewall mechanisms:

1. **Network ACL (NACL)** – subnet-level
2. **Security Group (SG)** – instance-level

They work together to control inbound/outbound traffic.

---

# **2. Network ACL (NACL)** – *Subnet-level firewall*

### **Key Characteristics**

- Applied at the **subnet** level
- Supports **ALLOW** and **DENY** rules
- **Stateless**: return traffic must be explicitly allowed
- Rules processed in order (lowest number first)
- Rules based on **IP addresses** only
- Default VPC NACL:
    - Allows all inbound traffic
    - Allows all outbound traffic

### **When used**

- Broad control over multiple instances
- Blocking specific IP addresses (because SG cannot deny)
- Additional layer of protection for security-sensitive environments

---

# **3. Security Group (SG)** – *Instance-level firewall*

### **Key Characteristics**

- Attached directly to **EC2 instances** (and other resources)
- **Only ALLOW rules**, no deny
- **Stateful**: return traffic is automatically allowed
- Can reference:
    - IP ranges
    - **Other security groups** (very common pattern)

### **When used**

- Fine-grained per-instance security
- Allow web traffic, SSH, database internal communication
- Microservice-to-microservice access control

---

# **4. SG vs NACL — The Key Differences**

| Feature | Security Group | NACL |
| --- | --- | --- |
| **Level** | Instance-level | Subnet-level |
| **Rules** | Only **ALLOW** | **ALLOW + DENY** |
| **State** | **Stateful** | **Stateless** |
| **Return traffic** | Automatically allowed | Must be explicitly allowed |
| **References** | Can reference other SGs | IP-based only |
| **Default behavior** | Deny everything unless allowed | Allow everything (default NACL) |

### 🌟 **3 Concepts CLF-C02 Loves:**

3. **SG = Instance-level, Allow-only, Stateful**
4. **NACL = Subnet-level, Allow+deny, Stateless**
5. **NACL can block specific IPs**, SG cannot

---

# **5. Default NACL Example**

Inbound:

- Rule 100 → Allow all
- Rule * → Deny

Outbound:

- Rule 100 → Allow all
- Rule * → Deny

This effectively allows all traffic.

You can customize by creating new rules, e.g.:

- Allow HTTPS only
- Deny specific IP ranges

# ⭐ **CLF-C02 Exam Tips**

### **1. “Stateful vs Stateless”**

> If exam mentions return traffic automatically allowed → Security Group
> If exam mentions **must allow return traffic explicitly** → **NACL**

---

### **2. “Can I block an IP?”**

SG: ❌ **No**

NACL: ✔ **Yes**

---

### **3. “Which is applied first?”**

Traffic hits:

6. **NACL** (subnet boundary)
7. **Security Group** (instance boundary)

---

### **4. Typical Exam Questions**

> “Which control allows both ALLOW and DENY?”
> ✔ Network ACL

> “Which rule type can reference another security group?”
> ✔ Security Group

> “Traffic is automatically allowed back on response—what is this?”
> ✔ Security Group (stateful)

> “Apply a broad firewall rule to multiple instances at once.”
> ✔ Network ACL
