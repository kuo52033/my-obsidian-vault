## What is Amazon MQ?

Amazon MQ is a **managed message broker service** for:

- **RabbitMQ**
- **ActiveMQ**

These are traditional, open-standard messaging systems commonly used **on-premises**.

---

## Why Amazon MQ Exists

SQS / SNS are **cloud-native**, use AWS’s own APIs and protocols.

Many legacy applications use open protocols such as:

- **MQTT**
- **AMQP**
- **STOMP**
- **OpenWire**
- **WSS**

When migrating to AWS, companies often **cannot rewrite** their applications to use SQS or SNS.

➡ **Amazon MQ enables “lift-and-shift” migration**

because it supports the *same open protocols* without code change.

---

## How Amazon MQ Works

Amazon MQ runs **message brokers on servers** (RabbitMQ / ActiveMQ).

It supports both:

- **Queues** → like SQS
- **Topics** → like SNS

All in one broker.

---

## Key Characteristics

### ✔ Supports open protocols

Required for legacy or enterprise systems.

### ✔ Managed by AWS

AWS handles:

- patching
- failover
- monitoring
- maintenance

### ✔ Multi-AZ for HA

Because brokers run on servers, you must deploy **multi-AZ** for high availability.

### ✔ Does NOT scale like SQS/SNS

Amazon MQ has **broker limits**, unlike SQS and SNS which scale almost infinitely.

---

## Amazon MQ vs SQS / SNS

| Feature | Amazon MQ | SQS | SNS |
| --- | --- | --- | --- |
| Protocols | MQTT, AMQP, STOMP, OpenWire | AWS proprietary | AWS proprietary |
| Model | Queue + Topic | Queue | Pub/Sub |
| Scaling | Limited (broker-based) | Massive, auto-scaling | Massive, auto-scaling |
| Use case | Legacy migration, open standards | Decoupling workers | Fan-out notifications |
| HA | Multi-AZ brokers | Built-in | Built-in |

## Exam Tips

- “Company uses RabbitMQ / ActiveMQ on-premises” → **Amazon MQ**
- “Needs MQTT, AMQP, STOMP, OpenWire, WSS” → **Amazon MQ**
- “Wants cloud-native, scalable messaging” → **Use SQS/SNS instead**
- “Moving enterprise legacy systems to AWS without code changes” → **Amazon MQ**
