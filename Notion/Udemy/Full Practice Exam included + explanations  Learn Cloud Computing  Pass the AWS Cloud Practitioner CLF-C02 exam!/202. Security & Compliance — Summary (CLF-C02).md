這是整個 Security & Compliance 章節的大總結。你只需要掌握以下關鍵服務的用途與差異即可應付考題。

---

# **🔒 1. Shared Responsibility Model**

- **AWS – Security *****of***** the Cloud**
Infrastructure、hardware、facility、hypervisor、managed services（S3/RDS/DynamoDB…）
- **Customer – Security *****in***** the Cloud**
OS patching（EC2）、SG/NACL、IAM、application-level controls、data encryption choices

👉 考點：

「誰負責 patch OS？」「誰負責資料加密？」「誰負責資料分類？」

→ 取決於 *是否為 managed service*。

---

# **🛡️ 2. DDoS Protection**

- **Shield Standard**：自動啟用，免費基礎 DDoS 防護（L3/L4）
- **Shield Advanced**：付費（約 $3000/mo），24/7 DDoS Response Team + 更強防護
- **WAF**：Layer 7 HTTP 防護（SQL injection, XSS, rate limiting）
- **CloudFront / Route 53**：借用 edge network 減少攻擊衝擊

👉 考題：

「如何保護 Web Application？」→ CloudFront + WAF + Shield。

---

# **🔑 3. Encryption Services**

### **KMS**

- AWS 主要的加密服務
- 用於 encryption at rest / in transit
- Key 類型：Customer-managed keys、AWS-managed keys、AWS-owned keys

### **CloudHSM**

- 專屬硬體 HSM（FIPS 140-2 Level 3）
- 客戶自行管理金鑰
- AWS 只負責硬體，不接觸 key material

👉 考題：

「需完全掌控加密金鑰」→ CloudHSM

「一般 AWS 服務加密」→ KMS

---

# **🔐 4. Certificates**

### **ACM — AWS Certificate Manager**

- 自動申請 / 管理 / 更新 SSL/TLS 憑證
- Load Balancer / CloudFront / API Gateway 直接整合
- Public certificates **免費**

👉 考題：

「如何提供 HTTPS？」→ ACM + Load Balancer / CloudFront

---

# **📄 5. Compliance & Monitoring**

### **CloudTrail**

- 記錄所有 API calls（CLI/SDK/Console）
- 用於 audit、安全分析、追查「誰做了什麼」

### **Config**

- 記錄所有 AWS 資源的「設定變更歷史」
- 可建立 **Config Rules** 檢查 compliance
- 範例：S3 不可 public、SG 不可允許 0.0.0.0/0 SSH

### **Artifact**

- 下載 AWS 合規文件（SOC、ISO、PCI…）
- 也可查看/接受法律協議（BAA、HIPAA 等）

---

# **🧠 6. Threat Detection & Vulnerability Analysis**

### **GuardDuty**

- ML & threat detection：分析
    - VPC Flow Logs
    - DNS logs
    - CloudTrail
- 自動偵測惡意行為（含 Crypto mining）

### **Inspector**

- 自動偵測漏洞（CVE）：
    - EC2（透過 SSM agent）
    - ECR image
    - Lambda functions

### **Macie**

- 專門掃描 S3，找出 **PII / 敏感資料**

### **Security Hub**

- 集中所有安全 findings 的 dashboard
- 聚合來源：Config / GuardDuty / Inspector / Macie / Firewall Manager …
- **必須啟用 Config 才能使用**

### **Detective**

- 深度分析 & 根因調查
- 對應：Security Hub → Detective（調查問題來源）

---

# **🌐 7. Network Security**

### **Network Firewall**

- VPC 層級的全向防火牆（L3–L7）
- 可保護所有流量（Inbound / Outbound / 點對點）

### **Firewall Manager**

- Org 等級，集中管理 **Security Groups / WAF / Shield / Network Firewall**
- 適合多帳號、多環境統一規則

---

# **🚫 8. Abuse Reporting**

- 若發現 AWS IP/資源被用於 spam、DDoS、scan、malware
→ 回報至 **abuse@amazonaws.com**

---

# **👑 9. Root User – 必考四項權限**

只能 root user 執行：

1. 修改 account 設定（email、name、root password）
2. 關閉 AWS account
3. 訂閱/取消 AWS Support Plan
4. 登記成為 Reserved Instance Marketplace 的 seller（轉賣 RI）

---

# **🔍 10. IAM Access Analyzer**

- 找出哪些 AWS 資源「被外部帳號」或「公開」存取
- 檢查：S3 / Role / KMS / Lambda / SQS / Secrets Manager
- 基於 **resource policies** 分析
- 用來找出 **unintended public exposure**

👉 考題：

「如何找出意外公開的 S3 bucket？」→ IAM Access Analyzer
