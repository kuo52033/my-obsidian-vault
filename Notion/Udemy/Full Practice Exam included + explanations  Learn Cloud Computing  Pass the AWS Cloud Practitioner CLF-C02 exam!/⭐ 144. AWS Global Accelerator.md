## What is AWS Global Accelerator?

AWS Global Accelerator (GA) improves **global application availability and performance** by routing user traffic through the **AWS global private network** instead of the unpredictable public internet.

---

## How It Works

1. Users connect to **two static Anycast IPs** provided by Global Accelerator.
2. Traffic is automatically routed to the **nearest AWS Edge Location**.
3. The edge location forwards the traffic through AWS's **private, optimized backbone** to your application endpoint.

This reduces latency and avoids unreliable internet routing.

---

## Why It’s Faster

Without GA:

Client → many public internet hops → AWS region → your app

→ unpredictable latency, congestion, packet loss.

With GA:

Client → nearest edge location → AWS global network → your app

→ highly optimized, fewer hops, more stable.

AWS claims up to **60% performance improvement** depending on geography.

---

## Benefits

- Uses AWS global network (low latency, high throughput)
- Provides **two static Anycast IPs** for your entire application
→ easier DNS + stable client entry points
- Automatic traffic routing to closest edge location
- Faster cross-region performance
- Supports **TCP and UDP**
- Built-in DDoS protection via AWS Shield

---

## When to Use Global Accelerator

Use it when:

- Users are **globally distributed**
- You need **low latency** for dynamic, non-cacheable content
- You require **static IP addresses** for your app
- You want **fast regional failover** (GA can switch endpoints in seconds)

Not ideal when:

- Your content is static and cacheable (→ use CloudFront instead)
- Users are mostly in one region

---

## Supported Endpoints

Global Accelerator can route traffic to:

- Application Load Balancer (ALB)
- Network Load Balancer (NLB)
- EC2 instances
- Elastic IPs
- Private endpoints inside VPCs

---

## Global Accelerator vs CloudFront (Important for Exam)

| Feature | Global Accelerator | CloudFront |
| --- | --- | --- |
| Purpose | Improve global performance for **dynamic or interactive traffic** | **Cache static content** at edge locations |
| Caching | ❌ No caching | ✅ Yes |
| Protocols | TCP + UDP | HTTP/HTTPS |
| Improves | Application routing performance | Content delivery speed |
| Static IPs | ✅ Yes (Anycast) | ❌ No |
| Best for | APIs, gaming, financial apps, real-time services | Images, videos, static files, websites |

# 🟦 💡 核心差異一句話版

| 服務 | 它加速什麼 | 是否有快取 | 適用場景 |
| --- | --- | --- | --- |
| **CloudFront** | 靜態內容 & API 的讀取 | ✔ 有 | 靜態網站、影片、圖片、API |
| **S3 Transfer Acceleration** | 專門加速 S3 的上傳/下載 | ❌ 無 | App 圖片上傳、大檔案跨國傳輸 |
| **Global Accelerator (GA)** | 加速整體 TCP/UDP 流量（L4） | ❌ 無 | 加速 API/網站的網路傳輸（非 cache） |

- **CloudFront = 加快“資料回應”**（因為 cache）
- **Transfer Acceleration = 加快“S3 上傳/下載管道”**
- **Global Accelerator = 加快“客戶端 → 你的後端”的連線**
